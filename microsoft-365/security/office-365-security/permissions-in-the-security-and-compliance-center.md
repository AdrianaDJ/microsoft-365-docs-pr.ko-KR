# <a name="permissions-in-the-security--compliance-center"></a>보안 및 준수 센터의 사용 권한

보안 그룹 & 준수 센터를 사용하면 장치 관리, 데이터 손실 방지, eDiscovery, 보존 등과 같은 준수 작업을 수행하는 사람에게 권한을 부여할 수 있습니다. 이러한 사용자들은 명시적으로 액세스 권한을 부여 받은 작업만 수행할 수 있습니다. 규정 준수 센터에서 & 사용자는 전역 관리자이거나 준수 센터 역할 그룹에서 하나 이상의 보안 & 합니다.

보안 센터의 & 부여된 사용 권한은 RBAC(역할 기반 액세스 제어) 사용 권한 모델을 기반으로 합니다. 이것은 Exchange에서 사용되는 것과 동일한 사용 권한 모델이므로 Exchange에 익숙한 경우 보안 보안 규정 준수 센터에서의 & 사용 권한 부여도 매우 유사합니다. 그렇지만 Exchange 역할 그룹 및 보안 & 규정 준수 센터 역할 그룹은 구성원 자격 또는 사용 권한을 공유하지 않습니다. 두 역할 그룹이 모두 조직 관리 역할 그룹을 갖지만 둘은 다른 것입니다. 부여하는 사용 권한과 역할 그룹의 구성원이 서로 다릅니다. 아래 준수 센터 역할 그룹의 보안 & 프로그램이 있습니다.

## <a name="relationship-of-members-roles-and-role-groups"></a>구성원, 역할 및 역할 그룹의 관계

**역할**은 특정 작업 집합을 수행하기 위한 권한을 부여합니다. 예를 들어 사례 관리 역할은 eDiscovery 사례를 사용할 수 있도록 합니다.

역할 **그룹은** 사용자가 준수 센터에서 작업을 수행할 수 있도록 & 해줍니다. 예를 들어 준수 관리자는 사례 관리, 콘텐츠 검색, 조직 구성 작업 및 작업을 수행하기 위한 권한이 필요하므로 준수 관리자 역할 그룹에는 해당 역할 및 기타 역할이 포함됩니다.

보안 & 보안 센터에는 사용자에게 할당해야 하는 가장 일반적인 작업 및 기능에 대한 기본 역할 그룹이 포함되어 있습니다. 개별 사용자를 구성원으로 기본 역할 그룹에 **추가하기만** 하는 것이 좋습니다.
