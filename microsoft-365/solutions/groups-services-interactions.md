# <a name="groups-services-interactions"></a>그룹 서비스 상호 작용

Microsoft 365 그룹은 Microsoft 365 플랫폼 내의 다양 한 서비스 및 작업에 대 한 공통 패브릭을 제공 하 여 최종 사용자에 게 연결 된 환경을 제공할 수 있도록 합니다. 핵심은 Microsoft 365 그룹이 제공 하는 것입니다.

- 구성원 자격 (Azure AD)을 관리 하는 방법
- 메시징과 대화를 수행할 장소 (Exchange 사서함, Microsoft 팀, Yammer)
- 파일을 저장할 위치 (SharePoint)
- 일정 예약 (Exchange)
- 메모를 캡처하기 위한 전자 필기장 (OneNote)

그룹을 만들 때 여러 다른 리소스가 프로 비전 되 고 서비스에서 처음으로 액세스 하기 전까지 표시 되지 않습니다.

- 그룹 작업을 관리 하는 보드 (Planner)
- 보고를 위한 작업 영역 (Power BI)
- 공유 비디오 영역 (Microsoft Stream)
- 공유 폼 영역 (폼)

Microsoft 365에서 다른 서비스는 Microsoft 365 그룹과 상호 작용 하 여 그룹 구성원에 게 추가 기능과 기능을 제공할 수 있습니다.
예를 들면 다음과 같습니다.

- 앱 용 파워 앱
- 워크플로의 전원 자동화
- 웹에 있는 프로젝트 및 폭포 기반 프로젝트 관리 로드맵
- 채널 기반 대화 팀
- 관심 커뮤니티를 위한 Yammer

## <a name="user-interactions-with-groups"></a>그룹과의 사용자 상호 작용

Microsoft 365 그룹은 관리자와 최종 사용자가 모두 다양 한 인터페이스에서 만들고 관리할 수 있습니다. 

### <a name="administrative-experiences"></a>관리 환경

관리자는 Graph API와 상호 작용 하는 사용자 지정 구성 된 앱 뿐 아니라 스크립팅을 지 원하는 여러 작업 관리 센터, 명령줄 인터페이스에서 Microsoft 365 그룹을 만들고 관리할 수 있습니다. 단, yammer 웹 인터페이스 내에서 만들어야 하는 Yammer 그룹은 여기에서 제외 됩니다.

**관련 설정**

그룹 설정을 관리할 수 있는 다양 한 관리 인터페이스에 걸쳐 몇 가지 중복 사항을 알고 있어야 합니다.

**Microsoft 365 관리 센터**

Microsoft 365 관리 센터에서 그룹에 대 한 게스트 액세스는 기본적으로 사용 하도록 설정 되며 소유자가 게스트를 추가할 수 있도록 합니다. 이 관리 센터에서 그룹에 대해 사용할 수 있는 조직 수준 컨트롤이 더 이상 없습니다.

**Azure AD 관리 센터**

Azure AD 관리 센터는 사용자가 Azure 포털에서 그룹을 만들거나 소유자를 할당 하 고 만료 및 이름 지정 정책 설정을 지정할 수 있는지 여부에 대 한 제어를 제공 합니다.

관리 센터는 또한 비 소유자가 게스트를 초대할 수 있는지 여부를 제한 하는 기능과 같이 Microsoft 365 관리 센터의 범위를 벗어나는 여러 가지 게스트 초대 제어 조치를 제공 합니다.

**SharePoint**

SharePoint 사이트는 소유자, 구성원 및 방문자 보안 그룹을 사용 하 여 작성 되며 처음 두 개는 해당 Microsoft 365 그룹을 포함 합니다. 일반적으로 SharePoint Online 사이트의 구성원 자격은 연결 된 Microsoft 365 그룹에 의해 관리 되지만 양방향 관계는 아닙니다. Microsoft 365 그룹 수준의 멤버 자격에 대 한 변경 내용은 SharePoint에 반영 되지만 SharePoint 그룹에서 구성원 자격이 변경 되는 경우에는 Microsoft 365 그룹에 반영 되지 않습니다.

### <a name="user-experiences"></a>사용자 환경

최종 사용자는 Microsoft 365 내의 몇 가지 서비스에서 그룹을 만들 수 있으며,이 경우에는 그룹과만 공유할 수 있습니다.

최종 사용자가 그룹을 만들 수 있도록 하는 서비스는 다음과 같습니다.
                         
웹용 Outlook Planner Project for SharePoint Stream Microsoft 팀별로 Yammer

**그룹 만들기 제한**

팀의 sprawl을 제어 하는 일반적인 방법은 사용자를 만들 수 있는 사람을 제한 하는 것입니다. 이 작업은 그룹 만들기를 제한 해야만 수행할 수 있습니다. 이렇게 하면 최종 사용자에 게 필요할 수 있는 다른 서비스에서 그룹을 만드는 기능이 영향을 받습니다. Microsoft 365 그룹에서는 일부 앱 이나 서비스에서 그룹을 만드는 것을 허용 하는 동시에 해당 사용자를 제한 하는 기능을 지원 하지 않습니다.

앱과 서비스에 따라 그룹 만들기 제한의 환경은 다양 합니다.


|앱 또는 서비스|환경|

|SharePoint|아직 그룹에 연결 되어 있지 않은 팀 사이트를 만들 수 있습니다.|

## <a name="azure-ad"></a>Azure AD

Azure AD는 Microsoft 365에서 기본 id 관리 기능을 제공 합니다.

**그룹에 제공 되는 주요 기능**

- 그룹 구성원 자격
- 명명 정책
- 만료 정책
- 게스트
- 그룹 만들기 제한

**Azure AD에서 그룹을 만들 수 있나요?**

예, Microsoft 365 그룹은 관리 웹 포털, PowerShell 또는 Graph API를 통해 Azure AD에서 만들 수 있습니다.

**Azure AD가 그룹 없이 존재 합니까?**

예, Azure AD는 Microsoft 365 그룹과 관계가 없는 많은 서비스를 수행 합니다. 각 Microsoft 365 그룹은 Azure AD에서 개체로 표시 됩니다.

**그룹 당 Azure AD의 인스턴스가 여러 개 있을 수 있나요?**

아니요, Azure AD 인스턴스가 하나만 있습니다.

**Azure AD는 여러 그룹과 연결 될 수 있나요?**

예, Azure AD는 그룹 멤버 자격 서비스를 제공 하는 기본 플랫폼 이기 때문입니다.

**Azure AD와 그룹 변경의 연결 가능 여부**

아니요, Azure AD는 그룹이 있는 기본 플랫폼입니다.

**인스턴스를 삭제 하 여 그룹을 삭제 하 시겠습니까?**

Azure AD에서 그룹을 삭제 하면 관련 그룹 관련 서비스 및 콘텐츠가 삭제 됩니다.

## <a name="teams"></a>Teams

팀은 다양 한 Microsoft 및 타사 서비스와 상호 작용 하기 위한 단일 인터페이스를 제공 하 여 공동 작업의 향상을 목표로 하는 채팅 중심 작업 영역입니다.

기본적으로 팀을 만들 때 Microsoft 365 그룹과 연결 된 사서함 및 일정은 Exchange의 전체 주소 목록 및 Outlook에서 모두 숨겨집니다. 사용자가 같은 Microsoft 365 그룹에서 Outlook과 팀을 모두 사용 하려는 경우 관리자가 수동으로이를 재정의할 수 있습니다.

**그룹에 제공 되는 주요 기능**

- Conversations
- 채널 & 탭
- Meetings

**팀에서 그룹을 만들 수 있습니까?**

예, 새 팀을 만들면 새 Microsoft 365 그룹이 만들어집니다. 현재 없는 기존 그룹에 대 한 팀을 만들 수도 있습니다.

**그룹을 사용 하지 않고 팀이 존재 합니까?**

아니요, 그룹 없이 팀이 존재 하지 않을 수 있습니다.

**그룹당 여러 팀이 될 수 있나요?**

아니요, 팀과 그룹의 관계는 1:1입니다.

**팀이 여러 그룹과 연결 될 수 있나요?**

아니요, 팀 자체는 단일 그룹에만 연결할 수 있습니다.

**팀이 그룹 변경에 연결할 수 있나요?**

아니요, 팀은 원래 연결 되었던 그룹에만 연결할 수 있습니다.

**팀을 삭제 하 여 그룹을 삭제 하 시겠습니까?**

예, Microsoft 팀에서 팀을 삭제 하면 그룹, 그룹에 연결 된 서비스 및 콘텐츠가 삭제 됩니다.

## <a name="exchange"></a>Exchange

Exchange Online은 메시징, 일정, 연락처 및 관련 기능을 제공 합니다. 그룹 컨텍스트에서는 전체 서비스 인스턴스와는 달리 단일 리소스가 연결 됩니다.

**그룹에 제공 되는 주요 기능**

- 사서함 및 일정
- 모든 그룹 구성원에 게 전자 메일 보내기 기능
- EDiscovery 목적에 대 한 팀 채널 대화 저장소, Planner 설명

**Exchange에서 그룹을 만들 수 있습니까?**

예, Exchange Online 관리 센터에서 뿐만 아니라 Outlook 에서도 그룹을 만들 수 있습니다. Exchange 메일 그룹을 Microsoft 365 그룹으로 변환할 수도 있습니다.

**그룹 없이 Exchange가 존재 하나요?**

예, Exchange Online은 그룹 연결 없이 공유 사서함 및 일정을 비롯 한 다양 한 서비스를 제공 합니다.

**그룹 당 Exchange 사서함 또는 일정의 인스턴스가 여러 개 있을 수 있습니까?**

아니요, 그룹에 대해 하나의 Exchange Online 사서함과 일정만 있을 수 있습니다.

**Exchange 사서함과 일정을 여러 그룹에 연결할 수 있나요?**

아니요, 사서함과 일정의 범위는 그룹에 1:1 관계가 있습니다. 다른 사용자 또는 그룹과 사서함을 공유할 수는 있지만 어떤 형태의 서비스 연결도 설정 하지는 않습니다.

**Exchange 사서함 또는 일정 그룹과 그룹의 연결을 변경할 수 있습니까?**

아니요, 사서함과 일정을 다른 그룹으로 변경할 수 없습니다. 그러나 Outlook 내에서 또는 타사 도구를 사용 하 여 사서함 간에 콘텐츠를 이동할 수 있습니다.

**사서함을 삭제 하 여 그룹을 삭제 하 시겠습니까?**

예, Exchange에서 사서함을 삭제 하면 그룹 및 그룹에 연결 된 서비스 및 콘텐츠가 삭제 됩니다.

## <a name="forms"></a>Forms

양식은 웹 기반 설문 조사 및 퀴즈를 제공 합니다.

**그룹에 제공 되는 주요 기능**

- 양식의 소유권

**폼에서 그룹을 만들 수 있습니까?**

아니요, 폼에서 그룹을 만들 수 없습니다.

**그룹을 사용 하지 않고 양식이 존재 합니까?**

예, 설문 조사 및 퀴즈는 최종 사용자의 계정에 직접 만들 수 있습니다.

**그룹당 여러 양식을 사용할 수 있나요?**

예, 그룹에 연결 된 양식이 여러 개 있을 수 있습니다.

**양식은 여러 그룹과 연결 될 수 있나요?**

아니요, 양식을 단일 그룹에만 연결할 수 있습니다.

**양식의 그룹과 연결을 변경할 수 있습니까?**

아니요, 양식이 그룹 (바로 아래에서 만들어짐 또는 개인이 보낸 소유권)에 연결 된 경우 다른 그룹으로 이동할 수 없습니다.

**양식을 삭제 하 여 그룹을 삭제 하 시겠습니까?**

아니요, 양식 인터페이스에서 그룹을 삭제할 수 없습니다.

## <a name="onenote"></a>OneNote

OneNote는 디지털 전자 필기장 응용 프로그램입니다. 그룹을 사용 하 여 만든 OneNote 전자 필기장은 그룹 연결 서비스가 아닌 연결 된 SharePoint 사이트의 파일입니다.

**그룹에 제공 되는 주요 기능**

- 공유 전자 필기장 (그룹 연결 SharePoint 라이브러리에 저장)

**OneNote에서 그룹을 만들 수 있나요?**

아니요, OneNote 응용 프로그램에서 그룹을 만들 수 없습니다.

**그룹을 사용 하지 않고 OneNote 전자 필기장이 존재 하나요?**

예, OneDrive 또는 다른 공유 위치에서 직접 전자 필기장을 만들 수 있습니다.

**그룹 마다 OneNote 전자 필기장이 여러 개 있을 수 있습니까?**

예, 기본적으로 전자 필기장이 만들어지며 다른 사용자는 추가할 수 있지만 그룹 연결 서비스에서 OneNote로 연결 하는 링크는 항상 기본 전자 필기장으로 이동 됩니다.

**OneNote 전자 필기장을 여러 그룹에 연결할 수 있나요?**

아니요, 전자 필기장이 그룹 연결 SharePoint 사이트 라이브러리에 저장 되 고 다양 한 인터페이스에서 연결 됩니다. 그러나 다른 그룹과 공유 하는 것과 같은 방법으로 사용자와 공유할 수 있습니다.

**전자 필기장에서 그룹과의 연결이 변경 될 수 있나요?**

아니요, 전자 필기장 자체가 그룹과 연결 되 고 다른 그룹 연결 서비스에서 직접 액세스할 수 있지만 콘텐츠를 OneNote 응용 프로그램 내에서 다른 전자 필기장으로 이동할 수 있습니다.

**전자 필기장을 삭제 하 여 그룹을 삭제 하 시겠습니까?**

아니오, OneNote 전자 필기장을 삭제 한 경우 일부 그룹 관련 서비스에 끊어진 링크가 있을 수 있습니다.

**작업 영역을 삭제 하 여 그룹을 삭제 하 시겠습니까?**

Power BI에서 새 작업 영역을 만들 때 Microsoft 365 그룹을 만들지 않으면 다른 방법으로 그룹을 만들면 Power BI에서 클래식이 아닌 새 작업 영역이 만들어집니다.

**그룹에 제공 되는 주요 기능**

- 데이터 보고

**작업 영역을 삭제 하 여 그룹을 삭제 하 시겠습니까?**

예, Power BI에서 작업 영역을 삭제 하면 그룹 및 그룹에 연결 된 서비스 및 콘텐츠가 삭제 됩니다.

예, SharePoint에서 사이트를 삭제 하면 그룹 및 그룹에 연결 된 서비스 및 콘텐츠가 삭제 됩니다.

예 및 아니요, Stream의 비디오는 원래 업 로더 또는 모임 레코더에서 소유 하며, 어떤 그룹과도 연결 될 수 있지만, 비디오 채널은 원래 만든 그룹과 연결 될 수만 있습니다.

**비디오나 채널 삭제가 그룹을 삭제 합니까?**

아니요, 비디오나 채널을 삭제 해도 그룹은 삭제 되지 않습니다. 그러나 Stream에서 그룹 자체를 삭제 하면 실제 비디오를 제외 하 고 그룹 관련 서비스 및 콘텐츠가 삭제 됩니다.

## <a name="yammer"></a>Yammer

Yammer는 조직 내에서 및 조직과 간의 커뮤니티 계약을 촉진 하도록 설계 된 엔터프라이즈 소셜 플랫폼입니다.

Yammer에서 이전에 "그룹" 이라고 알려진 커뮤니티를 만들면 사서함이 만들어지지만 현재는 사용 되지 않습니다.

Yammer와 연결 된 Microsoft 365 그룹은 Microsoft 팀의 팀과 함께 사용할 수 없습니다.

**그룹에 제공 되는 주요 기능**

- 대화 영역

**Yammer에서 Microsoft 365 그룹을 만들 수 있나요?**

예, Yammer에서 새 그룹을 만들면 플랫폼이 연결 되 고 사용자에 게 그룹을 만들 수 있는 경우 새 Microsoft 365 그룹이 만들어집니다.

Microsoft 365 그룹이 연결 된 Yammer 그룹은 Yammer 자체 이외의 인터페이스 또는 서비스에서 만들 수 없습니다.

**Microsoft 365 그룹이 없는 Yammer 그룹이 있습니까?**

예, Microsoft 365 그룹을 사용 하지 않고 Yammer 그룹을 만들 수 있습니다.

Yammer 플랫폼이 Microsoft 365 그룹에 연결 되어 있지 않거나 사용자에 게 microsoft 365 그룹을 만들 수 없는 경우 Microsoft 365 그룹 연결 없이 Yammer 그룹이 만들어집니다.

**Microsoft 365 그룹당 여러 Yammer 그룹을 사용할 수 있나요?**

아니요, Yammer 그룹과 Microsoft 365 그룹 간의 관계는 1:1입니다.

**Yammer 그룹을 여러 Microsoft 365 그룹과 연결할 수 있나요?**

아니요, Yammer 그룹은 단일 Microsoft 365 그룹에만 연결할 수 있습니다. 게시물을 다른 Yammer 그룹에서 공유 하거나 공유할 수 있습니다.

**Yammer 그룹에서 Microsoft 365 그룹을 변경할 수 있습니까?**

아니요, Yammer 그룹은 원래 연결 되었던 Microsoft 365 그룹에만 연결할 수 있습니다.

**Yammer 그룹 365 삭제를 삭제 하 시겠습니까?**

예, Yammer에서 그룹을 삭제 하면 관련 Microsoft 그룹과 그룹에 연결 된 서비스 및 콘텐츠가 삭제 됩니다.
