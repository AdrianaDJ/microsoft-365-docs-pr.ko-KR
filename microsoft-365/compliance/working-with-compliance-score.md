# <a name="working-with-microsoft-compliance-score-preview"></a>Microsoft 준수 점수를 사용한 작업 (미리 보기)

이 문서에서는 준수 점수의 주요 요소로 작업 하는 방법에 대해 설명 합니다. **개선 작업** 을 사용 하 여 준수 워크플로를 관리 하는 방법을 알아봅니다. 또한 **솔루션** 및 **평가** 페이지의 정보를 사용 하는 방법과 보고서를 생성 하는 방법에 대해서도 알아봅니다.

## <a name="manage-your-workflow-with-improvement-actions"></a>개선 작업을 사용 하 여 워크플로 관리

**향상 작업** 을 통해 준수 작업을 중앙에 집중 합니다. 각 향상 작업은 데이터 보호 규정 및 표준과의 맞춤을 지원 하기 위한 자세한 구현 지침을 제공 합니다. 조직의 사용자에 게 작업을 할당 하 여 구현 및 테스트 작업을 수행할 수 있습니다. 개선 작업 내에 설명서, 메모 및 레코드 상태 업데이트를 저장할 수도 있습니다.

## <a name="view-your-improvement-actions"></a>개선 작업 보기

규정 준수 점수 대시보드에는 **주요 향상 작업**, 즉 가장 중요 한 문제를 해결 하는 가장 많이 사용 가능한 점수가 표시 됩니다.

**개선 작업** 페이지에서 모든 개선 작업을 보려면 대시보드에서 **개선 작업** 탭을 선택 합니다. 또는 대시보드의 주요 향상 작업 목록 아래에 있는 **모든 개선 작업 보기** 를 선택 합니다.

작업 목록이 길면 보기를 필터링 하는 것이 유용할 수 있습니다. 작업 목록의 오른쪽 위 모서리에서 **필터** 를 선택 합니다. **필터** 플라이 아웃 창이 나타나면 규정 및 표준, 솔루션, 그룹에 따라 기준을 선택 합니다. 오른쪽 위 모서리에 있는 **그룹** 을 선택 하 여 보기를 사용자 지정할 수도 있습니다. 드롭다운 메뉴에서 그룹, 솔루션, 범주, 작업 유형 또는 상태별로 보기를 선택 합니다.

이 페이지의 기본 보기에는 **통과**테스트 상태의 개선 작업이 표시 되지 않습니다. 테스트에 통과 한 작업을 보려면 **필터** 플라이 아웃 창에서 **통과** 상자를 선택 합니다. 테스트 상태의 **합격** 한 작업만 점수에 대해 계산 됩니다.

향상 작업 페이지에는 각 향상 작업에 대 한 다음과 같은 데이터 요소가 표시 됩니다.

- **점수 영향**: 작업 완료 시 전체 점수가 증가 하는 점수입니다.
- **규정**: 작업과 관련 된 규정 또는 표준
- **그룹**: 작업을 할당 한 그룹입니다.
- **해결**방법: 작업을 수행 하기 위해 이동할 수 있는 솔루션
- **평가**: 작업이 상주 하는 평가 (특정 준수 목표를 충족 하도록 컨트롤을 구성)
- **범주**: 관련 된 데이터 보호 범주 (즉, 정보 보호, 장치 관리 등)
- **테스트 상태**:
    - **없음** -상태 업데이트가 기록 되지 않음
    - **평가 안** 됨-테스트가 시작 되지 않음
    - **범위에 없음** -규정 준수 점수 계산에서 제외 되며 점수가 증가 하지 않습니다.
    - **부분적으로 테스트** 됨-테스트가 아직 완료 되지 않음
    - **실패 한 높은 위험** -구현 테스트에 실패 했으며 해당 표준에 부합 하지 않는 위험이 높습니다.
    - **성공** -구현이 테스트 됨
- **달성**한 것: 진척 상황을 확인할 수 있는 최대 점수를 표시 합니다.

### <a name="improvement-actions-details"></a>향상 작업 세부 정보

각 향상 작업에는 세부 정보 페이지가 있습니다. 이 페이지에서는 **간략 한** 머리글 아래에 나열 된 관련 표준과 규정 요구 사항을 해결 하기 위한 권장 작업을 완료 하기 위한 자세한 구현 지침을 제공 합니다.

세부 정보 페이지에서 권장 작업을 시작할 수 있습니다. 또한 다른 사용자에 게 작업을 할당 하 고, 상태를 업데이트 하 고, 메모 및 문서를 첨부할 수 있습니다.

개선 작업의 세부 정보 페이지를 보려면:

1. 개선 작업 페이지로 이동 합니다.
2. 원하는 향상 작업의 행에서 아무 곳 이나 클릭 하거나 탭 하 여 세부 정보 페이지를 엽니다.

화면 오른쪽 위 모서리에 있는 위쪽 또는 아래쪽 화살표를 선택 하 여 목록에서 다음 또는 이전 개선 작업을 쉽게 확인할 수 있습니다. 개선 작업 페이지에서 목록을 필터링 한 경우 위 또는 아래로 이동 하면 필터링 된 목록 내의 다음 항목으로 이동할 수 있습니다.

## <a name="assign-improvement-actions"></a>개선 작업 할당

개선 작업에 대 한 구현 작업을 시작 하려면 직접 작업을 수행 하거나 다른 사용자에 게 할당할 수 있습니다. 배정 된 사용자는 다음 작업을 수행할 수 있습니다.

- 비즈니스 정책 소유자
- IT 구현자
- 작업 수행을 담당 하는 다른 직원

적절 한 사용자가 식별 되 면 해당 작업을 수행 하기 위해 준수 점수 (준수 관리자, 준수 데이터 관리자, 보안 관리자 또는 전역 관리자)에 충분 한 [역할](compliance-score-setup.md#set-user-permissions-and-assign-roles) 을 보유 하 고 있는지 확인 해야 합니다. 

1. 개선 작업 세부 정보 페이지에서 화면의 왼쪽 위 섹션 부근에 있는 **상태 편집** 을 선택 합니다. 

2. 상태 플라이 아웃 편집 창에서 **할당 대상** 상자를 클릭 하거나 탭 하 여 사용자의 제안 된 **사용자** 목록을 채웁니다. 목록에서 사용자를 선택 하거나 작업을 할당 하려는 사용자의 전자 메일 주소를 입력할 수 있습니다.

3. **저장 후 닫기를** 선택 하 여 할당을 완료 합니다. 할당 된 사용자에 게는 개선 작업이 할당 된 전자 메일을 받게 되며, 그런 다음 해당 대시보드에서 개선 작업을 열 수 있습니다.

할당 된 사용자는 구현 지침에 나와 있는 권장 작업을 수행할 수 있습니다.

## <a name="perform-work-and-store-documentation"></a>작업 수행 및 문서 저장

구현 작업을 수행 하는 경우 **메모 및 설명서** 섹션의 향상 작업에 파일 및 메모를 직접 업로드할 수 있습니다.  이 환경은 규정 준수 표준 및 규정을 충족 하기 위한 제어 만족도를 제공 하는 데 도움이 되는 안전 하 고 중앙화 된 리포지토리입니다. 읽기 전용 권한이 있는 사용자는이 섹션의 콘텐츠를 읽을 수 있습니다. 편집 권한이 있는 사용자만 파일을 업로드 및 다운로드 하 고 메모를 입력 하거나 편집할 수 있습니다.

**Notes 및 문서** 섹션의 필드에는 다음이 포함 됩니다.

**업로드 된 문서**

- **문서 관리** 를 선택 하 여 관련 파일을 업로드 합니다.
- 문서 플라이 아웃 관리 창이 열리면 **문서 추가**를 선택한 다음 시스템에서 파일을 선택 합니다. 허용 되는 파일 형식: 
  - 문서 (.doc, .xls, .ppt, .txt, .pdf)
  - 이미지 (.jpg, .png)
  - 비디오 (mkv)
  - 압축 된 파일 (.zip, rar)
- 창에서 파일이 확인 되 면 **닫기를** 선택 하 여 첨부 파일을 자동으로 저장 합니다. 업로드 된 문서 아래에 파일이 표시 됩니다 **.** 
- 문서를 다운로드 하거나 삭제 하려면 문서 목록 아래에서 **문서 관리** 를 선택 합니다. 플라이 아웃 창에서 문서 행을 클릭 하거나 탭 하 여 강조 표시 한 후 **다운로드** 또는 삭제를 선택 **합니다.**

**구현, 테스트 및 추가 참고 사항**

- 이러한 세 필드 중 하나에 메모를 추가 하려면 이러한 필드 아래에서 **구현 노트 편집** 을 선택 합니다.
- 플라이 아웃 창이 열리면 텍스트 필드에 메모를 입력 한 다음 **저장 후 닫기를**선택 합니다.
- 메모를 편집 하려면 **구현 메모 편집**을 선택 하 고 편집을 수행한 다음 **저장 후 닫기를**선택 합니다.

메모 필드에는 문자 제한이 없습니다. 개선 작업 세부 정보 페이지에서 쉽게 보고 편집할 수 있도록 노트를 간략하게 유지 하는 것이 좋습니다.

## <a name="change-improvement-action-status"></a>개선 작업 상태 변경

각 향상 작업에 대해 구현 상태 및 날짜와 테스트 상태 및 날짜를 기록할 수 있습니다. 사용 가능한 필드 및 상태 옵션은 다음과 같습니다.

- **구현 상태**: 다음 상태 옵션 중에서 선택 합니다.
    - **구현 되지 않음** -작업이 아직 구현 되지 않았습니다.
    - **구현** 됨-작업 구현 됨
    - **대체 구현** -타사 도구를 사용 하거나 Microsoft 권장 사항에 포함 되지 않은 다른 작업을 수행한 경우에이 옵션을 선택 합니다.
    - **계획** 됨-구현을 위한 조치가 계획 됩니다.
    - **범위에 없음** -조직에 해당 하지 않는 작업에 대 한 옵션입니다. 이 상태를 선택 하면 작업의 점수 매기기에서 제외 됩니다. 작업을 점수 매기기에 포함 하 여 선택을 취소 합니다.
- **구현 날짜**: 구현 상태가 **구현** 됨 또는 **대체 구현**으로 설정 된 경우 사용할 수 있는 필드입니다. 달력 팝업을 전환 하 여 날짜 선택
- **테스트 상태**: 다음 옵션 중에서 선택 합니다.
    - **평가 안** 됨-테스트가 시작 되지 않음
    - **성공**-구현이 테스트 됨
    - **실패 한 낮은 위험** -테스트 실패, 낮은 위험
    - **실패 한 중간 위험** -테스트 실패, 보통 위험
    - **실패 한 높은 위험** -테스트 실패, 높은 위험
- **테스트 날짜**: 달력 팝업을 전환 하 여 날짜를 선택 합니다.

> [!NOTE]
> 구현 및 테스트 상태 필드는 사용자 **에게만 할당** 되는 것이 아니라 편집 권한이 있는 모든 사용자가 편집할 수 있습니다.

## <a name="assign-improvement-action-to-assessor-for-completion"></a>평가자에 개선 작업을 할당 하 여 완료

작업을 완료 하 고 증거를 업로드 한 후에는 구현 상태와 날짜를 설정 하 고, 유효성 검사를 위해 평가자에 게 개선 작업을 할당 합니다.

평가자의 형식은 다음과 같습니다.

- 조직 내에서 컨트롤의 유효성 검사를 수행 하는 내부 평가자
- Microsoft 클라우드 서비스를 감사 하는 타사 독립적인 조직과 같이 준수를 조사, 확인 및 인증 하는 외부 평가자

평가자는 작업의 유효성을 검사 하 고 설명서를 검토 하 고 적절 한 테스트 상태를 선택 합니다.

**테스트 상태가 "통과" 인 경우**작업이 완료 되 고 예상 되는 **점수** 에 전체 준수 점수에 따라 달성 되 고 계산 되는 최대 점수를 표시 합니다.

**테스트 상태가 "실패" 인 경우**: 동작이 요구 사항을 충족 하지 않으며 평가자가 추가 작업을 위해 해당 사용자에 게 다시 할당할 수 있습니다.

## <a name="solutions-page"></a>솔루션 페이지

향상 작업을 통해 점수를 높일 수 있는 **솔루션 페이지** 는 다른 시작 지점입니다. 

솔루션 페이지로 이동 하려면 대시보드에서 **솔루션** 탭을 선택 하거나 대시보드의 오른쪽 위 섹션에서 **점수에 영향을 주는 솔루션** 아래의 **모든 솔루션 보기** 를 선택 합니다.

솔루션 페이지에는 솔루션 별로 구성 된 획득 및 잠재 점수 공유가 표시 됩니다. 이 보기에서 남은 점과 향상 작업을 보면 즉각적인 주의가 필요한 솔루션을 파악할 수 있습니다.

### <a name="filtering-your-solutions-view"></a>솔루션 보기 필터링

솔루션 보기를 필터링 하려면 다음을 수행 합니다.

1. 평가 목록의 왼쪽 위 모서리에서 **필터** 를 선택 합니다.
2. 플라이 아웃 **필터** 창에서 원하는 조건 (표준 및 규정, 솔루션, 작업 유형, 준수 관리자 그룹, 범주) 옆에 확인을 배치 합니다.
3. **적용** 단추를 선택 합니다. 필터 창이 닫히고 필터링 된 보기가 표시 됩니다.

평가 목록 위의 **그룹** 드롭다운 메뉴에서 그룹화 유형을 선택 하 여 그룹, 제품 또는 규정 별로 평가를 확인 하도록 보기를 수정할 수도 있습니다.

### <a name="taking-actions-from-the-solutions-page"></a>솔루션 페이지에서 작업 수행

솔루션 페이지에는 개선 작업에 연결 된 조직의 솔루션이 표시 됩니다. 이 표에서는 각 솔루션의 전체 점수, 해당 솔루션 내에서 달성 하 고 가능한 점수 및 해당 솔루션에 그룹화 되어 있으며 점수를 높일 수 있는 남은 개선 작업 수를 보여 줍니다.

다음 두 가지 방법으로이 화면에서 작업을 수행할 수 있습니다.

1. 원하는 솔루션의 행에서 **남은 작업** 열 아래에서 하이퍼링크 된 번호를 선택 합니다. 해당 솔루션에 대 한 테스트 되지 않은 개선 작업을 보여 주는 개선 작업 화면의 필터링 된 보기가 표시 됩니다.

2. 원하는 솔루션의 행에서 **솔루션 열기** 열 아래에 있는 **열기**를 선택 합니다. Microsoft 365 및 Office 365 보안 및 준수 센터에서 권장 조치를 취할 수 있는 솔루션 또는 위치가 표시 됩니다.

## <a name="assessments-page"></a>평가 페이지

평가 페이지에는 조직에 대해 추적 하기 위해 선택한 평가가 나열 됩니다. 준수 점수 분모는 추적 된 모든 평가에 따라 결정 됩니다. 더 많은 평가가 추가 되 고, 개선 작업 페이지에 더 많은 향상 작업이 발생 하며, 점수 분모가 더 높습니다.

평가 페이지에 액세스 하려면 대시보드에서 **평가** 탭을 선택 합니다.

이 페이지에서는 각 평가에 대 한 중요 한 정보를 빠르게 확인할 수 있습니다.

- **상태**: 평가의 모든 개선 작업을 완료 하기 위한 상태는 다음 중 하나로 표시 됩니다.
    - **비호환**: 평가의 개선 작업이 구현 되지 않았으며 올바르게 테스트 되지 않았습니다. 작업이 아직 시작 되지 않았습니다.
    - **진행**중: 개선 작업을 구현 하거나 테스트 하는 중에 작업이 진행 되 고 있습니다. 예를 들어, 평가의 향상 된 작업은 작업에 할당 되었으며 구현 및 테스트 중인 프로세스입니다.
- **평가 진행률**: 성공적으로 테스트를 거친 컨트롤 수에 따라 측정 된 평가의 최종 완료를 향해 수행 된 작업의 비율입니다.
- **고객 관리 작업**: 고객 관리 컨트롤 구현을 충족 하기 위해 완료 된 작업의 수입니다.
- **Microsoft 관리 작업**: microsoft 관리 컨트롤의 구현을 충족 하기 위해 완료 된 작업의 수입니다.
- **평가 그룹**: 만든 그룹의 이름으로, 평가가 상주 합니다.
- **관련 서비스**: 연결 된 Microsoft 365 서비스
- **관련 규정**: 평가에서 충족 해야 하는 규정 표준, 정책 또는 법률입니다.

### <a name="default-assessments"></a>기본 평가

기본적으로 평가 페이지에는 Microsoft 365 데이터 보호 기준 평가가 표시 됩니다. 규정 준수 점수가 몇 가지 사전 구성 평가도 제공 합니다 ([전체 목록 보기](compliance-score.md#templates)). 준수 관리자에서 추가 규정 및 표준을 다루는 평가를 더 추가할 수 있습니다.

### <a name="managing-assessments"></a>평가 관리

공개 미리 보기 중에는 규정 준수 관리자 도구로 이동 하 여 평가를 사용자 지정 하 고 관리 합니다.

규정 준수 점수의 **평가** 페이지에서 평가 목록 맨 위의 **준수 관리자에서 평가 관리** 를 선택 합니다. 이 링크를 사용 하 여 준수 관리자의 **평가** 대시보드로 이동할 수 있습니다.

평가 목록 맨 위에 있는 다른 링크로, **준수 관리자의 microsoft 작업**에서는 준수 점수에 기여 하는 microsoft 컨트롤을 표시 하는 준수 관리자의 **컨트롤 정보** 대시보드로 이동 합니다.

### <a name="filtering-your-assessments-view"></a>평가 보기 필터링

평가 보기를 필터링 하려면:

1. 평가 목록의 왼쪽 위 모서리에서 **필터** 를 선택 합니다.
2. 플라이 아웃 **필터** 창에서 원하는 조건 (표준 및 규정, 준수 관리자 그룹) 옆에 확인을 배치 합니다.
3. **적용** 단추를 선택 합니다. 필터 창이 닫히고 필터링 된 보기가 표시 됩니다.

평가 목록 위의 **그룹** 드롭다운 메뉴에서 그룹화 유형을 선택 하 여 그룹, 제품 또는 규정 별로 평가를 확인 하도록 보기를 수정할 수도 있습니다.

### <a name="managing-improvement-actions-within-an-assessment"></a>평가 내의 개선 작업 관리

평가 목록의 **고객 관리 작업** 열에서 원하는 평가의 행에 있는 연결 된 텍스트를 선택 합니다. 이 작업을 수행 하면 해당 평가 내의 작업을 보여 주는 향상 된 기능 페이지의 필터링 된 보기가 표시 됩니다.

## <a name="reporting"></a>Reporting

모든 개선 작업의 보고서를 준수 점수에 내보낼 수 있습니다. **개선 작업** 페이지에서 화면 왼쪽 위 모서리에 있는 작업 목록 위에 **Export** 를 선택 합니다. 그러면 **향상 작업 페이지에** 표시 된 모든 개선 작업과 필터 범주가 포함 된 Excel 워크시트가 생성 됩니다.

다음 단계를 수행 하 여 준수 관리자의 보고서를 내보낼 수도 있습니다.

1. 준수 관리자에서 **컨트롤 정보** 대시보드로 이동 합니다.
2. **평가** 탭과 **서식 파일** 탭이 표시 됩니다.  
3. 평가를 내보내려면 **평가** 탭을 선택 합니다. **그룹** 및 **평가** 드롭다운 메뉴를 사용 하 여 내보내려는 평가를 선택 합니다. 화면 오른쪽 위 부근에서 **내보내기를** 선택 합니다. Excel 파일이 다운로드 됩니다. 여기에는 컨트롤 별로 그룹화 된 작업 목록과 구현 및 테스트 세부 정보가 포함 되어 있습니다.
4. 서식 파일을 **내보내려면 서식 파일 탭을** 선택 하 고 **서식 파일** 드롭다운 메뉴에서 내보내려는 서식 파일을 선택 합니다. 화면 오른쪽 위 부근에서 **내보내기를** 선택 합니다. Excel 파일이 다운로드 됩니다. 여기에는 컨트롤 별로 그룹화 된 작업 목록과 구현 및 테스트 세부 정보가 포함 되어 있습니다.
