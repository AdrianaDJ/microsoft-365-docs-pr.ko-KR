# <a name="get-started-with-insider-risk-management-settings"></a>참가자 위험 관리 설정 시작

참가자 위험 관리 설정은 정책을 만들 때 선택한 서식 파일에 관계 없이 모든 참가자 위험 관리 정책에 적용 됩니다. 설정은 모든 참가자 위험 관리 탭의 맨 위에 있는 **참가자 위험 설정** 제어를 사용 하 여 구성 됩니다. 이러한 설정은 다음 영역에 대 한 정책 구성 요소를 제어 합니다.

- 개인 정보
- 슬라이더
- 정책 시간 표시 막대
- 지능적 감지
- 알림 내보내기 (미리 보기)
- 우선 순위 사용자 그룹 (미리 보기)
- 우선 순위 물리적 자산 (미리 보기)
- 전원 자동화 흐름 (미리 보기)
- Microsoft 팀 (미리 보기)

시작 하 고 참가자 위험 관리 정책을 만들기 전에 이러한 설정을 이해 하 고 조직의 준수 요구에 가장 적합 한 수준 설정을 선택 해야 합니다.

## <a name="privacy"></a>개인 정보

정책이 일치 하는 사용자의 개인 정보를 보호 하는 것은 중요 하며, objectivity의 데이터 조사 및 분석 검토에서 의견을 승격 하는 데 도움을 받을 수 있습니다. 참가자 위험 정책이 일치 하는 사용자에 대해 다음 설정 중 하나를 선택할 수 있습니다.

- **익명 버전의**사용자 이름 표시: 관리자, 데이터 investigators 및 검토자가 정책 알림과 연결 된 사용자를 볼 수 없도록 하는 익명입니다. 예를 들어 사용자가 ' AnonIS8-988 '와 같은 임의의 pseudonym "유예 비트 관리자 '를 사용 하 여 참가자 위험 관리 환경의 모든 영역에 표시 됩니다. 이 설정을 선택 하면 현재 및 이전의 정책이 일치 하는 모든 사용자가 anonymizes 모든 정책에 적용 됩니다. 이 옵션을 선택 하면 참가자 위험 경고 및 사례 세부 정보에 사용자 프로필 정보가 제공 되지 않습니다. 그러나 새 사용자를 기존 정책에 추가할 때 또는 새 정책에 사용자를 할당할 때 표시 됩니다. 이 설정을 해제 하면 현재 또는 이전의 정책 일치 하는 모든 사용자에 대해 사용자 이름이 표시 됩니다.
- **익명 버전의 사용자 이름 표시 안 함**: 경고 및 사례에 대 한 모든 현재 및 이전의 정책 일치에 대해 사용자 이름이 표시 됩니다. 모든 참가자 위험 관리 경고 및 사례에 대해 사용자에 대해 사용자 프로필 정보 (이름, 제목, 별칭, 조직 또는 부서)가 표시 됩니다.

## <a name="indicators"></a>슬라이더

참가자 위험 정책 템플릿은 검색 및 조사할 위험 활동 유형을 정의 합니다. 각 정책 템플릿은 특정 트리거 및 위험 활동에 해당 하는 특정 지표를 기반으로 합니다. 기본적으로 모든 지표는 사용 하지 않도록 설정 되며, 참가자 위험 관리 정책을 구성 하기 전에 하나 이상의 정책 지표를 선택 해야 합니다.

경고는 사용자가 필요한 임계값을 충족 하는 정책 표시기와 관련 된 작업을 수행할 때 정책에 의해 트리거됩니다. 참가자 위험 관리에는 다음과 같은 두 가지 유형의 지표가 사용 됩니다.

- **이벤트 트리거**: 사용자가 참가자 위험 관리 정책에 대 한 활성 상태 인지 여부를 확인 하는 이벤트입니다. 사용자가 참가자에 게 추가 된 경우 위험 관리 정책에 트리거 이벤트가 없으면 사용자 작업이 정책에 의해 평가 되지 않습니다. 예를 들어 사용자 A는 departing users 정책 서식 파일을 사용 하 *여 데이터 절도* 에서 만든 정책에 추가 되 고 정책 및 MICROSOFT 365 HR 커넥터가 올바르게 구성 됩니다. 사용자 a에 게 HR 커넥터에서 종료 날짜를 보고 하기 전에는이 참가자의 위험 관리 정책으로 인해 활동이 평가 되지 않습니다. 트리거하는 이벤트의 또 다른 예로는 *데이터 누수* 정책을 사용 하는 경우 사용자에 게 심각도가 *높은* DLP 정책 경고가 표시 되는 경우를 들 수 있습니다.
- **정책 지표**: 내 사용자에 대 한 위험 점수를 결정 하는 데 사용 되는 참가자 위험 관리 정책에 포함 된 지표입니다. 이러한 정책 지표는 사용자에 대해 트리거하는 이벤트가 발생 한 후에만 활성화 됩니다. 사용자가 개인 클라우드 저장소 서비스 또는 휴대용 저장 장치에 데이터를 복사 하거나, 사용자가 허용 되지 않은 외부 사용자와 내부 파일 및 폴더를 공유 하는 경우 정책 표시기의 몇 가지 예는 다음과 같습니다.

정책 표시기는 다음과 같은 영역으로 분할 됩니다. 참가자 위험 정책을 만들 때 각 표시기 수준에 대해 표시기 이벤트 제한을 활성화 하 고 사용자 지정 하는 지표를 선택할 수 있습니다.

- **Office 지표**: 여기에는 SharePoint 사이트, 팀 및 전자 메일 메시징의 정책 표시기가 포함 됩니다.
- **장치 지표**: 네트워크 또는 장치와 파일을 공유 하는 등의 작업에 대 한 정책 표시기가 여기에 포함 됩니다. Microsoft Office 파일과 관련 된 활동을 포함 하는 표시기가 있습니다. CSV 파일 및. PDF 파일 **장치 표시기**를 선택 하면 Windows 10 빌드 1809 이상인 장치에 대해서만 활동이 처리 됩니다. 참가자 위험과의 통합을 위해 장치를 구성 하는 방법에 대 한 자세한 내용은 [장치 지표 및 온보드 장치 사용](insider-risk-management-settings.md#OnboardDevices) 섹션을 참조 하십시오.
- **보안 정책 위반 지표**: 여기에는 승인 되지 않거나 악성 소프트웨어 설치와 관련 된 MICROSOFT Defender ATP에서 제공 되거나 보안 제어를 바이패스 하는 표시기가 포함 됩니다. 참가자 위험 관리에서 알림을 받으려면 활성 Microsoft Defender ATP 라이선스 및 참가자 위험 통합을 사용 하도록 설정 해야 합니다. 참가자 위험 관리 통합을 위해 Microsoft Defender ATP를 구성 하는 방법에 대 한 자세한 내용은 [Microsoft DEFENDER atp에서 고급 기능 구성을](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features\#share-endpoint-alerts-with-microsoft-compliance-center)참조 하세요.
- **위험 점수 boosters**: 비정상적인 활동 또는 이전의 정책 위반에 대 한 위험 점수를 발생 시키는 것을 포함 합니다. 위험 점수를 사용 하도록 설정 하면 boosters 위험 성과와 이러한 유형의 활동에 대 한 알림 가능성을 높일 수 있습니다. 위험 점수 boosters는 위에서 하나 이상의 지표를 선택 하는 경우에만 선택할 수 있습니다.

![참가자 위험 관리 지표 설정](../media/insider-risk-settings-indicators.png)

경우에 따라 조직의 참가자 위험 정책에 적용 되는 참가자 위험 정책 지표를 제한할 수 있습니다. 모든 참가자 위험 정책에서 사용 하지 않도록 설정 하 여 특정 영역에 대 한 정책 지표를 해제할 수 있습니다. 참가자 위험 정책 서식 파일에 대해 트리거 이벤트를 수정할 수 없습니다.

모든 참가자 위험 정책에서 사용 하도록 설정 된 참가자 위험 정책 표시기를 정의 하려면 **참가자 위험 설정**  >  **지표로** 이동 하 여 하나 이상의 정책 지표를 선택 합니다. 정책 마법사에서 참가자 위험 정책을 만들거나 편집할 때 지표 설정 페이지에서 선택한 지표를 개별적으로 구성할 수는 없습니다.

>[!NOTE]
>수동으로 추가 된 새 사용자가 **사용자 대시보드에**표시 되는 데 몇 시간이 걸릴 수 있습니다. 이러한 사용자에 대 한 이전 90 일 동안의 활동은 표시 하는 데 최대 24 시간이 걸릴 수 있습니다. 수동으로 추가 된 사용자에 대 한 활동을 보려면 **사용자 대시보드에서** 사용자를 선택 하 고 세부 정보 창에서 **사용자 활동** 탭을 엽니다.

### <a name="enable-device-indicators-and-onboard-devices"></a>장치 지표 및 온보드 장치 사용
<a name="OnboardDevices"> </a>

장치에서 위험 활동을 모니터링 하 고 이러한 활동에 대 한 정책 표시기를 포함 하려면 장치가 다음 요구 사항을 충족 해야 하며, 다음 온 보 딩 단계를 완료 해야 합니다.

#### <a name="step-1-prepare-your-endpoints"></a>1 단계: 끝점 준비

참가자 위험 관리에서 보고를 계획 하는 Windows 10 장치가 이러한 요구 사항을 충족 하는지 확인 합니다.

1. Windows 10 x64 빌드 1809 이상 버전을 실행 해야 하며 [windows 10 업데이트 (OS 빌드 17763.1075 2020)](https://support.microsoft.com/help/4537818/windows-10-update-kb4537818) 가 설치 되어 있어야 합니다.
2. 모든 장치가 [AAD(Azure Active Directory)에 가입](https://docs.microsoft.com/azure/active-directory/devices/concept-azure-ad-join)되어 있거나 Hybrid Azure AD에 가입되어 있어야 합니다.
3. 클라우드 업로드 활동에 대 한 작업을 모니터링 하려면 끝점 장치에 Microsoft Chromium에 지 브라우저를 설치 합니다. [Chromium 기반 새 Microsoft Edge 다운로드하기](https://support.microsoft.com/help/4501095/download-the-new-microsoft-edge-based-on-chromium)를 참조하세요.

#### <a name="step-2-onboarding-devices"></a>2 단계: 온 보 딩 장치
<a name="OnboardStep2"> </a>

장치에 대 한 참가자 위험 관리 활동을 모니터링 하려면 장치 모니터링을 사용 하도록 설정 하 고 끝점을 등록 해야 합니다. 이러한 작업은 모두 Microsoft 365 규정 준수 포털에서 수행됩니다.

아직 등록 되지 않은 장치를 등록 하려는 경우 다음 단계에 설명 된 대로 적절 한 스크립트를 다운로드 하 고 배포 합니다.

이미 [엔드포인트용 Microsoft Defender](https://docs.microsoft.com/windows/security/threat-protection/)에 온보딩된 장치가 있으면 해당 장치가 관리되는 장치 목록에 표시됩니다. 다음 섹션에서 [3 단계: 장치에 Microsoft Defender for 등록가 있는 경우](insider-risk-management-settings.md#OnboardStep3) 를 수행 합니다.

이 배포 시나리오에서는 아직 등록 되지 않은 장치를 온보드 것 이며, Windows 10 장치에서의 참가자 위험 활동도 모니터링 하 고 싶을 것입니다.

1. [Microsoft 규정 준수 센터](https://compliance.microsoft.com)를 엽니다.
2. 규정 준수 센터 설정 페이지를 열고 **장치 온보딩**을 선택합니다.

   > [!NOTE]
   > 일반적으로 장치 온보딩이 활성화되는 데 60초 정도 소요되지만, Microsoft 지원에 연락하기 전에 30분까지 기다려보세요.

3. **장치 관리**를 선택하여 **장치** 목록을 엽니다. 목록은 장치가 온보딩될 때까지 비어 있습니다.
4. **온보딩**을 선택하여 온보딩 프로세스를 시작합니다.
5. **배포 방법** 목록에서 이러한 추가 장치에 배포할 방법을 선택한 다음 **패키지를 다운로드합니다**.
6. [Windows 10 컴퓨터용 온보딩 도구와 방법](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/configure-endpoints)의 해당 절차를 따르세요. 이 링크를 누르면 5단계에서 선택한 배포 패키지와 일치하는 엔드포인트용 Microsoft Defender에 액세스할 수 있는 랜딩 페이지로 이동합니다.
    - 그룹 정책을 사용하여 Windows 10 컴퓨터 온보딩하기
    - Microsoft Endpoint Configuration Manager를 사용하여 Windows 컴퓨터 온보딩하기
    - 모바일 장치 관리 도구를 사용하여 Windows 10 컴퓨터 온보딩하기
    - 로컬 스크립트를 사용하여 Windows 10 컴퓨터 온보딩하기
    - 비영구적 VDI(가상 데스크톱 인프라) 머신 온보딩하기

완료 되 고 끝점이 등록 되 면 장치 목록에 표시 되 고 끝점은 감사 작업 로그를 참가자 위험 관리에 대해 보고 하기 시작 합니다.

> [!NOTE]
> 이 환경은 라이선스 적용하에 있습니다. 필수 라이선스가 없으면 데이터가 표시되지 않거나 테이터에 액세스할 수 없습니다.

#### <a name="step-3-if-you-have-devices-onboarded-into-microsoft-defender-for-endpoint"></a>3 단계: 장치가 Microsoft Defender for Endpoint에 등록 있는 경우
<a name="OnboardStep3"> </a>

끝점에 대 한 Microsoft Defender가 이미 배포 되었으며 끝점 보고가 있으면 이러한 모든 끝점이 관리 되는 장치 목록에 표시 됩니다. [2 단계: 온 보 딩 devices](insider-risk-management-settings.md#OnboardStep2) 섹션을 사용 하 여 새 장치를 참가자 위험 관리로 계속 해 서 등록할 수 있습니다.

1. [Microsoft 규정 준수 센터](https://compliance.microsoft.com)를 엽니다.
2. 규정 준수 센터 설정 페이지를 열고 **장치 모니터링 사용**을 선택합니다.
3. **장치 관리**를 선택하여 **장치** 목록을 엽니다. 이미 엔드포인트용 Microsoft Defender에 보고하고 있는 장치 목록이 표시되어야 합니다.
4. 추가 디바이스를 온보딩해야 하는 경우에는 **온보딩**을 선택합니다.
5. **배포 방법** 목록에서 이러한 추가 장치에 배포할 방법을 선택한 다음 **패키지를 다운로드합니다**.
6. [Windows 10 컴퓨터용 온보딩 도구와 방법](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/configure-endpoints)의 해당 절차를 따르세요. 이 링크를 누르면 5단계에서 선택한 배포 패키지와 일치하는 엔드포인트용 Microsoft Defender에 액세스할 수 있는 랜딩 페이지로 이동합니다.
    - 그룹 정책을 사용하여 Windows 10 컴퓨터 온보딩하기
    - Microsoft Endpoint Configuration Manager를 사용하여 Windows 컴퓨터 온보딩하기
    - 모바일 장치 관리 도구를 사용하여 Windows 10 컴퓨터 온보딩하기
    - 로컬 스크립트를 사용하여 Windows 10 컴퓨터 온보딩하기
    - 비영구적 VDI(가상 데스크톱 인프라) 머신 온보딩하기

작업이 완료 되 고 끝점이 등록 **장치** 테이블 아래에 표시 되 고 끝점은 감사 작업 로그에서 위험 관리를 참가자에 게 보고 하기 시작 합니다.

> [!NOTE]
>이 환경은 라이선스 적용하에 있습니다. 필수 라이선스가 없으면 데이터가 표시되지 않거나 테이터에 액세스할 수 없습니다.

### <a name="indicator-level-settings-preview"></a>지표 수준 설정 (미리 보기)

정책 마법사에서 정책을 만들 때 일별 위험 이벤트가 참가자 위험 경고에 대 한 위험 점수에 미치는 영향을 구성할 수 있습니다. 이러한 지표 설정은 조직의 위험 요인 발생 횟수와 이러한 이벤트에 대해 관련 된 경고 심각도를 영향을 미치는 방법을 제어 하는 데 도움이 됩니다. 원하는 경우 사용 하도록 설정 된 모든 지표에 대해 Microsoft에서 권장 하는 기본 이벤트 임계값 수준을 유지 하도록 선택할 수도 있습니다.

예를 들어, 참가자 위험 정책 설정에서 SharePoint 지표를 사용 하도록 설정 하 고 새 참가자 위험 *데이터 누수* 정책에 대 한 표시기를 구성할 때 sharepoint 이벤트에 대 한 사용자 지정 임계값을 설정 하기로 결정 합니다. 참가자 위험 정책 마법사에서 각 SharePoint 표시기에 대해 세 가지 다른 일별 이벤트 수준을 구성 하 여 이러한 이벤트와 관련 된 경고에 대 한 위험 점수에 영향을 줍니다.

![참가자 위험 관리 사용자 지정 지표 설정](../media/insider-risk-custom-indicators.png)

첫 번째 일별 이벤트 수준에서는 이벤트에 대 한 위험 점수에 대 한 영향을 낮추고 하루 중에 이벤트에 대 한 위험 성과에 영향을 주는 *20* 개 이상의 이벤트가 발생 하는 것을 최소화 하기 위해 하루 중 하나 이상의 *이벤트* 에 임계값을 더 작게 설정 *하 고 이벤트* 에 대 한 위험 점수에 높은 영향을 줄 수 있습니다. 이러한 설정은 다음을 의미 합니다.

- 이벤트를 트리거한 후 발생 하는 SharePoint 이벤트가 1-9 위험 점수가 최소한 영향을 받게 되며 경고를 생성 하지 않는 경향이 있습니다.
- 트리거 이벤트 후 발생 하는 SharePoint 이벤트가 10-19에 해당 하는 경우 위험 점수가 본질적으로 낮고 경고 심각도 수준이 낮은 수준으로 유지 되는 경향이 있습니다.
- 트리거 후 발생 하는 SharePoint 이벤트가 20-29에 해당 하는 경우 위험 점수가 본질적으로 더 높고 경고 심각도 수준이 보통 수준으로 유지 되는 경향이 있습니다.
- 트리거 후에 발생 하는 SharePoint 이벤트가 30 개 이상 있는 경우 위험 점수가 기본적으로 더 높고 경고 심각도 수준이 높은 수준으로 유지 되는 경향이 있습니다.

## <a name="policy-timeframes"></a>정책 기한

정책 기간을 사용 하면 참가자 위험 관리 정책 템플릿에 대 한 이벤트 및 작업을 기반으로 하는 정책 일치 이후에 트리거되는 이전의 및 이후 검토 기간을 정의할 수 있습니다. 선택한 정책 템플릿에 따라 다음과 같은 정책 기한을 사용할 수 있습니다.

- **정품 인증 창**: 모든 정책 템플릿에서 사용할 수 있는 *정품 인증 창은* 트리거 이벤트가 발생 **한 후** 창에서 활성화 되는 지정 된 기간 (일)입니다. 이 창은 정책에 할당 된 모든 사용자에 대해 트리거 이벤트가 발생 한 후 1 ~ 30 일까 지 활성화 됩니다. 예를 들어, 참가자 위험 관리 정책을 구성 하 고 *정품 인증 기간* 을 30 일로 설정 했습니다. 정책을 구성한 후 몇 개월이 경과 했으며 정책에 포함 된 사용자 중 하나에 대해 트리거 이벤트가 발생 합니다. 트리거 이벤트가 *활성화 창을* 활성화 하 고 해당 사용자에 대해 정책이 트리거 이벤트가 발생 한 후 30 일 동안 활성 상태가 됩니다.
- **이전 활동 검색**: 모든 정책 서식 파일에 대해 사용 가능한 *이전 활동 검색* 은 이벤트가 트리거되는 이벤트 **보다 먼저** 활성화 되는 지정 된 기간 (일)입니다. 정책에 할당 된 모든 사용자에 대해 트리거 이벤트가 발생 하기에는 0 ~ 180 일 동안 창이 활성화 됩니다. 예를 들어, 참가자 위험 관리 정책을 구성 하 고 *이전 활동 검색* 을 90 days로 설정 했습니다. 정책을 구성한 후 몇 개월이 경과 했으며 정책에 포함 된 사용자 중 하나에 대해 트리거 이벤트가 발생 합니다. 트리거하는 이벤트는 *이전 활동 검색* 을 활성화 하 고 정책에서 해당 사용자에 대 한 기록 활동을 트리거하는 이벤트 보다 90 일 전에 수집 합니다.

![참가자 위험 관리 기간 설정](../media/insider-risk-settings-timeframes.png)

## <a name="intelligent-detections"></a>지능적 감지

지능형 검색 설정은 경고에 대해 위험한 활동의 검색이 처리 되는 방식을 조정 하는 데 도움이 됩니다. 경우에 따라 무시할 파일 형식을 정의 해야 하거나, 파일에 대 한 검색 수준을 적용 하 여 경고에 대 한 최소 도구 모음을 정의할 수 있습니다. 공격적인 언어 정책을 사용 하는 경우에는 보고 된 정책 일치 항목의 양을 제어 하기 위해 검색 민감도를 늘리거나 줄여야 할 수 있습니다. 이러한 설정을 사용 하 여 전체 경고 볼륨, 파일 형식 제외, 파일 볼륨 제한 및 공격적인 언어 검색 민감도를 제어할 수 있습니다.

![참가자 위험 관리 지능형 검색 설정](../media/insider-risk-settings-detections.png)

### <a name="anomaly-detections"></a>변칙 감지

비정상적인 검색에는 파일 형식 제외 및 파일 볼륨 제한에 대 한 설정이 포함 됩니다.

- **파일 형식 제외**: 모든 참가자 위험 관리 정책 일치에서 특정 파일 형식을 제외 하려면 파일 형식 확장명을 쉼표로 구분 하 여 입력 합니다. 예를 들어 특정 유형의 음악 파일을 정책 일치에서 제외 하려면 **파일 형식 제외** 필드에 *aac, mp3, wav, wma* 를 입력할 수 있습니다. 이러한 확장명을 가진 파일은 모든 참가자 위험 관리 정책에 의해 무시 됩니다.
- **파일 볼륨 컷오프 제한**: 활동 경고가 참가자 위험 정책에 보고 되기 전에 최소 파일 수준을 정의 하려면 파일 수를 입력 합니다. 예를 들어 사용자가 10 개 이하의 파일을 다운로드 하는 경우, 즉 정책에서이 작업을 변칙으로 간주 하더라도 참가자 위험 알림을 생성 하지 않으려면 ' 10 '을 입력 합니다.

### <a name="offensive-language-detections"></a>공격적인 언어 감지

>[!IMPORTANT]
>2020 년 10 월 16 일부터이 템플릿을 사용 하 여 더 이상 정책을 만들 수 없습니다. 이 서식 파일을 사용 하는 모든 활성 정책은 1 월 2021 일에 영구적으로 제거 될 때까지 작동 합니다. 이 서식 파일을 지 원하는 방식의 기본 제공 분류자에는 많은 수의 가양성이 생성 되었기 때문입니다. 공격적인 언어에 대 한 위험 문제를 해결 하려면 Microsoft 365 [통신 준수](communication-compliance.md) 정책을 사용 하는 것이 좋습니다. 기본 제공 분류자에 대 한 자세한 내용은 [trainable 분류자 시작](classifier-get-started-with.md)을 참조 하십시오.

*전자 메일 템플릿에서 공격적인 언어* 를 사용 하 여 정책에 대 한 비속어의 민감도를 조정 하려면 다음 설정 중 하나를 선택 합니다.

- **Low**: 공격 하지 않는 언어 및 sentiment 검색을 위한 가장 넓은 범위의 높은 민감도 수준입니다. 공격적인 잘못 된 언어 일치에 대 한 가양성의 확률은 상승 된 것입니다.
- **Medium**: 공격적인 언어 및 sentiment를 감지 하기 위한 균형 범위가 있는 중간 수준의 민감도입니다. 공격적인 잘못 된 언어 일치에 대 한 가양성의 확률은 평균입니다.
- **High**: 더 공격적인 언어 및 sentiment 검색을 위한 좁은 범위의 높은 민감도 수준입니다. 공격적인 잘못 된 언어 일치에 대 한 가양성의 확률은 낮음입니다.

### <a name="alert-volume"></a>경고 볼륨

참가자 위험 정책에 따라 검색 되는 사용자 활동에는 특정 위험 점수가 할당 되며이 점수에는 경고 심각도 (낮음, 중간, 높음)가 지정 됩니다. 기본적으로는 낮음, 중간 및 높은 심각도 경고가 표시 되지만 필요에 따라 볼륨을 늘리거나 낮출 수 있습니다. 모든 참가자 위험 관리 정책에 대 한 경고의 양을 조정 하려면 다음 설정 중 하나를 선택 합니다.

- **알림 수가 적을수록**심각도가 낮은 경고가 표시 되는 중요도가 적고 심각도가 낮을 수 있는 경고는 낮습니다. 이 설정 수준은 일부 실제 긍정을 놓칠 수 있음을 의미 합니다.
- **기본 볼륨**: 심각도가 높은 경고가 모두 표시 되 고 중요도가 중간 수준 및 낮은 심각도 경고를 볼 수 있습니다.
- **추가 경고**: 중간 심각도 및 중요도가 높은 경고 및 심각도가 가장 낮은 경고가 표시 됩니다. 이 설정 수준으로 인해 가양성이 더 많이 발생할 수 있습니다.

### <a name="microsoft-defender-advanced-threat-protection-preview"></a>Microsoft Defender Advanced Threat Protection (미리 보기)

[Microsoft Defender (Advanced Threat Protection](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection) )는 엔터프라이즈 네트워크가 고급 위협에 대해 예방, 감지, 조사 및 대응 하도록 설계 된 엔터프라이즈 끝점 보안 플랫폼입니다. 조직의 보안 위반을 보다 명확 하 게 표시 하려면 참가자 위험 관리 보안 위반 정책 템플릿에서 만든 정책에 사용 된 작업에 대 한 Microsoft Defender ATP 경고를 가져오고 필터링 하면 됩니다.

관심 있는 신호의 유형에 따라 Microsoft Defender ATP 경고 문제 분류 상태에 따라 위험 관리에 대해 알림을 가져오도록 선택할 수 있습니다. 전역 설정에서 다음과 같은 알림 심사 상태를 하나 이상 정의 하 여 가져올 수 있습니다.

- 알 수 없음
- 신규
- 진행 중
- 해결됨

Microsoft Defender ATP의 알림은 매일 가져옵니다. 선택한 심사 상태에 따라 Microsoft Defender ATP의 심사 상태 변경과 같은 경고에 대해 여러 사용자 활동이 표시 될 수 있습니다.

예를 들어이 설정에 대해 *새로 만들기*, *진행 중*및 *해결 됨* 을 선택 하 고, Microsoft Defender ATP 알림이 생성 되 고 상태가 *신규*이면 사용자에 대 한 초기 경고 활동을 참가자 위험으로 가져옵니다. Microsoft Defender ATP에서 상태가 *진행 중*으로 변경 되 면 사용자에 대해이 경고에 대 한 두 번째 활동을 insider 위험성이 가져오게 됩니다. *확인* 된 최종 MICROSOFT Defender ATP 심사 상태가 설정 된 경우이 경고에 대 한 세 번째 활동을 사용자의 참가자 위험으로 가져옵니다. 이 기능을 사용 하면 investigators에서 Microsoft Defender ATP 알림의 진행을 따르고 조사에 필요한 표시 수준을 선택할 수 있습니다.

>[!IMPORTANT]
>보안 위반 알림을 가져오려면 해당 조직에 Microsoft Defender ATP를 구성 하 고 Defender 보안 센터의 참가자 위험 관리 통합을 위해 Microsoft Defender atp를 사용 하도록 설정 해야 합니다. 참가자 위험 관리 통합을 위해 Microsoft Defender ATP를 구성 하는 방법에 대 한 자세한 내용은 [Microsoft DEFENDER atp에서 고급 기능 구성을](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features\#share-endpoint-alerts-with-microsoft-compliance-center)참조 하세요.

### <a name="domains-preview"></a>도메인 (미리 보기)

도메인 설정은 특정 도메인에 대 한 통신의 위험 수준을 정의 하는 데 도움이 됩니다. 이러한 통신에는 파일 공유, 전자 메일 메시지 또는 콘텐츠 다운로드가 포함 됩니다. 이러한 설정에서 도메인을 지정 하면 이러한 도메인을 사용 하 여 수행 되는 작업에 대 한 위험 점수 매기기를 향상 시키거나 낮출 수 있습니다. 예를 들어 contoso.com 및 sales.wingtiptoys.com을 허용 도메인으로 지정 하려면 **허용 도메인** 필드에 ' contoso.com sales.wingtiptoys.com '를 입력 합니다.

다음의 각 도메인 설정에 대해 최대 500 개의 도메인을 입력할 수 있습니다.

- **허용 되지 않은 도메인:** 허용 되지 않은 도메인을 지정 하면 이러한 도메인을 사용 하 여 수행 되는 작업에 *더 높은* 위험 점수가 발생 합니다.
- **허용 된 도메인:** 설정에서 허용 도메인을 지정 하면 이러한 도메인을 사용 하 여 수행 되는 활동은 위험 점수가 *더 낮으므로* 내부 조직 활동이 처리 되는 방식과 비슷하게 처리 됩니다. 예를 들어 이러한 도메인에 대 한 전자 메일 활동은 내부 전자 메일 활동이 분석 되는 방식과 비슷하게 분석 됩니다.
- 타사 **도메인:** 타사 도메인은 조직에서 비즈니스 목적으로 사용 되는 도메인 이며 중요 한 콘텐츠가 이러한 위치에 저장 될 수 있습니다. 타사 도메인을 지정 하면 이러한 도메인에서 위험한 활동에 대 한 알림을 받을 수 있습니다.

## <a name="priority-user-groups-preview"></a>우선 순위 사용자 그룹 (미리 보기)

조직의 사용자는 자신의 위치, 중요 한 정보에 대 한 액세스 수준 또는 위험 기록에 따라 다양 한 수준의 위험을 가질 수 있습니다. 이러한 사용자의 활동을 조사 및 평가 하는 우선 순위를 지정 하면 조직에 대 한 결과를 높일 수 있는 잠재적 위험을 알리는 데 도움이 될 수 있습니다. 우선 순위 사용자 그룹의 참가자 위험 관리에서는 보다 근접 한 검사 및 중요 한 위험 점수 매기기가 필요한 조직의 사용자를 정의할 수 있습니다. 우선 순위 사용자에의 한 *보안 정책 위반과* 함께 priority 사용자에의 한 *데이터 누출* 정책 서식 파일을 사용 하 여 우선 순위 사용자 그룹에 추가 된 사용자는 더 높은 심각도 수준에 따라 참가자 위험 경고 및 경고의 가능성을 높일 수 있습니다.

![참가자 위험 관리 우선 순위 사용자 그룹 설정](../media/insider-risk-settings-priority-users.png)

예를 들어 사용자가 중요 한 정보에 액세스할 수 있는 고도로 기밀 프로젝트에 대 한 데이터 누출을 보호 해야 합니다. 조직의 사용자에 대해이 프로젝트에서 작업 하는 *기밀 프로젝트* *사용자* 우선 순위 사용자 그룹을 만들도록 선택 합니다. 정책 마법사 및 *우선 순위 사용자* 에의 한 데이터 누수 정책 서식 파일을 사용 하 여 새 정책을 만들고 *기밀 프로젝트 사용자* 우선 순위 사용자 그룹을 정책에 할당 합니다. *기밀 프로젝트 사용자* 우선 순위 사용자 그룹의 구성원에 대 한 정책에서 검사 하는 작업은 위험 및 이러한 사용자의 작업에 대 한 중요 한 영향을 받게 되므로 경고를 생성 하 고 심각도 수준이 높은 알림을 받을 수 있습니다.

### <a name="create-a-priority-user-group"></a>우선 순위 사용자 그룹 만들기

새 우선 순위 사용자 그룹을 만들려면 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션에서 설정 컨트롤을 사용 합니다. 우선 순위 사용자 그룹을 만들려면 *참가자 위험 관리* 또는 *참가자 위험 관리 관리자* 역할 그룹의 구성원 이어야 합니다.

우선 순위 사용자 그룹을 만들려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정을**선택 합니다.
2. **우선 순위 사용자 그룹** 탭 선택
3. **우선 순위 사용자 그룹** 탭에서 **우선 순위 사용자 그룹 만들기** 를 선택 하 여 그룹 만들기 마법사를 시작 합니다.
4. **그룹 정의** 페이지에서 다음 필드에 정보를 입력 합니다.
    - **이름 (필수)**: 우선 순위 사용자 그룹의 이름을 입력 합니다. 마법사를 완료 한 후에는 priority 사용자 그룹의 이름을 변경할 수 없습니다.
    - **설명 (선택 사항)**: 우선 순위 사용자 그룹에 대 한 설명을 입력 합니다.
5. 계속 하려면 **다음** 을 선택 합니다.
6. **구성원 선택** 페이지에서 검색할 **구성원 선택을** 선택 하 고 그룹에 포함 된 메일 사용 가능 사용자 계정을 선택 하 고 **모두 선택** 확인란을 선택 하 여 조직의 모든 사용자를 그룹에 추가 합니다. 계속 하려면 **추가** 를 선택 하 고, 그룹에 사용자를 추가 하지 않고 닫으려면 **취소** 합니다.
7. 계속 하려면 **다음** 을 선택 합니다.
8. **검토** 페이지에서 우선 순위 사용자 그룹에 대해 선택한 설정을 검토 합니다. **편집** 을 선택 하 여 그룹 값을 변경 하거나 **제출을** 선택 하 여 우선 순위 사용자 그룹을 만들고 활성화 합니다.
9. 확인 페이지에서 **완료** 를 선택 하 여 마법사를 종료 합니다.

### <a name="update-a-priority-user-group"></a>우선 순위 사용자 그룹 업데이트

기존 우선 순위 사용자 그룹을 업데이트 하려면 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션에서 설정 컨트롤을 사용 합니다. 우선 순위 사용자 그룹을 업데이트 하려면 *참가자 위험 관리* 또는 *참가자 위험 관리 관리자* 역할 그룹의 구성원 이어야 합니다.

우선 순위 사용자 그룹을 편집 하려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정을**선택 합니다.
2. **우선 순위 사용자 그룹** 탭 선택
3. 편집할 우선 순위 사용자 그룹을 선택 하 고 **그룹 편집**을 선택 합니다.
4. **그룹 정의** 페이지에서 필요한 경우 설명 필드를 업데이트 합니다. 우선 순위 사용자 그룹의 이름은 업데이트할 수 없습니다. 계속 하려면 **다음** 을 선택 합니다.
5. **구성원 선택** 페이지에서 **구성원 선택** 컨트롤을 사용 하 여 그룹에 새 구성원을 추가 합니다. 그룹에서 사용자를 제거 하려면 제거할 사용자 옆에 있는 ' X '를 선택 합니다. 계속 하려면 **다음** 을 선택 합니다.
6. **검토** 페이지에서 우선 순위 사용자 그룹에 대해 선택한 업데이트 설정을 검토 합니다. **편집** 을 선택 하 여 그룹 값을 변경 하거나 **제출을** 선택 하 여 우선 순위 사용자 그룹을 업데이트 합니다.
7. 확인 페이지에서 **완료** 를 선택 하 여 마법사를 종료 합니다.

### <a name="delete-a-priority-user-group"></a>우선 순위 사용자 그룹 삭제

기존 우선 순위 사용자 그룹을 삭제 하려면 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션에서 설정 컨트롤을 사용 합니다. 우선 순위 사용자 그룹을 삭제 하려면 *참가자 위험 관리* 또는 *참가자 위험 관리 관리자* 역할 그룹의 구성원 이어야 합니다.

>[!IMPORTANT]
>우선 순위 사용자 그룹을 삭제 하면 할당 된 모든 활성 정책에서이를 제거 합니다. 활성 정책에 할당 된 우선 순위 사용자 그룹을 삭제 하는 경우 정책에는 범위 내 사용자가 포함 되지 않으며 실제로 유휴 상태가 되며 알림을 만들지 않습니다.

우선 순위 사용자 그룹을 삭제 하려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정을**선택 합니다.
2. **우선 순위 사용자 그룹** 탭 선택
3. 편집할 우선 순위 사용자 그룹을 선택 하 고 대시보드 메뉴에서 **삭제** 를 선택 합니다.
4. **삭제** 대화 상자에서 **예** 를 선택 하 여 우선 순위 사용자 그룹을 삭제 하거나 **취소** 를 선택 하 여 대시보드로 돌아갑니다.

## <a name="priority-physical-assets-preview"></a>우선 순위 물리적 자산 (미리 보기)

우선 순위 물리적 자산에 대 한 액세스를 확인 하 고 사용자 이벤트에 대 한 액세스 활동을 연결 하는 것은 준수 인프라의 중요 구성 요소입니다 이러한 실제 자산은 회사 건물, 데이터 센터 또는 서버 채팅방과 같은 조직의 우선 순위 위치를 나타냅니다. 참가자 위험 활동은 특이 한 시간을 작업 하는 사용자에 게 연결 될 수 있으며, 승인 되지 않은 중요 또는 보안 영역에 액세스 하 고, 합법적인 요구 없이 높은 수준의 영역에 대 한 액세스를 요청 합니다.

우선 순위 물리적 자산을 사용 하도록 설정 하 고 [물리적 배지 획득 데이터 커넥터](import-physical-badging-data.md) 를 구성 하면, 참가자 위험 관리는 실제 제어 및 액세스 시스템에서 신호를 다른 사용자 위험 활동과 통합 합니다. 실제 액세스 시스템에서 동작의 패턴을 검사 하 고 이러한 활동을 다른 참가자 위험 이벤트와 연결 하면 참가자 위험 관리를 통해 규정 준수 investigators 및 분석가에 게 경고에 대 한 응답 의사 결정을 더 많이 받을 수 있습니다. 우선 순위에 대 한 액세스는 중요 하지 않은 자산의 액세스와는 다른 방식으로 이해를 받습니다.

예를 들어 조직에는 정상 작업 및 중요 한 프로젝트 영역에 대 한 실제 액세스를 모니터링 하 고 승인 하는 사용자를 위한 배지 획득 시스템이 있습니다. 중요 한 프로젝트에서 여러 명의 사용자가 작업을 수행 하는 경우 해당 사용자가 프로젝트가 완료 되 면 조직의 다른 영역으로 돌아옵니다. 중요 한 프로젝트가 완료 되 면 프로젝트 작업의 기밀을 유지 하 고 프로젝트 영역에 대 한 액세스를 긴밀 하 게 제어 해야 합니다.

Microsoft 365에서 실제 배지 획득 데이터 커넥터를 사용 하 여 실제 배지 획득 시스템에서 액세스 정보를 가져오고, 참가자 위험 관리에서 우선 순위 물리적 자산을 지정 하도록 선택 합니다. 배지 획득 시스템에서 정보를 가져오고 실제 액세스 정보를 참가자 위험 관리에서 식별 된 다른 위험 활동과 연결 하 여, 프로젝트의 사용자 중 한 명에 게 정상적인 작업 시간 후에 프로젝트 사무소에 액세스 하 고 있으며 많은 양의 데이터를 일반 작업 영역에서 개인 클라우드 저장소 서비스로 내보내는 것을 확인할 수 있습니다. 온라인 활동에 연결 된이 물리적 액세스 활동은 가능한 데이터 절도 및 준수 investigators를 가리킬 수 있으며, 분석가는이 사용자의 상황에 따라 적절 한 작업을 수행할 수도 있습니다.

### <a name="configure-priority-physical-assets"></a>우선 순위 물리적 자산 구성

우선 순위 물리적 자산을 구성 하려면 실제 배지 획득 커넥터를 구성 하 고 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션에서 설정 컨트롤을 사용 합니다. 우선 순위 실제 자산을 구성 하려면 *참가자 위험 관리* 또는 *참가자 위험 관리 관리자 역할 그룹*의 구성원 이어야 합니다.

우선 순위 물리적 자산을 구성 하려면 다음 단계를 완료 합니다.

1. [참가자 위험 관리 시작](insider-risk-management-configure.md) 문서에서 참가자 위험 관리에 대 한 구성 단계를 따릅니다. 3 단계에서 실제 배지 획득 커넥터를 구성 했는지 확인 합니다.

    >[!IMPORTANT]
    >참가자 위험 관리 정책에 대해 departing 및 종료 된 사용자와 관련 한 신호 데이터를 사용 하 고 실제 제어 및 액세스 플랫폼과의 이벤트 데이터와 상호 연결 하려면 Microsoft 365 HR 커넥터도 구성 해야 합니다. Microsoft 365 HR 커넥터를 사용 하지 않고 실제 배지 획득 커넥터를 사용 하도록 설정 하는 경우, 참가자 위험 관리 정책은 조직의 사용자에 대 한 실제 액세스 작업에 대 한 이벤트만 처리 합니다.

2. [Microsoft 365 준수 센터](https://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정**  >  **우선 순위 실제 자산**을 선택 합니다.
3. **우선 순위 실제 자산** 페이지에서 실제 배지 획득 커넥터로 가져온 자산 이벤트에 대해 모니터링할 실제 자산 id를 수동으로 추가 하거나 a를 가져올 수 있습니다. 실제 배지 획득 커넥터에서 가져온 모든 실제 자산 Id의 CSV 파일: a) 실제 자산 Id를 수동으로 추가 하려면 **우선 순위 물리적 자산 추가**를 선택 하 고, 실제 자산 ID를 입력 한 후에 **추가**를 선택 합니다. 추가 물리적 자산 Id를 입력 하 고 **우선 순위를 지정한 실제 자산** 을 선택 하 여 입력 한 모든 자산을 저장 합니다.
    b)에서 실제 자산 Id 목록을 추가 합니다. CSV 파일에서 **우선 순위 물리적 자산 가져오기를**선택 합니다. 파일 탐색기 대화 상자에서를 선택 합니다. CSV 파일을 가져온 다음 **열기**를 선택 합니다. 의 실제 자산 Id입니다. CSV 파일이 목록에 추가 됩니다.
4. 설정에서 **정책 지표** 탭으로 이동 합니다.
5. **정책 지표** 페이지에서 **물리적 액세스 지표** 섹션으로 이동한 **후 중요 한 자산에 대 한 액세스 종료 또는 실패 한 실제 액세스**확인란을 선택 합니다.
6. **저장** 을 선택 하 여 구성 하 고 종료 합니다.

### <a name="delete-a-priority-physical-asset"></a>우선 순위 물리적 자산 삭제

기존 우선 순위 실제 자산을 삭제 하려면 Microsoft 365 준수 센터의 참가자 위험 관리 솔루션에서 설정 컨트롤을 사용 합니다. 우선 순위 실제 자산을 삭제 하려면 사용자의 참가자 위험 관리 또는 참가자 관리 역할 그룹의 구성원 이어야 합니다.

>[!IMPORTANT]
>우선 순위 실제 자산을 삭제 하면 이전에 포함 되었던 활성 정책에서 해당 항목을 검사 하는 것이 취소 됩니다. 우선 순위 물리적 자산과 연결 된 작업에 의해 생성 된 경고는 삭제 되지 않습니다.

우선 순위 물리적 자산을 삭제 하려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정**  >  **우선 순위 실제 자산**을 선택 합니다.
2. **우선 순위 실제 자산** 페이지에서 삭제할 자산을 선택 합니다.
3. 작업 메뉴에서 **삭제** 를 선택 하 여 자산을 삭제 합니다.

## <a name="power-automate-flows-preview"></a>전원 자동화 흐름 (미리 보기)

[Microsoft Power 자동화](https://docs.microsoft.com/power-automate/getting-started) 는 응용 프로그램 및 서비스에서 작업을 자동화 하는 워크플로 서비스입니다. 템플릿에서 흐름을 사용 하거나 수동으로 만든 경우 이러한 응용 프로그램 및 서비스와 관련 된 일반적인 작업을 자동화할 수 있습니다. 참가자 위험 관리에 대해 전원 자동화 흐름을 사용 하도록 설정 하는 경우 사례 및 사용자에 대 한 중요 한 작업을 자동화할 수 있습니다. 사용자, 경고 및 사례 정보를 검색 하 고이 정보를 관련자 및 기타 응용 프로그램과 공유 하 고, 사례 노트에 게시 하는 등의 참가자 위험 관리 작업을 자동화 하는 방법에 대 한 전원 자동화 흐름을 구성할 수 있습니다. 전원 자동화 흐름은 사례 및 정책 범위에 있는 모든 사용자에 게 적용 됩니다.

Microsoft 365 구독에 참가자 위험 관리를 포함 하는 고객은 권장 되는 참가자 위험 관리 파워 자동화 템플릿을 사용 하기 위해 추가 전원 자동화 라이선스가 필요 하지 않습니다. 이러한 서식 파일을 사용자 지정 하 여 조직을 지원 하 고 핵심 참가자 위험 관리 시나리오를 처리할 수 있습니다. 이러한 서식 파일에서 고급 고급 자동화 기능을 사용 하도록 선택 하는 경우 Microsoft 365 준수 커넥터를 사용 하 여 사용자 지정 서식 파일을 만들거나 Microsoft 365의 기타 준수 영역에 대해 Power 자동화 서식 파일을 사용 하는 경우에는 추가 전원 자동화 라이선스가 필요할 수 있습니다.

>[!IMPORTANT]
>전원 자동화 흐름을 테스트할 때 추가 라이선스 유효성 검사에 대 한 메시지를 받고 있습니까? 조직에서이 미리 보기 기능에 대 한 서비스 업데이트를 아직 받지 못한 것일 수 있습니다. 업데이트를 배포 하는 중 이며, 참가자 위험 관리를 포함 하는 Microsoft 365 구독을 가진 모든 조직은 권장 되는 전원 자동화 서식 파일에서 만든 흐름에 대 한 라이선스 지원이 있어야 합니다 (10 월 30 일, 2020).

다음 파워 자동화 템플릿은 고객에 게 참가자 위험 관리 사용자 및 사례에 대 한 프로세스 자동화를 지원 하도록 제공 됩니다.

- **사용자에 게 참가자 위험 정책에 추가 될 때 알림**:이 서식 파일은 사용자가 위험 관리 정책을 받을 때 알려야 하는 내부 정책, 개인 정보 또는 규정 요구 사항이 있는 조직에 해당 합니다. 사용자에 대해 사용자에 대해이 흐름을 구성 하 고 선택 하면 사용자가 참가자 위험 관리 정책에 추가 될 때 사용자 및 관리자에 게 전자 메일 메시지가 전송 됩니다. 또한이 서식 파일은 SharePoint 사이트에서 호스트 되는 SharePoint 목록을 업데이트 하 여 날짜/시간 및 메시지 받는 사람 등의 알림 메시지 세부 정보를 추적할 수 있도록 지원 합니다. **개인 정보 설정**에서 사용자를 않지만 익명화 선택한 경우이 서식 파일에서 만든 흐름은 사용자 개인 정보를 유지 관리 하기 위해 정상적으로 작동 하지 않습니다. 이 템플릿을 사용 하는 전원 자동화 흐름은 **사용자 대시보드에서**사용할 수 있습니다.
- **사용자에 대해 hr 또는 business에 대 한 정보를 요청**합니다. 대/소문자를 함께 사용 하는 경우 참가자 위험 분석가와 investigators가 사례 활동의 컨텍스트를 이해 하기 위해 hr 또는 기타 관련자와 의견을 함께 받아야 할 수 있습니다. 사례에 대해이 흐름을 구성 하 고 선택 하면 분석가와 investigators에서이 흐름에 대해 구성 된 HR 및 비즈니스 관련자에 게 전자 메일 메시지를 보냅니다. 각 받는 사람에 게 미리 구성 되거나 사용자 지정 가능한 응답 옵션이 포함 된 메시지가 전송 됩니다. 받는 사람이 응답 옵션을 선택 하면 응답이 사례 메모로 기록 되 고 받는 사람 및 날짜/시간 정보가 포함 됩니다. **개인 정보 설정**에서 사용자를 않지만 익명화 선택한 경우이 서식 파일에서 만든 흐름은 사용자 개인 정보를 유지 관리 하기 위해 정상적으로 작동 하지 않습니다. 이 템플릿을 사용 하는 전원 자동화 흐름은 **사례 대시보드에서**사용할 수 있습니다.
- **알림 관리자에 게 사용자의 참가자 위험 경고 발생**: 사용자에 게 참가자 위험 관리 경고가 있을 경우 일부 조직에서 즉각적인 관리 알림을 받아야 할 수 있습니다. 이 흐름을 구성 하 고 선택 하면 사례 사용자에 대 한 관리자에 게 전자 메일 메시지를 전송 하 고 모든 사례 경고에 대 한 다음 정보가 표시 됩니다. 
    - 경고에 적용할 수 있는 정책
    - 경고 날짜/시간
    - 경고 심각도 수준입니다.

    흐름에서는 메시지가 전송 된 경우 및 흐름이 활성화 되었음을 나타내는 사례 정보를 자동으로 업데이트 합니다. **개인 정보 설정**에서 사용자를 않지만 익명화 선택한 경우이 서식 파일에서 만든 흐름은 사용자 개인 정보를 유지 관리 하기 위해 정상적으로 작동 하지 않습니다. 이 템플릿을 사용 하는 전원 자동화 흐름은 **사례 대시보드에서**사용할 수 있습니다.

- **참가자 위험에 대 한 추가 작업에 일정 미리 알림 추가**:이 서식 파일을 사용 하면 위험 investigators 및 분석가가 Office 365 Outlook 일정에 사례에 대 한 일정 미리 알림을 추가할 수 있습니다. 이 흐름은 사례를 처리 하 고 알림을 심사 할 때 사용자가 자신의 참가자 위험 관리 워크플로를 끝내 거 나 전환할 필요성을 제거 합니다. 이 흐름을 구성 하 고 선택 하면 흐름을 실행 하는 사용자에 대해 Office 365 Outlook 일정에 미리 알림이 추가 됩니다. 이 템플릿을 사용 하는 전원 자동화 흐름은 **사례 대시보드에서**사용할 수 있습니다.

### <a name="create-a-power-automate-flow-from-insider-risk-management-template"></a>참가자 위험 관리 템플릿에서 전원 자동화 흐름 만들기

권장 되는 참가자 위험 관리 템플릿에서 전원 자동화 흐름을 만들려면 **서비스 케이스** 또는 **사용자 대시보드에서**직접 작업할 때 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션 또는 **전원 자동화 흐름 관리** 옵션의 설정 컨트롤을 **사용 합니다.**

설정 영역에서 전원 자동화 흐름을 만들려면 *참가자 위험 관리* 또는 *참가자 위험 관리 관리자* 역할 그룹의 구성원 이어야 합니다. **전원 자동화 흐름 관리** 옵션을 사용 하 여 전원 자동화 흐름을 만들려면 하나 이상의 참가자 위험 관리 역할 그룹의 구성원 이어야 합니다.

권장 되는 참가자 위험 관리 템플릿에서 전원 자동화 흐름을 만들려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정**에 대 한  >  **전원 자동화 흐름**을 선택 합니다. 또한 **자동**으로 **Cases** **Users dashboards**  >  **전원 자동화 흐름 관리**를 선택 하 여 사례 또는 사용자 대시보드 페이지에서 액세스할 수 있습니다.
2. **전원 자동화 흐름** 페이지의 페이지에 있는 **참가자 위험 관리 서식 파일** 섹션에서 권장 되는 서식 파일을 선택 합니다.
3. 흐름에는 흐름에 필요한 포함 된 연결이 나열 되며, 연결 상태를 사용할 수 있는지 여부를 확인 합니다. 필요한 경우 표시 되지 않는 모든 연결을 사용 가능으로 업데이트 합니다. **계속**을 선택 합니다.
4. 기본적으로 권장 되는 흐름은 해당 흐름에 대해 할당 된 작업을 완료 하는 데 필요한 권장 되는 참가자 위험 관리 및 Microsoft 365 서비스 데이터 필드를 사용 하 여 미리 구성 됩니다. 필요한 경우 **고급 옵션 표시** 컨트롤을 사용 하 고 흐름 구성 요소에 대해 사용 가능한 속성을 구성 하 여 흐름 구성 요소를 사용자 지정 합니다.
5. 필요한 경우 **새 단계** 단추를 선택 하 여 흐름에 추가 단계를 추가 합니다. 대부분의 경우 권장 되는 기본 서식 파일에는이를 사용할 필요가 없습니다.
6. 추가 구성에 대 한 흐름을 저장 하려면 **초안 저장** 을 선택 하 고 흐름에 대 한 구성을 완료 하려면 **저장** 을 선택 합니다.
7. **닫기를** 선택 하 여 **전원 자동화 흐름** 페이지로 돌아갑니다. 새 서식 파일은 **내 흐름** 탭에 흐름으로 나열 되며, 흐름을 만드는 사용자의 참가자 위험 관리 사례로 작업할 **때 자동으로 드롭다운 컨트롤** 에서 사용할 수 있습니다.

>[!IMPORTANT]
>조직의 다른 사용자가 흐름에 액세스 해야 하는 경우 흐름을 공유 해야 합니다.

### <a name="create-a-custom-power-automate-flow-for-insider-risk-management"></a>참가자 위험 관리를 위한 사용자 지정 전원 자동화 흐름 만들기

조직에 대 한 일부 프로세스 및 워크플로는 권장 되는 참가자 위험 관리 흐름 서식 파일의 외부에 있을 수 있으며, 참가자 위험 관리 영역에 대해 사용자 지정 전원 자동화 흐름을 만들 필요가 있을 수 있습니다. 전원 자동화 흐름은 융통성이 높으며 광범위 한 사용자 지정을 지원 하지만, 참가자 위험 관리 기능과 통합 하려면 몇 가지 단계를 수행 해야 합니다.

다음 단계를 완료 하 여 참가자 위험 관리를 위한 사용자 지정 전원 자동화 서식 파일을 만듭니다.

1. **전원 자동화 흐름 라이선스 확인**: 참가자 위험 관리 트리거를 사용 하는 사용자 지정 된 전원 자동화 흐름을 만들려면 전원 자동화 라이선스가 필요 합니다. 권장 되는 참가자 위험 관리 흐름 템플릿은 추가 라이선스가 필요 하지 않으며, 사용자의 참가자 위험 관리 라이선스의 일부로 포함 됩니다.
2. **자동화 된 흐름 만들기**: 참가자 위험 관리 이벤트에 의해 트리거된 후 하나 이상의 작업을 수행 하는 흐름을 만듭니다. 자동화 된 흐름을 만드는 방법에 대 한 자세한 내용은 [전원 자동화에서 흐름 만들기](https://docs.microsoft.com/power-automate/get-started-logic-flow)를 참조 하십시오.
3. **Microsoft 365 준수 커넥터**:를 검색 하 여 microsoft 365 준수 커넥터를 선택 합니다. 이 커넥터는 참가자 위험 관리 트리거 및 작업을 사용 하도록 설정 합니다. 커넥터에 대 한 자세한 내용은 [커넥터 참조 개요](https://docs.microsoft.com/connectors/connector-reference/) 문서를 참조 하세요.
4. **흐름에 대해 참가자 위험 관리 트리거 선택**: 참가자 위험 관리에는 사용자 지정 전원 자동화 흐름에 사용할 수 있는 다음과 같은 두 가지 트리거가 있습니다.
    - **선택한 참가자 위험 관리 사례**:이 트리거를 사용 하는 흐름을 참가자 위험 관리 사례 대시보드 페이지에서 선택할 수 있습니다.
    - **선택한 참가자 위험 관리 사용자의 경우**:이 트리거를 사용 하는 흐름은 참가자 위험 관리 사용자 대시보드 페이지에서 선택할 수 있습니다.
5. 흐름에 대 한 참가자 위험 관리 작업 선택: 사용자 지정 흐름에 포함할 참가자 위험 관리의 몇 가지 작업을 선택할 수 있습니다.
    - 참가자 위험 관리 알림 받기
    - 참가자 위험 관리 사례 가져오기
    - 참가자 위험 관리 사용자 가져오기
    - 사례에 대 한 참가자 위험 관리 알림 받기
    - 참가자 위험 관리 사례 추가 정보

### <a name="share-a-power-automate-flow"></a>전원 자동화 흐름 공유

기본적으로 사용자가 만든 전원 자동화 흐름은 해당 사용자만 사용할 수 있습니다. 다른 참가자 위험 관리 사용자에 게 액세스 및 흐름을 사용 하려면 흐름 작성자가 흐름을 공유 해야 합니다. 흐름을 공유 하려면 **사례** 또는 **사용자 대시보드** 페이지에서 직접 작업할 때 Microsoft 365 준수 센터의 **참가자 위험 관리 솔루션** 또는 자동 제어에서 **전원 자동화 흐름 관리** 옵션의 설정 컨트롤을 사용 합니다. 흐름을 공유한 후에는 공유 된 모든 사용자가 **사례** 및 **사용자 대시보드의** **자동화** 컨트롤 드롭다운에서 흐름에 액세스할 수 있습니다.

설정 영역에서 전원 자동화 흐름을 공유 하려면 사용자의 *참가자 위험 관리* 또는 *참가자 위험 관리 관리자* 역할 그룹의 구성원 이어야 합니다. 전원 자동화 흐름 **관리** 옵션을 사용 하 여 전원 자동 흐름을 공유 하려면 하나 이상의 참가자 위험 관리 역할 그룹의 구성원 이어야 합니다.

전원 자동화 흐름을 공유 하려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](htttps://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정**에 대 한  >  **전원 자동화 흐름**을 선택 합니다. 또한 **자동**으로 **Cases** **Users dashboards**  >  **전원 자동화 흐름 관리**를 선택 하 여 사례 또는 사용자 대시보드 페이지에서 액세스할 수 있습니다.
2. **전원 자동화 흐름** 페이지에서 **내 흐름** 또는 **팀 흐름** 탭을 선택 합니다.
3. 공유할 흐름을 선택한 다음 흐름 옵션 메뉴에서 **공유** 를 선택 합니다.
4. 흐름 공유 페이지에서 흐름의 소유자로 추가할 사용자 또는 그룹의 이름을 입력 합니다.
5. 사용 된 **연결** 대화 상자에서 **확인** 을 선택 하 여 추가 된 사용자 또는 그룹이 흐름에 대 한 전체 액세스 권한을 수락 함을 승인 합니다.

흐름을 편집 하려면 **사례** 또는 **사용자 대시보드에서**직접 작업을 수행할 때 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션 또는 **전원 자동화 흐름 관리** 옵션의 설정 컨트롤을 **사용 합니다.**

설정 영역에서 전원 자동화 흐름을 편집 하려면 사용자의 *참가자 위험 관리* 또는 *참가자 관리* 역할 그룹의 구성원 이어야 합니다. **전원 자동화 흐름 관리** 옵션을 사용 하 여 절전 자동 흐름을 편집 하려면 하나 이상의 참가자 위험 관리 역할 그룹의 구성원 이어야 합니다.

전원 자동화 흐름을 편집 하려면 다음 단계를 완료 합니다.

3. 흐름 구성 요소를 변경 하려면 **줄임표 (설정)**  >  **설정을** 선택 하 고 흐름도를 삭제 하려면 **줄임표**를  >  **삭제** 합니다.
4. **저장** 을 선택 하 고 **닫기를** 클릭 하 여 흐름 편집을 완료 합니다.

### <a name="delete-a-power-automate-flow"></a>전원 자동화 흐름 삭제

흐름을 삭제 하려면 Microsoft 365 준수 센터의 **참가자 위험 관리** 솔루션 또는 **사례** 또는 **사용자 대시보드에서**직접 작업을 수행 하는 경우 **자동화** 컨트롤에서 **전원 자동화 흐름 관리** 옵션의 설정 컨트롤을 사용 합니다. 흐름을 삭제 하면 모든 사용자에 대 한 옵션으로 제거 됩니다.

설정 영역에서 전원 자동화 흐름을 삭제 하려면 *참가자 위험 관리* 또는 *참가자 관리* 역할 그룹의 구성원 이어야 합니다. **전원 자동화 흐름 관리** 옵션을 사용 하 여 전원 자동화 흐름을 삭제 하려면 적어도 하나의 참가자 위험 관리 역할 그룹의 구성원 이어야 합니다.

전원 자동화 흐름을 삭제 하려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](htttps://compliance.microsoft.com)에서 **참가자 위험 관리** 로 이동 하 여 **참가자 위험 설정**에 대 한  >  **전원 자동화 흐름**을 선택 합니다. 또한 **자동**으로 **Cases** **Users dashboards**  >  **전원 자동화 흐름 관리**를 선택 하 여 사례 또는 사용자 대시보드 페이지에서 액세스할 수 있습니다.
2. **전원 자동화 흐름** 페이지에서 삭제할 흐름을 선택 하 고 흐름 제어 메뉴에서 **삭제** 를 선택 합니다.
3. 삭제 확인 대화 상자에서 **삭제** 를 선택 하 여 흐름을 제거 하거나 **취소** 를 선택 하 여 삭제 작업을 종료 합니다.

## <a name="microsoft-teams-preview"></a>Microsoft 팀 (미리 보기)

규정 준수 분석가와 investigators은 참가자 위험 관리 사례에 대 한 공동 작업을 위해 Microsoft 팀을 쉽게 사용할 수 있습니다. Microsoft 팀의 다른 관련자와 협력 하 여 다음을 수행할 수 있습니다.

- 비공개 팀 채널에서 사례에 대 한 응답 활동 조정 및 검토
- 개별 사례와 관련 된 파일 및 증거를 안전 하 게 공유 및 저장
- 분석가 및 investigators의 응답 활동 추적 및 검토

Microsoft 팀이 참가자 위험 관리를 사용 하도록 설정한 후에는 경고가 확인 되 고 사례를 만들 때마다 전용 Microsoft 팀 팀이 만들어집니다. 기본적으로 팀은 *참가자 위험 관리*, *참가자 위험 관리 분석가*및 *참가자 위험 관리 Investigators* 역할 그룹 (최대 100 초기 사용자)의 모든 구성원을 자동으로 포함 합니다. 조직 작성자를 만들고 적절 한 경우 팀에 추가할 수 있습니다. Microsoft 팀을 사용 하도록 설정 하기 전에 만든 기존 사례에 대해 분석가 및 investigators는 필요한 경우 사례에서 작업 하는 경우 새 Microsoft 팀 팀을 만들 수 있습니다.  참가자 위험 관리에서 관련 사례를 해결 한 후에는 팀이 자동으로 보관 됩니다 (숨김 및 읽기 전용으로 이동 됨).

Microsoft 팀에서 팀 및 채널을 사용 하는 방법에 대 한 자세한 내용은 [Microsoft 팀의 팀 및 채널 개요](https://docs.microsoft.com/MicrosoftTeams/teams-channels-overview)를 참조 하세요.

Microsoft 팀에서 사례에 대 한 지원을 사용 하도록 설정 하는 것은 쉽고 빠르게 구성 합니다. Microsoft 팀이 참가자의 위험 관리를 사용 하도록 설정 하려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](htttps://compliance.microsoft.com)에서 **참가자 위험 관리**  >  **참가자의 위험 설정**으로 이동 합니다.
2. **Microsoft 팀** 탭을 선택 합니다.
3. 참가자 위험 관리를 위해 Microsoft 팀 통합을 사용 하도록 설정 합니다.
4. **저장** 을 선택 하 여 구성 하 고 종료 합니다.

### <a name="create-a-microsoft-teams-team-for-existing-cases"></a>기존 사례를 위한 Microsoft 팀 팀 만들기

기존 사례가 있는 후 Microsoft 팀이 참가자 위험 관리를 지원 하도록 설정 하는 경우 필요에 따라 각 사례에 대 한 팀을 수동으로 만들어야 합니다. 참가자 위험 관리 설정에서 Microsoft 팀 지원을 사용 하도록 설정 하면 새 사례에서 새 Microsoft 팀 팀을 자동으로 만듭니다.

조직에 Microsoft 365 그룹을 만들 수 있는 권한이 있어야 사용자에 게 Microsoft 팀 팀이 사례에서 작성할 수 있습니다. Microsoft 365 그룹의 사용 권한을 관리 하는 방법에 대 한 자세한 내용은 [microsoft 365 그룹을 만들 수 있는 사용자 관리](https://docs.microsoft.com/microsoft-365/solutions/manage-creation-of-groups)를 참조 하세요.

팀을 만들려면 기존 사례에서 직접 작업 하는 경우 Microsoft 팀 구성 컨트롤을 사용 합니다. 새 팀을 만들려면 다음 단계를 완료 합니다.

1. [Microsoft 365 준수 센터](htttps://compliance.microsoft.com)에서 **참가자 위험 관리**  >  **사례로** 이동 하 여 기존 사례를 선택 합니다.
2. 사례 작업 메뉴에서 **Microsoft 팀 만들기**를 선택 합니다.
3. **팀 이름** 필드에 새 Microsoft 팀 팀의 이름을 입력 합니다.
4. **Microsoft 팀 만들기** 를 선택한 다음 **닫기를**선택 합니다.

참가자 위험 관리 역할 그룹에 할당 된 사용자 수에 따라, 사례에 대해 모든 investigators 및 분석가를 Microsoft 팀 팀에 추가 하는 데 15 분 정도 걸릴 수 있습니다.
