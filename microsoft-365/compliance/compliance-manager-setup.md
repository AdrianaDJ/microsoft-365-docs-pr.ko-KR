# <a name="get-started-with-compliance-manager"></a>규정 준수 관리자 시작

**이 문서의 내용** 이 문서는 준수 관리자를 설정 하는 데 도움이 됩니다. 준수 관리자에 **액세스** 하 고, **역할 및 사용 권한을 설정** 하 고, **개선 작업의 자동 테스트** 를 구성 하는 방법을 알아봅니다. **준수 관리자 대시보드** 를 안내 하 고 주요 페이지: 개선 작업 페이지, 솔루션 페이지, 평가 페이지 및 평가 템플릿 페이지를 이해 합니다.

## <a name="who-can-access-compliance-manager"></a>준수 관리자에 액세스할 수 있는 사용자

준수 관리자는 Office 365 및 Microsoft 365 라이선스가 있는 조직에서 사용할 수 있습니다. 평가 가용성 및 관리 기능은 사용권 계약에 따라 달라 집니다.  [서비스 설명 세부 정보를 확인](https://go.microsoft.com/fwlink/?linkid=2132371)합니다.

## <a name="before-you-begin"></a>시작하기 전에

조직에 대 한 Microsoft 365 전역 관리자는 준수 관리자에 게 액세스 하는 첫 번째 사용자 일 것입니다. 준수 관리자를 처음 방문 하면 전역 관리자 로그인 및 아래에 설명 된 대로 사용자 권한을 설정 하는 것이 좋습니다.

## <a name="sign-in"></a>로그인

1. [Microsoft 365 준수 센터로](https://compliance.microsoft.com/) 이동 하 여 microsoft 365 전역 관리자 계정으로 **로그인** 합니다.
2. 왼쪽 탐색 창에서 **준수 관리자** 를 선택 합니다. [준수 관리자 대시보드에](#understand-the-compliance-manager-dashboard)도착 합니다.

액세스 준수 관리자에 대 한 직접 링크는 [https://compliance.microsoft.com/compliancemanager](https://compliance.microsoft.com/compliancemanager) 입니다.

## <a name="set-user-permissions-and-assign-roles"></a>사용자 권한 설정 및 역할 할당

준수 관리자는 RBAC (역할 기반 액세스 제어) 권한 모델을 사용 합니다. 역할이 할당 된 사용자만 준수 관리자에 액세스할 수 있으며, 각 사용자가 허용한 작업은 [역할 유형에](#role-types)따라 제한 됩니다.

### <a name="where-to-set-permissions"></a>사용 권한을 설정 하는 위치

조직에 대 한 전역 관리자 역할을 보유 하 고 있는 사용자는 Azure Active Directory (Azure AD) 뿐만 아니라 Microsoft 365 준수 센터에서 사용자 권한을 설정할 수 있습니다.

Microsoft 365 준수 센터 내에서 사용 권한을 설정 하 고 역할을 할당 하려면 다음 단계를 수행 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)의 아무 곳에서 나 왼쪽 탐색 창에 대 한 **사용 권한을** 선택 합니다.

2. 위쪽의 **"Office 365에서 역할을 보고 관리 하려면 여기를 클릭 하세요."** 라는 링크를 선택 합니다. 새 탭이 Office 365 보안 & 준수 센터에 열립니다 ([리디렉션되는 이유를 알아봅니다](microsoft-365-compliance-center.md#frequently-asked-questions)).

3. 사용자를 한 명 이상 추가할 역할 그룹을 찾은 다음 그룹 이름 왼쪽에 있는 확인란을 선택 합니다. [아래의 역할 및 관련 기능 목록을](#role-types)참조 하세요. 역할 그룹 이름은 역할 이름을 모방 합니다.

4. 해당 그룹의 플라이 아웃 창에서 **구성원** 헤더 아래에 있는 **편집** 을 선택 합니다.

5. **구성원 선택을** 선택 합니다. 다른 플라이 아웃 창이 표시 됩니다.

6. **+ 추가** 를 선택 하 여 그룹에 추가할 사용자를 한 명 이상 선택 합니다.

7. 추가할 이름 옆의 확인란을 선택한 다음 아래쪽에 있는 **추가** 단추를 선택 합니다.

8. 사용자 지정이 끝나면 **완료** 를 선택 하 고 **저장** 을 선택한 다음 **닫기를** 선택 합니다.

##### <a name="more-about-the-office-365-secruity--compliance-center"></a>Office 365 Sec티 & 준수 센터에 대 한 자세한 정보

자세한 내용은 [Office 365 보안 & 준수 센터의 사용 권한에](https://docs.microsoft.com/microsoft-365/security/office-365-security/permissions-in-the-security-and-compliance-center)대 한 자세한 내용을 알아보세요.

Office 365 보안 및 준수 센터에 대 한 액세스 권한이 없거나 Microsoft Service Trust Portal에서 준수 관리자의 클래식 버전에 액세스 해야 하는 경우 서비스 트러스트 포털의 관리 설정에 따라 역할을 할당 하는 다른 방법이 제공 됩니다 ([지침 보기](meet-data-protection-and-regulatory-reqs-using-microsoft-cloud.md#assigning-compliance-manager-roles-to-users)). 이러한 역할은 해당 기능에서 더 제한 됩니다.

##### <a name="more-about-azure-ad"></a>Azure AD에 대 한 자세한 정보

Azure AD에서 역할을 할당 하 고 사용 권한을 설정 하려면 [Azure Active Directory를 사용 하 여 사용자에 게 관리자 및 비관리자 역할 할당](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-users-assign-role-azure-portal)을 참조 하세요.

Office 365 또는 Microsoft 365 구독을 사용 하지 않는 Azure AD id가 있는 사용자는 Microsoft 365 준수 센터에서 준수 관리자에 액세스할 수 없습니다. 준수 관리자에 대 한 액세스를 지원 하려면 [cmresearch@microsoft.com](mailto:cmresearch@microsoft.com)에 문의 하십시오.

## <a name="settings-for-automated-testing-and-user-history"></a>자동 테스트 및 사용자 기록 설정

Microsoft 365 준수 센터의 준수 관리자 설정을 통해 개선 작업의 자동 테스트를 사용 하도록 설정 하 고 사용 하지 않도록 설정할 수 있습니다. 또한이 설정을 사용 하면 향상 작업을 다른 사용자에 게 다시 할당 하는 기능을 포함 하 여 개선 작업과 연관 된 사용자의 데이터를 관리할 수 있습니다.  전역 관리자 또는 준수 관리자 관리자 역할이 있는 사용자만 준수 관리자 설정에 액세스할 수 있습니다.

### <a name="set-up-automated-testing"></a>자동화 된 테스트 설정

준수 관리자의 일부 향상 작업은 [Microsoft 보안 점수](https://docs.microsoft.com/microsoft-365/security/mtp/microsoft-secure-score)에 의해서도 모니터링 됩니다. 공동 모니터링 되는 작업에 대 한 자동화 된 테스트를 설정할 수 있는데,이는 동작이 보안 점수에 따라 테스트 및 업데이트 되 면 해당 결과가 준수 관리자의 동일한 작업과 동기화 되 고 준수 점수에 도달 하는 것을 의미 합니다.

자동 테스트는 규정 준수 관리자에 게 신규 조직의 경우 기본적으로 설정 됩니다. Microsoft 365 또는 Office 365을 처음 배포 하는 경우 보안 점수가 약 7 일인 경우에만 데이터를 완벽 하 게 수집 하 고 준수 점수를 파악 합니다.  자동화 된 테스트를 설정 하면 작업의 테스트 날짜가 업데이트 되지 않지만 테스트 상태는 업데이트 됩니다. 새 평가를 만들면 점수가 자동으로 Microsoft 제어 점수와 보안 점수 통합을 포함 합니다.

조직의 전역 관리자는 언제 든 지 자동 테스트 설정을 변경할 수 있습니다. 일반 향상 작업에 대 한 자동화 된 테스트를 해제 하거나 개별 작업에 대해이 기능을 설정할 수 있습니다. 아래 지침에 따라 자동화 된 테스트 설정을 변경 합니다.

#### <a name="to-manage-your-automated-testing-settings"></a>자동화 된 테스트 설정을 관리 하려면:

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)의 아무 곳에서 나 왼쪽 탐색 창의 **설정을** 선택 합니다.

2. 설정 페이지에서 **준수 관리자** 를 선택 합니다.

3. 왼쪽 탐색 창에서 **자동화 된 테스트** 를 선택 합니다.

4. 해당 하는 단추를 선택 하 여 모든 개선 작업에 대 한 자동 테스트를 설정 하거나, 모든 작업에 대해 기능을 해제 하거나, 개별 작업을 설정 합니다.

5. **각 업그레이드 작업에 대해 사용** 을 선택 하면 목록에는 선택할 수 있는 사용 가능한 향상 작업이 모두 표시 됩니다.  자동으로 테스트할 작업 옆의 상자를 선택 합니다.

6. **저장** 을 선택 하 여 설정을 저장 합니다. 화면 맨 위에 선택 영역이 저장 되었음을 알리는 확인 메시지가 표시 됩니다. 오류 알림이 표시 되 면 다시 시도 합니다.

**참고:** 전역 관리자만 모든 작업에 대해 자동 업데이트를 설정 하거나 해제할 수 있습니다. 준수 관리자 관리자는 개별 작업에 대해 자동 업데이트를 설정할 수 있지만 모든 작업에 대해 전역적으로 수행 되지 않습니다.

### <a name="manage-user-history"></a>사용자 기록 관리

**사용자 기록 관리** 설정은 준수 관리자에서 개선 작업을 수행한 사용자를 빠르게 식별 하는 데 도움이 됩니다. 개선 작업에 연결 된 식별 가능한 사용자 데이터에는 구현 및 테스트 중인 작업, 업로드 한 문서, 입력 한 메모 등이 포함 됩니다. 조직의 규정 준수 요구에 따라 이러한 유형의 데이터를 이해 하 고 검색 해야 할 수 있습니다.

사용자 기록 설정을 사용 하면 사용자 간에 모든 개선 작업을 다시 할당할 수도 있습니다.

**사용자 기록 설정을 찾으려면**

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)의 아무 곳에서 나 왼쪽 탐색 창의 설정을 선택 합니다.

2. 설정 페이지에서 **준수 관리자** 를 선택 합니다.

3. 왼쪽 탐색 창에서 **사용자 기록 관리** 를 선택 합니다.

**사용자 기록 관리** 페이지에는 개선 작업에 할당 된 모든 사용자의 전자 메일 주소에 대 한 목록이 표시 됩니다. **검색** 단추를 사용 하 여 전자 메일 주소를 입력 하 여 특정 사용자를 빠르게 찾을 수 있습니다.

각 사용자의 전자 메일 주소 오른쪽에 있는 **선택** 드롭다운 메뉴에서 보고서 내보내기, 개선 작업 다시 할당 또는 삭제 기록 옵션을 제공 합니다. 각 옵션에 대 한 자세한 내용은 아래 각 섹션을 참조 하십시오.

#### <a name="export-a-report-of-user-history-data"></a>사용자 기록 데이터 보고서 내보내기

현재 사용자에 게 할당 된 개선 작업 목록을 포함 하는 Excel 파일을 내보낼 수 있습니다.  또한이 보고서에는 해당 사용자가 업로드 한 모든 증거 파일이 나열 됩니다. 이 정보는 오픈 개선 작업을 다시 할당 하는 데 도움이 될 수 있습니다.

보고서에는 개선 작업의 상태를 만든 날짜에 반영 합니다. 해당 상태나 할당에 대 한 이전의 모든 변경 내용에 대 한 기록 보고서가 아닙니다 ( [개선 작업 페이지에서 보고서를 내보내는](compliance-manager-improvement-actions.md#export-a-report)방법 알아보기).

**다음 단계에 따라 사용자가 보고서를 내보냅니다.**

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)의 아무 곳에서 나 왼쪽 탐색 창의 **설정을** 선택 합니다.

2. 설정 페이지에서 **준수 관리자** 를 선택 합니다.

3. 왼쪽 탐색 창에서 **사용자 기록 관리** 를 선택 합니다.

4. 목록 전자 메일 주소를 검색 하거나 **검색** 을 선택 하 고 사용자의 전자 메일 주소를 입력 하 여 원하는 사용자를 찾습니다.

5. **선택** 드롭다운 메뉴에서 **보고서 내보내기를** 선택 합니다.

6. 보고서의 Excel 파일이 생성 되 면 연 다음 로컬 컴퓨터에 저장할 수 있습니다.

#### <a name="reassign-improvement-actions-to-another-user"></a>다른 사용자에 게 개선 작업 다시 할당

사용자 한 명에 게 향상 작업을 다시 할당할 수 있습니다. 작업을 다시 할당 하면 문서 업로드 기록도 변경 되지 않지만 해당 문서를 처음 업로드 한 사용자의 이름이 향상 된 작업에 더 이상 표시 되지 않습니다.

**다른 사용자에 게 개선 작업을 다시 할당 하려면 다음 단계를 수행 합니다.**

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)의 아무 곳에서 나 왼쪽 탐색 창의 **설정을** 선택 합니다.

2. 설정 페이지에서 **준수 관리자** 를 선택 합니다.

3. 왼쪽 탐색 창에서 **사용자 기록 관리** 를 선택 합니다.

4. 목록 전자 메일 주소를 검색 하거나 **검색** 을 선택 하 고 해당 사용자의 전자 메일 주소를 입력 하 여 사용자를 찾습니다.

5. **선택** 드롭다운 메뉴에서 **개선 작업 다시 할당** 을 선택 합니다. **개선 작업 다시 할당** 플라이 아웃 창이 표시 됩니다.

6. **사용자 검색** 필드에 향상 작업을 할당할 사용자의 이름이 나 전자 메일 *주소를 입력* 합니다.

7. **개선 동작이 할당** 되는 사용자의 이름이 표시 되 면 해당 사용자를 선택 하 고 **작업 할당** 을 선택 합니다.

8. 재배정이 완료 되 면 플라이 아웃 창에 이전 사용자의 모든 개선 작업이 새 사용자에 게 다시 할당 되었는지 확인 하는 확인 메시지가 표시 됩니다. 다시 할당 하는 오류 메시지가 표시 되 면 해당 창을 닫고 나 서 시도 합니다. 플라이 아웃 창을 닫으려면 **완료** 를 선택 합니다.

새 담당자는 개선 작업에 할당 된 전자 메일을 받습니다. 이 전자 메일에는 개선 작업의 세부 정보 페이지에 대 한 직접 링크가 포함 되어 있습니다.
 
 > [!NOTE]
> 보류 중인 업데이트가 있는 작업을 다시 할당 하는 경우 재할당 후 업데이트가 수락 되 면 재배정 전자 메일의 해당 작업에 대 한 직접 링크가 중단 됩니다. 업데이트를 수락한 후 사용자에 게 작업을 다시 할당 하 여이 문제를 해결할 수 있습니다. [향상 작업 업데이트](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)에 대해 더 자세히 알아보세요.

#### <a name="delete-user-history"></a>사용자 기록 삭제

사용자의 기록을 삭제 하면 개선 작업의 소유자로 제거 되며 준수 관리자의 다른 모든 필드에서 해당 이름이 제거 됩니다. 사용자의 기록을 삭제할 경우 소유한 향상 작업에 새 사용자가 할당 될 때까지 값이 **할당** 되지 않습니다. 개선 작업으로 업로드 된 문서에는 삭제 된 사용자 이름 대신 **사용자가 제거** 됨이 표시 됩니다. 사용자 기록을 삭제 하는 것은 영구적입니다.

사용자 기록을 삭제 하려면 다음 단계를 수행 합니다.

1. [Microsoft 365 준수 센터](https://compliance.microsoft.com/)의 아무 곳에서 나 왼쪽 탐색 창의 **설정을** 선택 합니다.

2. 설정 페이지에서 **준수 관리자** 를 선택 합니다.

3. 왼쪽 탐색 창에서 **사용자 기록 관리** 를 선택 합니다.

4. 목록 전자 메일 주소를 검색 하거나 **검색** 을 선택 하 고 해당 사용자의 전자 메일 주소를 입력 하 여 사용자를 찾습니다.

5. **선택** 드롭다운 메뉴에서 **기록 삭제** 를 선택 합니다.

6. 사용자의 기록을 영구적으로 삭제할지 묻는 창이 나타납니다. 삭제를 계속 하려면 **기록 삭제** 를 선택 합니다. 기록을 삭제 하지 않고 나가려면 **Cancel (취소** )을 선택 합니다.

7. 사용자 **기록을 관리** 하는 페이지의 맨 위에 확인 메시지가 표시 된 상태로 사용자의 변경 내용 관리가 다시 도착 합니다.

## <a name="understand-the-compliance-manager-dashboard"></a>준수 관리자 대시보드 이해

준수 관리자 대시보드는 현재 준수 상태를 한눈에 파악할 수 있도록 설계 되었습니다.

![준수 관리자-대시보드](../media/compliance-manager-dashboard.png "준수 관리자 대시보드")

### <a name="overall-compliance-score"></a>전반적인 준수 점수

준수 점수는 맨 위에 있는 아주 명확 하 게 제공 됩니다. 주요 데이터 보호 표준 및 규정을 해결 하는 향상 작업을 완료 하는 데 필요한 점수를 기반으로 하는 백분율을 보여 줍니다. 내 Microsoft에서 관리 되는 [microsoft 작업](compliance-manager-assessments.md#microsoft-actions-tab)의 점수 또한 준수 점수를 향해 계산 됩니다.

처음으로 준수 관리자가 제공 되는 경우 초기 점수는 [Microsoft 365 데이터 보호 기준을](compliance-manager-assessments.md#data-protection-baseline-default-assessment)기반으로 합니다. 모든 조직에서 사용할 수 있는이 기본 평가는 일반적인 업계 규정 및 표준을 포함 하는 컨트롤 집합입니다. 준수 관리자는 기존 Microsoft 365 솔루션을 검색 하 고 현재 개인 정보 및 보안 설정에 따라 초기 평가를 제공 합니다. 조직과 관련 된 평가를 추가 하면 점수가 보다 중요 하 게 드러납니다.

**자세한 정보:** [준수 점수가 계산 되는 방식을 이해](compliance-score-calculation.md)합니다.

### <a name="key-improvement-actions"></a>주요 개선 작업

이 섹션에서는 전체 준수 점수에 가장 큰 긍정적인 영향을 줄 수 있는 주요 향상 작업을 소개 합니다. **모든 개선 작업 보기** 를 선택 하 여 개선 작업 페이지로 이동 합니다.

### <a name="solutions-that-affect-your-score"></a>점수에 영향을 주는 솔루션

이 섹션에서는 점수에 긍정적인 영향을 줄 수 있는 개선 작업 및 해당 솔루션에서 해결 되지 않은 향상 작업의 수를 보여 주는 솔루션을 소개 합니다. 솔루션 페이지를 방문 하려면 **모든 솔루션 보기** 를 선택 합니다.

### <a name="compliance-score-breakdown"></a>규정 준수 점수 분석

이 섹션에서는 다음과 같은 두 가지 방법으로 점수에 대 한 자세한 내용을 확인할 수 있습니다.

- **범주** : "정보 보호" 또는 "장치 관리"와 같은 데이터 보호 범주 내의 전체 점수 비율을 표시 합니다.
- **평가** : GDPR 또는 NIST 800-53와 같은 특정 준수 및 데이터 보호 표준, 규정 또는 법규에 대 한 평가 관리에서 진행률을 관리할 비율을 보여 줍니다.

### <a name="filtering-your-dashboard-view"></a>대시보드 보기 필터링

대시보드 보기를 필터링 하 여 특정 규정과 표준, 솔루션, 작업 유형, 평가 그룹 또는 데이터 보호 범주와 관련 된 항목만 볼 수 있습니다. 이 방식으로 보기를 필터링 하면 대시보드의 점수가 필터링 되며, 필터 조건을 기준으로 하 여 총 가능한 점수를 얻은 점의 수를 보여 줍니다.

필터를 적용 하려면

1. 대시보드의 오른쪽 위에 있는 **필터** 를 선택 합니다.
2. **필터** 플라이 아웃 창에서 필터 조건을 선택한 다음 **적용** 을 선택 합니다.

필터를 적용 한 후 점수가 실시간으로 조정 되는 것을 볼 수 있습니다. 준수 점수 비율 및 분석 정보 및 개선 작업 및 솔루션은 필터 기준에 포함 된 데이터에만 영향을 받습니다. 준수 관리자에서 로그 아웃 하면 다시 로그인 할 때 필터링 된 보기가 유지 됩니다.

필터를 제거 하려면

- 준수 점수 위의 **적용 된 필터** 제목에서 제거할 개별 필터 옆에 있는 **X** 를 선택 합니다. 사용자나
- 대시보드의 오른쪽 위에 있는 **필터** 를 선택 하 고 **필터** 플라이 아웃 창에서 **필터 지우기를** 선택 합니다.

## <a name="improvement-actions-page"></a>개선 작업 페이지

[향상 작업](compliance-manager-improvement-actions.md) 은 조직에서 관리 하는 작업입니다. 향상 작업을 사용 하면 규정 준수 작업을 중앙 집중화 하 고 데이터 보호 규정 및 표준에 맞게 조정할 수 있습니다. 각 향상 작업에서는 적절 한 솔루션을 시작 하기 위한 링크 및 자세한 구현 지침을 제공 합니다. 조직의 사용자에 게 향상 작업을 할당 하 여 구현 및 테스트 작업을 수행할 수 있습니다. 개선 작업 내에 설명서, 메모 및 레코드 상태 업데이트를 저장할 수도 있습니다.

### <a name="view-your-improvement-actions"></a>개선 작업 보기

준수 관리자 대시보드에 **주요 향상 작업이 표시 됩니다.** 모든 개선 작업을 보려면 대시보드에서 개선 작업 탭을 선택 하 여 개선 작업 페이지로 가져옵니다. 대시보드의 주요 향상 작업 목록 아래에 있는 모든 개선 작업 보기를 선택 하 여 개선 작업 페이지로 이동할 수도 있습니다.

향상 작업 페이지에는 조직에서 관리 하는 모든 개선 작업이 표시 됩니다. Microsoft에서 관리 하는 작업은 각 평가 내에서 볼 수 있습니다 ( [microsoft 작업](compliance-manager-assessments.md#microsoft-actions-tab)에 대 한 자세한 정보).

개선 작업 페이지에 긴 작업 목록이 있으면 보기를 필터링 하는 것이 유용할 수 있습니다. 작업 목록의 오른쪽 위 모서리에서 **필터** 를 선택 합니다. **필터** 플라이 아웃 창이 나타나면 규정 및 표준, 솔루션, 그룹에 따라 기준을 선택 합니다. 오른쪽 위 모서리에 있는 **그룹** 을 선택 하 여 보기를 사용자 지정할 수도 있습니다. 드롭다운 메뉴에서 그룹, 솔루션, 범주, 작업 유형 또는 상태별로 보기를 선택 합니다.

이 페이지의 기본 보기에는 **통과** 테스트 상태의 개선 작업이 표시 되지 않습니다. 테스트에 통과 한 작업을 보려면 필터 플라이 아웃 창에서 **통과** 상자를 선택 합니다. 테스트 상태의 **합격** 한 작업만 점수에 대해 계산 됩니다. 일부 작업에는 **보류 중인 업데이트 레이블이** 표시 될 수 있습니다. [향상 작업 업데이트](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)에 대해 더 자세히 알아보세요.

향상 작업 페이지에는 각 향상 작업에 대 한 다음과 같은 데이터 요소가 표시 됩니다.

- **획득 한 점수** : 작업을 완료 하 여 사용할 수 있는 합계에서 얻은 점수 수
- **규정** : 작업과 관련 된 규정 또는 표준
- **그룹** : 작업을 할당 한 그룹입니다.
- **해결** 방법: 작업을 수행 하기 위해 이동할 수 있는 솔루션
- **평가** : 작업을 포함 하는 평가
- **범주** : 관련 된 데이터 보호 범주 (즉, 정보 보호, 장치 관리 등)
- **테스트 상태** :
    - **없음** – 상태 업데이트가 기록 되지 않음
    - **평가 안** 됨-테스트가 시작 되지 않음
    - **성공** -구현이 테스트 됨
    - **실패 한 낮은 위험** -테스트 실패, 낮은 위험
    - **실패 한 중간 위험** -테스트 실패, 보통 위험
    - **실패 한 높은 위험** -테스트 실패, 높은 위험
    - **범위를 벗어남** -작업이 평가 범위에 없으며 점수에 영향을 주지 않습니다.
    - **검색** 됨-수동 테스트의 경우 작업이 구현 되었지만 테스트 되지 않았음을 나타냅니다. 자동화 된 테스트의 경우 자동화 결과를 기다리는 작업이 실행 됨을 나타냅니다.
    - **검색할** 수 없음-자동화 상태를 확인할 수 없음
    - **부분적으로 테스트** 됨-부분 점수가 수상 인 자동 점수 매기기

**자세한** [내용은 개선 작업을 할당 하 고 작업을 수행 하는 방법을 참조](compliance-manager-improvement-actions.md)하세요.

## <a name="solutions-page"></a>솔루션 페이지

솔루션 페이지에는 솔루션 별로 구성 된 획득 및 잠재 점수 공유가 표시 됩니다. 이 보기에서 남은 점과 향상 작업을 보면 즉각적인 주의가 필요한 솔루션을 파악할 수 있습니다.

준수 관리자 대시보드에서 **솔루션** 탭을 선택 하 여 솔루션 페이지를 찾습니다. 대시보드의 오른쪽 위 섹션에서 **점수에 영향을 주는 솔루션** 아래의 **모든 솔루션 보기** 를 선택할 수도 있습니다.

### <a name="filtering-your-solutions-view"></a>솔루션 보기 필터링

솔루션 보기를 필터링 하려면 다음을 수행 합니다.

1. 평가 목록의 왼쪽 위 모서리에서 **필터** 를 선택 합니다.
2. **필터** 플라이 아웃 창에서 원하는 조건 (표준 및 규정, 솔루션, 작업 유형, 준수 관리자 그룹, 범주) 옆에 확인을 배치 합니다.
3. **적용** 단추를 선택 합니다. 필터 창이 닫히고 필터링 된 보기가 표시 됩니다.

평가 목록 위의 **그룹** 드롭다운 메뉴에서 그룹화 유형을 선택 하 여 그룹, 제품 또는 규정 별로 평가를 확인 하도록 보기를 수정할 수도 있습니다.

### <a name="taking-action-from-the-solution-page"></a>솔루션 페이지에서 작업 수행

솔루션 페이지에는 개선 작업에 연결 된 조직의 솔루션이 표시 됩니다. 이 표에서는 각 솔루션의 전체 점수, 해당 솔루션 내의 점수 및 가능한 점수를 나열 하 고 해당 솔루션에 그룹화 되어 점수가 증가할 수 있는 남은 개선 작업 수를 보여 줍니다.

다음 두 가지 방법으로이 화면에서 작업을 수행할 수 있습니다.

1. 원하는 솔루션의 행에서 **남은 작업** 열 아래에서 하이퍼링크 된 번호를 선택 합니다. 해당 솔루션에 대 한 테스트 되지 않은 개선 작업을 보여 주는 개선 작업 화면의 필터링 된 보기가 표시 됩니다.

2. 원하는 솔루션의 행에서 **솔루션 열기** 열 아래에 있는 **열기** 를 선택 합니다. Microsoft 365 및 Office 365 보안 및 준수 센터에서 권장 조치를 취할 수 있는 솔루션 또는 위치가 표시 됩니다.

## <a name="assessments-page"></a>평가 페이지

평가 페이지에는 조직에 대해 설정한 모든 [평가가](compliance-manager-assessments.md) 나열 됩니다. 준수 점수 분모는 추적 된 모든 평가에 따라 결정 됩니다. 평가를 더 추가 하면 개선 작업 페이지에 나열 된 추가 개선 작업이 표시 되 고 준수 점수 분모가 증가 합니다.

평가 페이지에서는 각 평가에 대 한 주요 정보를 요약 합니다.

- **평가** : 평가의 이름입니다.
- **상태** :
    - **전체** -모든 컨트롤의 상태가 "통과" 이거나, 하나 이상 통과 되며 나머지는 "범위를 벗어남"입니다.
    - **불완전** 함-하나 이상의 컨트롤에 "failed" 상태인 컨트롤이 있습니다.
    - **없음** -모든 컨트롤이 테스트 되지 않았습니다.
    - **진행** 중인 개선 작업에는 "진행 중", "부분 신용" 또는 "감지 되지 않음"을 비롯 한 다른 상태가 포함 됩니다.
- **평가 진행률** : 성공적으로 테스트를 거친 컨트롤 수로 측정 된 완료까지 작업의 완료율입니다.
- **향상 작업** : 컨트롤 구현을 충족 하기 위해 완료 된 작업의 수
- **Microsoft actions** : microsoft 컨트롤 구현을 충족 하기 위해 완료 된 작업 수
- **그룹** : 평가가 속한 그룹의 이름입니다.
- **제품** : 연결 된 Microsoft 365 서비스
- **규정** : 평가에 적용 되는 규정 표준, 정책 또는 법입니다.

### <a name="filtering-your-assessments-view"></a>평가 보기 필터링

평가 보기를 필터링 하려면:

1. 평가 목록의 왼쪽 위 모서리에서 **필터** 를 선택 합니다.
2. **필터** 플라이 아웃 창에서 원하는 기준을 확인 합니다.
3. **적용** 단추를 선택 합니다. 필터 창이 닫히고 필터링 된 보기가 표시 됩니다.

평가 목록 위의 **그룹** 드롭다운 메뉴에서 그룹화 유형을 선택 하 여 그룹, 제품 또는 규정 별로 평가를 확인 하도록 보기를 수정할 수도 있습니다.

### <a name="default-assessment"></a>기본 평가

기본적으로 평가 페이지에는 [데이터 보호 기준](compliance-manager-assessments.md#data-protection-baseline-default-assessment) 평가가 표시 됩니다. 또한 규정 준수 관리자는 평가를 빌드하기 위한 미리 작성 된 몇 가지 [템플릿도](compliance-manager-templates-list.md) 제공 합니다.

## <a name="assessment-templates-page"></a>평가 템플릿 페이지

템플릿은 준수 관리자에서 평가를 만드는 데 사용할 수 있는 프레임 워크입니다. 평가 템플릿 페이지에 서식 파일 및 주요 세부 정보 목록이 표시 됩니다. 이 목록에는 준수 관리자가 제공 하는 서식 파일과 조직이 수정 하거나 만든 서식 파일이 포함 되어 있습니다. 필터를 적용 하 여 인증, 제품 범위, 국가, 산업 및 만든 사람에 따라 서식 파일을 찾을 수 있습니다.

해당 행에서 서식 파일을 선택 하 여 서식 파일에 대 한 설명과 인증, 범위 및 컨트롤 세부 정보에 대 한 자세한 정보가 포함 된 세부 정보 페이지를 엽니다. 이 페이지에서는 적절 한 단추를 선택 하 여 평가를 만들거나 서식 파일 데이터를 Excel로 내보내거나 템플릿을 수정할 수 있습니다.

**자세한 내용은 방법:** [평가 템플릿을 사용 하는 방법 읽기를 참조](compliance-manager-templates.md)하세요.
