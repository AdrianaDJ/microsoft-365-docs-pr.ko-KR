# <a name="insider-risk-management-content-explorer"></a>참가자 위험 관리 콘텐츠 탐색기

Insider 위험성이 관리 콘텐츠 탐색기를 사용 하 여 **참가자 위험 관리 Investigators** 역할을 할당 받은 사용자는 경고에 캡처되는 통신의 컨텍스트 및 세부 정보를 확인할 수 있습니다. 모든 경고에 대해 데이터 및 메시지 파일의 복사본은 저장소 원본에 있는 원본 파일 및 메시지를 유지 하면서 항목의 시간에 스냅샷으로 보관 됩니다. 데이터와 메시지를 복사 하는 것은 알림과 연결 된 사용자와 콘텐츠의 소유자에 게 투명 합니다. 데이터에 대 한 사용 권한 설정 및 액세스 권한은 복사한 콘텐츠 및 메시지 및 위험 분석가에 대해 유지 관리 되며, 파일을 열고 확인 해야 하는 경우에는 이러한 사용 권한 및 investigators 필요 합니다. 각 파일 및 메시지에는 관리 목적을 위해 참가자 위험 관리의 경우 고유한 파일 ID가 자동으로 할당 됩니다. 장치 표시기 작업에 연결 된 문서는 콘텐츠 탐색기에 포함 되지 않습니다.

![참가자 위험 관리 콘텐츠 탐색기](../media/insider-risk-content-explorer.png)

## <a name="column-options"></a>열 옵션

위험 분석가와 investigators가 캡처한 데이터와 메시지를 보다 쉽게 검토 하 고 상황을 사례를 검토할 수 있도록 하기 위해 여러 필터링 및 정렬 도구를 콘텐츠 탐색기에 포함 합니다. 기본 정렬을 위해 **Date** 및 **File 클래스** 열은 콘텐츠 큐 창의 열 제목을 사용 하 여 정렬을 지원 합니다. 다른 큐 열을 보기에 추가 하 여 파일 및 메시지에 대 한 다양 한 피벗을 제공할 수 있습니다.

콘텐츠 큐의 열 머리글을 추가 하거나 제거 하려면 **열 편집** 컨트롤을 사용 하 고 다음 열 옵션을 선택 합니다. 이러한 열은 콘텐츠 탐색기에서 지원 되 고이 문서의 뒷부분에 나와 있는 일반, 전자 메일 및 문서 속성 조건에 매핑됩니다.

| **만든 이** | 문서를 복사하는 경우 유지되는 Office 문서의 만든 이 필드입니다. 예를 들어 사용자가 문서를 만들고이를 다른 사람에 게 전자 메일로 업로드 한 경우이 문서는 원래 작성자를 계속 유지 합니다. |

## <a name="advanced-search-conditions"></a>고급 검색 조건

검색 조건을 추가 하 여 검색 범위를 좁히고 보다 구체화 된 결과 집합을 반환할 수 있습니다. 각 조건은 검색을 시작할 때 생성 및 실행되는 검색 쿼리에 절을 추가합니다. 조건은 AND 연산자의 기능과 비슷한 논리 연산자 (c:c로 표시 됨)에 의해 키워드 상자에 지정 된 키워드 쿼리에 논리적으로 연결 됩니다. 즉, 항목은 검색 결과에 포함할 키워드 쿼리와 하나 이상의 조건을 만족 해야 합니다. 이 기능은 조건을 구체화 하는 데 도움이 되는 방식입니다.

고급 필터 및 검색 도구를 보려면 콘텐츠 큐 왼쪽의 **필터** 창을 확장 합니다. 조건 **추가** 단추를 선택 하 여 조건 목록을 엽니다.

### <a name="operators-used-with-conditions"></a>조건과 함께 사용되는 연산자

|**연산자**|**연산자와 동일한 쿼리**|**설명**|
|:-----------|:-------------------|:--------------|
| **마치면** |`property>date`| 날짜 조건과 함께 사용됩니다. 지정된 날짜 이후에 전송, 수신 또는 수정된 항목을 반환합니다.|
| **Before** |`property<date`| 날짜 조건과 함께 사용됩니다. 지정된 날짜 이전에 전송, 수신 또는 수정된 항목을 반환합니다.|
| **간의** |`date..date`| 날짜 및 크기 조건과 함께 사용됩니다. 날짜 조건과 함께 사용될 경우 지정된 날짜 범위 내에서 전송, 수신 또는 수정된 항목을 반환합니다. 크기 조건과 함께 사용될 경우 해당 크기가 지정된 범위 내에 속하는 항목을 반환합니다.|
| **모두 포함** |`(property:value) OR (property:value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정 된 문자열 값을 포함 하는 항목을 반환 합니다. |
| **Contains any of** |`(property:value) OR (property:value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정된 문자열 값의 일부를 포함하는 항목을 반환합니다.|
| **포함 하지 않음** |`-property:value`  <br/> `NOT property:value`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 지정된 문자열 값의 어떤 부분도 포함하지 않는 항목을 반환합니다.|
| **다음 중 하나에 해당 하지 않음** |`-property=value`  <br/> `NOT property=value`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 특정 문자열을 포함하지 않는 항목을 반환합니다.|
| **내년** |`size=value`| 지정 된 크기와 같은 항목을 반환 합니다. <sup>1</sup>|
| **Equals any of** |`(property=value) OR (property=value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정된 문자열 값과 정확히 일치하는 항목을 반환합니다.|
| **일치 하지 않음** |`(property=value) OR (property=value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정 된 문자열 값과 일치 하지 않는 항목을 반환 합니다. |
| **보다 큼** |`size>value`| 지정한 속성이 지정한 값 보다 큰 항목을 반환 합니다. <sup>1</sup>|
| **Greater or equal** |`size>=value`| 지정한 속성이 지정한 값 보다 크거나 같은 항목을 반환 합니다. <sup>1</sup>|
| **보다 작음** |`size<value`| 특정 값 보다 크거나 같은 항목을 반환 합니다. <sup>1</sup>|
| **Less or equal** |`size<=value`| 특정 값 보다 크거나 같은 항목을 반환 합니다. <sup>1</sup>|
| **Not equal** |`size<>value`| 지정 된 크기와 다른 항목을 반환 합니다. <sup>1</sup>|

> [!NOTE]
> <sup>1</sup> 이 연산자는 Size 속성을 사용 하는 조건에만 사용할 수 있습니다.

### <a name="email-property-conditions"></a>전자 메일 속성 조건

다음 표에는 콘텐츠 탐색기에서 사용할 수 있는 전자 메일 메시지 속성 조건이 나와 있습니다.

| **조건 옵션** | **설명** |

| **얼마나** | 보낸 사람이 메시지를 보낼 때 지정할 수 있는 전자 메일 메시지의 중요도입니다. 기본적으로 보낸 사람이 중요도를 **높음** 또는 **낮음**으로 설정하지 않았다면 메시지는 보통 중요도로 전송됩니다. |
| **모임 종료 날짜** | 모임의 모임 종료 날짜입니다. |
| **모임 시작 날짜** | 모임에 대 한 모임 시작 날짜입니다. |
| **메시지 종류** | 검색할 전자 메일 메시지의 유형입니다. 가능한 값: 연락처, 문서, 전자 메일, 외부 데이터, 팩스, im, 업무 일지, 모임, microsoft 팀의 채팅, 모임 및 통화에서 항목을 반환 하는 경우, notes, 게시물, rssfeeds, 작업, 음성 메일 |
| **참가자 도메인** | 메시지 참가자의 모든 도메인 목록입니다. |
| **할당** | 전자 메일 메시지의 모든 사람 필드입니다. 이 필드는 From, To, Cc 및 Bcc입니다. |
| **Received** | 받는 사람이 전자 메일 메시지를 받은 날짜입니다. |
| **받는 사람 도메인** | 메시지를 받는 사람의 모든 도메인 목록입니다. |
| **보낸 사람** | 메시지 유형 (보낸 사람) 필드입니다.  형식은 **DisplayName \<SmtpAddress> **입니다. |
| **보낸 사람 도메인** | 보낸 사람의 도메인입니다. |
| **제목** | 전자 메일 메시지 제목 줄의 텍스트입니다.  <br/> **참고:** 쿼리에서 Subject 속성을 사용 하면 검색에서 제목 줄에 검색 중인 텍스트가 포함 된 모든 메시지가 반환 됩니다. 즉, 쿼리는 정확 하 게 일치 하는 메시지만 반환 하지는 않습니다. 예를 들어를 검색 하는 경우 `subject:"Quarterly Financials"` 결과에는 제목이 "분기별 Financials 2018" 인 메시지가 포함 됩니다. |
| **To** | 전자 메일 메시지의 받는 사람 필드입니다. |
| **전자 메일 설정의 고유** | 전자 메일 집합에 첨부 파일이 중복 된 경우 False입니다. |

## <a name="document-property-conditions"></a>문서 속성 조건

다음 표에는 콘텐츠 탐색기에서 사용할 수 있는 문서 속성 조건이 나와 있습니다. 이러한 속성 조건의 대부분은 [고급 eDiscovery 사례](document-metadata-fields-in-Advanced-eDiscovery.md)에 포함 된 검토 집합과 공유 됩니다.

| **조건 옵션** | **설명** |
|:---------------------|:----------------|
| **변호사-클라이언트 권한 점수** | 변호사-클라이언트 권한 모델 콘텐츠 점수입니다. |
| **만든 이** | 문서를 복사하는 경우 유지되는 Office 문서의 만든 이 필드입니다. 예를 들어 사용자가 문서를 만들고이를 다른 사람에 게 전자 메일로 업로드 한 경우이 문서는 원래 작성자를 계속 유지 합니다. |
| **준수 레이블** | Office 365에 규정 준수 레이블이 적용 되었습니다. |
| **컴파운드 경로** | 항목의 원본을 설명 하는 사람이 읽을 수 있는 경로입니다. |
| **대화 ID** | 메시지의 대화 Id입니다. |
| **만든 시간** | 파일 또는 전자 메일 메시지를 만든 시간입니다. |
| **Custodian** | 항목이 연결 된 custodian의 이름입니다. |
| **주요 테마** | 분석을 위해 계산 되는 기준 테마입니다. |
| **가족 번호** | 가족 번호 모든 항목을 그룹화 합니다. 전자 메일의 경우이 필드에는 메시지와 모든 첨부 파일이 포함 됩니다. 문서의 경우이 필드에는 문서 및 포함 된 항목이 포함 됩니다. |
| **File 클래스** | SharePoint 및 OneDrive의 콘텐츠: **문서**; Exchange의 콘텐츠: * * 전자 메일 또는 **첨부 파일** |
| **파일 형식** | 파일의 확장명입니다. 예를 들면 .docx, one, .pptx 또는 .xlsx가 있습니다. |
| **변호사 참가자 있음** | 하나 이상의 참가자가 변호사 목록에 있으면 True이 고, 그렇지 않으면 false입니다. 그렇지 않은 경우이 값은 False입니다. |
| **변경할 수 없는 ID** | Office 365에 저장 된 변경 불가능 한 Id입니다. |
| **포함 유형** | 분석에 대해 계산 된 포괄 유형: **0** -포함 안 함 **1 개** (포함) **2 개** 포함 마이너스; **3** -포함 복사본 |
| **Item 클래스** | Exchange server에서 제공 하는 항목 클래스 예를 들어 **IPM. 참고 사항** |
| **마지막으로 수정한 날짜** | 문서를 마지막으로 변경한 날짜입니다. |
| **부하 ID** | 항목이 검토 집합에 로드 된 부하 Id입니다. |
| **위치 이름** | 항목의 원본을 식별 하는 문자열입니다.  Exchange의 경우이 필드는 사서함의 SMTP 주소입니다. SharePoint 및 OneDrive의 경우 사이트 모음의 URL입니다. |
| **담당자로 표시 됨** | 정확히 중복 된 각 집합의 한 문서를 담당자로 표시 합니다. |
| **네이티브 파일 확장명** | 항목의 네이티브 확장명입니다. |
| **기본 파일 이름** | 항목의 기본 파일 이름입니다. |
| **NdEtSortExclAttach** | 검토 시간을 효율적으로 정렬 하기 위해 전자 메일 설정 및 ND 집합을 연결 합니다. D는 ND 집합에 접두사로 추가 되 고 E는 전자 메일 집합에 추가 됩니다. |
| **피벗 ID** | 피벗의 ID입니다. |
| **잠재적으로 권한이 부여 된** | True 이면 변호사 권한 검색 모델이 문서에 잠재적으로 권한이 부여 된 것으로 간주 합니다. |
| **처리 상태** | 항목을 검토 집합에 추가한 후의 처리 상태 |
| **백분위 수** | 관련성에 따라 문서에 대 한 백분위 수를 읽습니다. |
| **관련성 점수** | 관련성을 기준으로 문서의 관련성 점수입니다. |
| **관련성 태그** | 관련성을 기준으로 문서의 관련성 점수입니다. |
| **대표 번호** | 정확히 중복 된 각 집합의 숫자 식별자입니다. |
| **태그** | 검토 집합에 적용 된 태그 |
| **테마 목록** | 분석을 위해 계산 되는 테마 목록 |
| **Title** | 문서의 제목입니다. Title 속성은 Office 문서에 지정된 메타데이터입니다. 문서의 파일 이름과는 다릅니다. |
| **재구성 됨** | 항목이 재구성 된 경우 True이 고 그렇지 않으면 False입니다. |
| **단어 개수** | 파일의 단어 수입니다. |
