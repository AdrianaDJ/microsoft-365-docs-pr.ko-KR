# <a name="understand-the-proposal-workflow"></a>제안 워크플로 이해

제안은 microsoft 제품 및 서비스를 구매할 수 있는 Microsoft의 조직에 대 한 공식적인 제안입니다. Microsoft 담당자와 직접 협력 하 여 제안에 대 한 특정 제품, 서비스 및 용어를 확인 합니다.

Microsoft 대표 초안은 사용자와 담당자가 설명한 항목을 포함 하는 제안입니다. 담당자가 사용자에 게 제안 사이트에 대 한 링크를 포함 하는 전자 메일을 보냅니다. 이 사이트에는 사용자와 조직에 적합 한 제안이 준비 되어 있습니다.

알림 전자 메일을 받은 후 제안 사이트에 대 한 링크를 따르세요. 사이트에 로그인 한 후 제안 검토 프로세스를 시작할 수 있습니다.

## <a name="prerequisites-for-buying-items-with-a-proposal"></a>제안서로 항목을 구입 하기 위한 필수 구성 요소

제안의 항목을 구입 하려면 먼저 대금 청구 계정 및 Microsoft와 계약을 받아야 합니다.

### <a name="billing-account"></a>청구 계정

청구 계정을 사용 하 여 계정 설정, 송장, 대금 청구 프로필, 제품 및 서비스를 관리 합니다. 아직 대금 청구 계정이 없는 경우 Microsoft 담당자가 계정을 만듭니다.
그렇지 않은 경우 해당 청구 계정을 사용할 수 있는 권한이 있는 경우 조직에 대 한 기존 대금 청구 계정을 사용 합니다.

청구 계정 사용 권한은 청구 계정 소유자가 관리 합니다.
전역 관리자는 자신을 청구 계정 소유자 역할에 할당 한 다음 다른 사용자의 청구 계정 소유자를 지정할 수 있습니다.

청구 계정에 대 한 자세한 내용은 [청구 계정 관리](manage-billing-accounts.md)를 참조 하세요.

### <a name="microsoft-customer-agreement"></a>Microsoft 고객 계약

Microsoft 고객 계약 (MCA)을 사용 하면 조직에서 Microsoft 제품 및 서비스를 구매할 수 있습니다. 자세한 내용은 [Microsoft 고객 계약](https://www.microsoft.com/en-us/Licensing/how-to-buy/microsoft-customer-agreement)을 참조 하십시오.

## <a name="permissions-needed-to-sign-an-agreement-or-pay-for-items"></a>계약에 서명 하거나 항목에 대 한 요금을 지불 하는 데 필요한 사용 권한

청구 계정에 할당 된 역할이 없으면 제안을 볼 때 기본 독자 역할이 할당 된 것입니다. 이 역할을 사용 하면 제안을 볼 수 있으 나 아무 조치도 취할 필요가 없습니다. 계약에 서명 하거나 제품 및 서비스를 구매할 수 있으려면 먼저 청구 계정 소유자 또는 청구 계정 참가자 역할을 할당 받아야 합니다. 청구 계정 소유자는이 역할을 사용자에 게 할당할 수 있습니다.

대금 청구 계정 역할에 대 한 자세한 내용은 [청구 계정에 대 한 액세스 이해](manage-billing-accounts.md#understand-access-to-billing-accounts)를 참조 하세요.

새 청구 계정인 경우 계약을 수락한 사용자가 없는 경우에는 다음과 같은 경우에 자동으로 청구 계정 소유자가 됩니다.

- 제안에서 이름이 지정 된 사람 \
    또는
- 이미 조직의 [Azure Active Directory 전역 관리자](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles) 인 경우

## <a name="what-is-the-overall-workflow"></a>전체 워크플로 란?

전체 제안 워크플로는 다음과 같습니다.

- Microsoft 담당자가 제안을 만들고 전자 메일로 사용자에 대 한 링크를 보냅니다.

- 링크를 사용 하 여 제안 로그인 페이지로 이동 합니다.

- 조직의 정보를 검토 합니다.

- 제안서를 검토 하 고 필요한 경우 MCA를 수락한 다음 체크아웃 프로세스를 완료 합니다.

    > [!IMPORTANT]
    > 조직을 대신 하 여 MCA에 서명할 수 있는 권한이 있어야 합니다. 해당 권한이 없는 경우이 단계를 수행 해야 하는 사용자가 됩니다.

- 체크 아웃이 완료 되 면 제품 및 서비스를 설정 하기 위한 추가 링크가 제공 됩니다.

## <a name="proposal-terms"></a>제안 용어

다음 표에는 제안 및 제안 사이트에 표시 되는 용어 및 정의가 나와 있습니다.

| **용어** | **정의** |
|------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 청구 계정 | 계정 설정, 송장, 지불 방법 및 제품을 관리 하는 데 사용 되는 계정입니다. |
| 청구 프로필 | 송장에 포함 되는 항목을 사용자 지정할 수 있도록 하는 조직에 대 한 정보 및 송장에 대해 지불 하는 방법입니다. 대금 청구 프로필에는 청구 계정 이름, 특정 대금 청구 프로필에 사용 되는 지불 방법, 연락처 정보, 송장 설정, 청구 프로필을 변경 하 고 금액을 지불 하 고 제품 및 서비스를 구매할 수 있는 권한이 포함 됩니다. |
| 기존 계약 | 조직이 이미 Microsoft와 함께 사용 하 고 있는 계약입니다. 여기에는 엔터프라이즈 계약, Microsoft 제품 & 서비스 계약 또는 Microsoft 고객 계약을 포함할 수 있습니다. |
| Microsoft 고객 계약 (MCA) | Microsoft와 조직에서 보유 한 계정의 사용 약관을 대략적으로 설명 하는 계약입니다. |
| Microsoft 담당자 | 사용자 및 조직에 대 한 제안서를 준비한 승인 된 Microsoft 담당자입니다. |
| 조직 | Microsoft 제품, 기술 또는 서비스를 사용 하는 법인 단체입니다. |
| 준비한 사람 | 제안을 준비한 Microsoft 담당자의 전자 메일 주소입니다. |
| 추가 용어 | 조직과 관련 된 용어를 포함 하는 MCA를 Amendments. 추가 용어를 수락 하려면 DocuSign을 사용 하 여 전자 서명을 녹음 해야 합니다. |

## <a name="step-1-review-organization-information"></a>1 단계: 조직 정보 검토

로그인 한 후에는 먼저 조직의 정보를 검토 합니다.

### <a name="your-organization"></a>조직

**조직** 섹션에는 해당 역할과 연결 된 청구 계정이 표시 됩니다. 청구 계정 정보는 기존 청구 계정에서 가져오거나 Microsoft 담당자가 직접 만듭니다. 조직이 다른 조직의 계열사 인 경우 해당 조직의 이름과 주소와 함께 **잠재 고객 조직** 섹션도 표시 됩니다.

조직에서 Microsoft와의 상업적 관계를 처음으로 설정 하는 경우, 아직 MCA에 서명 하지 않은 경우에는 **조직** 또는 **잠재 고객 조직** 에 있는 정보가 잘못 된 경우 담당자에 게 문의 하 여 변경 작업을 수행 합니다. MCA를 수락한 후 Microsoft 365 관리 센터의 [청구 계정](https://go.microsoft.com/fwlink/p/?linkid=2084771) 페이지에서 조직의 주소 및 연락처 정보를 검토 하 고 변경할 수 있습니다. 조직 이름이 변경 되 면 서비스 요청이 업데이트 되도록 요청을 엽니다. [서비스 요청을 여는 방법 알아보기](../admin/contact-support-for-business-products.md)

### <a name="your-information"></a>사용자 정보

새 고객 인 경우 **정보**에 사용자 이름, 전자 메일 주소 및 전화 번호를 입력 한 다음 **저장**을 선택 합니다. 기존 고객 인 경우 정보가 올바른지 확인 합니다. 수정 작업을 수행 하려면 **편집**을 선택 하 고 필요한 사항을 변경한 다음 **저장**을 선택 합니다.

준비가 되 면 **계속** 을 선택 하 여 다음 단계로 이동 합니다.

## <a name="step-2-review-proposal"></a>2 단계: 제안 검토

이 제안에는 Microsoft 담당자와 함께 설명 하는 항목에 대 한 세부 정보가 포함 됩니다. 전자 메일을 제안 링크와 함께 전달 하 여 조직의 다른 관련자와 공유할 수 있습니다. 이 링크를 사용 하는 다른 모든 사용자에 게는 제안에 대 한 읽기 전용 보기가 있습니다.

검토 후 제안을 변경 하려면 Microsoft 담당자에 게 문의 하세요.

### <a name="proposal-contents"></a>제안 내용

이 제안의 다음 정보를 포함 합니다.

| 단면 | 설명 |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 조직 이름 | 제안이 준비 된 조직의 이름입니다. |
| 유효 기간 만료 날짜 | 제안 제안이 만료 되는 날짜입니다. 이 만료 날짜를 놓친 경우 Microsoft 담당자에 게 문의 하 여 제안에 계속 관심이 있음을 알려야 합니다. |
| 통화 | 제안의 항목 비용을 계산 하는 데 사용 되는 통화입니다. |
| 준비 | 대금 청구 계정 이름, 주소, 연락처 전자 메일 주소 및 제안을 요청한 사용자의 전화 번호입니다. |
| 준비한 사람 | 제안을 준비한 Microsoft 담당자의 전자 메일 주소입니다. |
| 요약 | 제안서와 연관 된 부분합을 표시 합니다. 필요한 경우 비용을 계산 하는 데 사용 되는 f (외부 exchange) 속도도 표시 됩니다. |
| 제안 품목 | 이 섹션에는 제안서에 포함 된 모든 항목의 수량, 단가 및 부분합이 포함 되어 있습니다. |
| 다음 단계 | 이 섹션에서는 수행 해야 하는 필수 작업을 나타냅니다. |

MCA에 서명 하려면 **다음 단계**아래의 단추를 선택 합니다. 추가 용어에 서명 해야 하는 경우 링크를 통해 DocuSign 사이트로 이동 하 여 문서에 서명 하는 단계를 수행 합니다.

필요한 계약 또는 추가 약관에 서명 했으면 **체크아웃으로 이동을**선택 합니다.

## <a name="step-3-checkout"></a>3 단계: 체크 아웃

체크 아웃 페이지에는 다음 섹션이 포함 되어 있습니다.

### <a name="sold-to"></a>판매 대상

이 섹션에는 제안에 사용 된 청구 계정이 표시 됩니다. 정보를 변경 해야 하는 경우 **편집** 링크를 선택 합니다. **편집** 링크를 사용 하 여 조직의 세금 ID를 추가할 수도 있습니다. 세금 ID는 **판매** 섹션에 나열 된 국가와 연결 되어야 합니다. 세금 면제를 사용 하는 경우에는 지원 티켓을 열고 세금 면제 상태를 요청 해야 합니다.

세금 Id에 대 한 자세한 내용과 세금 면제 상태를 적용 하는 방법에 대 한 자세한 내용은 [세금 정보](billing-and-payments/tax-information.md)를 참조 하십시오.

### <a name="billed-to"></a>청구 대상

이 섹션에서는 송장에 포함 되는 항목을 결정 하는 데 사용 되는 청구 프로필 및 청구서 지불 방법을 보여 줍니다. 각 대금 청구 주기에서는 각 대금 청구 프로필에 대해 별도의 송장을 받게 됩니다. 확인 또는 전선 전송 또는 Azure prepayment을 사용 하 여 송장을 지불 합니다. 아직 대금 청구 프로필이 없는 경우 Microsoft 담당자가 해당 프로필을 만듭니다. 체크 아웃 하는 동안 다른 대금 청구 프로필을 선택 하거나, 대금 청구 프로필의 이름을 변경 하거나, 사서함을 추가할 수 있습니다. 수. 새 대금 청구 프로필을 만들 수도 있습니다.

대금 청구 프로필에 대 한 자세한 내용은 [청구 프로필 관리](billing-and-payments/manage-billing-profiles.md)를 참조 하세요.

### <a name="proposal-items-in-this-order"></a>다음 주문의 제안 항목

이 섹션에는 제안에 포함 된 모든 항목의 목록이 표시 됩니다. 이 목록에는 다음 범주 중 하나 이상이 포함 될 수 있습니다.

- **추가 용어** 조직에 대 한 용어를 포함 하는 모든 MCA에 대 한 amendments 목록 예를 들어이 목록에는 HIPAA 또는 GDPR 약관이 포함 될 수 있습니다.

- **지금 구입** 제안 승인 워크플로가 끝나면 체크 아웃 하는 동안 지불 하는 항목 목록입니다.

- **할인 (향후 요금에 적용)** 제안의 일부로 받은 할인 목록입니다.

- **포함** 추가 비용 없이 제안 패키지의 일부로 포함 되는 항목 목록입니다. 이러한 항목 중 일부에는 나중에 관련 비용이 소요 될 수 있습니다.

### <a name="summary"></a>요약

이 섹션에는 주문에 대 한 지불 되는 항목 수, 소계, 예상 세금 및 총 금액이 표시 됩니다.

주문을 하려면 **주문 배치** 또는 동의 **& 배치**를 선택 합니다.

주문을 한 후에는 다음 단계를 수행 해야 하는 확인 메시지가 표시 됩니다. Azure 요금제를 구매한 경우 다음 단계는 Azure portal에서 청구 계정을 설정 하는 것입니다.

## <a name="step-4-set-up-your-new-billing-account-azure-customers-only"></a>4 단계: 새 대금 청구 계정 설정 (Azure 고객만 해당)

신규 고객이 며 제안의 일부로 Azure 제품을 구매한 경우에는 새 대금 청구 계정을 설정 해야 합니다. 방법에 [대 한 자세한 내용은 Microsoft 고객 계약에 대 한 청구 계정 설정을](https://docs.microsoft.com/azure/cost-management-billing/manage/mca-setup-account)참조 하십시오.

기업 계약이 있는 기존 Azure 고객 인 경우 처음에 MCA에 서명 하는 경우 다음 단계에서는 계약 간의 변경 사항과 새 청구 계정을 사용 하 여 작업을 완료 하는 방법을 알아봅니다. 자세한 내용은 [Microsoft 고객 계약에 대 한 청구 계정의 전체 엔터프라이즈 계약 작업](https://docs.microsoft.com/azure/cost-management-billing/manage/mca-enterprise-operations)을 참조 하세요.

## <a name="understand-invoicing"></a>송장 이해

주문 내용을 확인 하 고 완료 한 후 초기 송장은 24-48 시간 이내에 전송 됩니다. 그 후에는 매월 5 일에 송장을 받게 됩니다. 월별 송장에는 이전 달의 금액이 포함 됩니다. 사용자 계정에 대 한 크레딧이 있는 경우 청구 프로필의 통화 제작진에서이를 deducted 송장 잔액에 적용 됩니다. 잔액이 적용 된 후 남은 잔액이 대차 때문입니다. 청구 날짜 로부터 30 일 이내에 송장을 지불 합니다.

확인 또는 전송 되는 위치에 대 한 결제 지침은 송장의 PDF 복사본에 포함 됩니다. 송장을 보거나 다운로드 하려면 [청구서 또는 송장 보기](billing-and-payments/view-your-bill-or-invoice.md)를 참조 하세요.
