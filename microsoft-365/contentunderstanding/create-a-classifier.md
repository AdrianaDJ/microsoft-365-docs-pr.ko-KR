# <a name="create-a-classifier-in-microsoft-sharepoint-syntex"></a>Microsoft SharePoint Syntex에서 분류자 만들기

이 문서에서 설명 하는 내용은 Project Cortex 비공개 미리 보기에 대 한 것입니다. [자세한 내용은 Project Cortex를 참조](https://aka.ms/projectcortex)하세요.

</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4CL0R]  

</br>

분류자는 문서 형식의 식별 및 분류를 자동화 하는 데 사용할 수 있는 모델 유형입니다. 예를 들어 다음 그림에 나와 있는 것과 같이 문서 라이브러리에 추가 된 모든 *계약 갱신* 문서를 확인할 수 있습니다.

![계약 갱신 문서](../media/content-understanding/contract-renewal.png)

분류자를 만들면 모델과 연결 될 새 [SharePoint 콘텐츠 형식을](https://docs.microsoft.com/sharepoint/governance/content-type-and-workflow-planning#content-type-overview) 만들 수 있습니다.

분류자를 만들 때 모델을 정의 하는 *설명을* 작성 해야 합니다. 이를 통해이 문서 유형을 일관 되 게 찾을 수 있는 일반적인 데이터를 확인할 수 있습니다. 

문서 형식의 예 ("example files")를 사용 하 여 모델을 "학습" 하 여 콘텐츠 형식이 같은 파일을 식별 합니다.

분류자를 만들려면 다음을 수행 해야 합니다.
1. 모델 이름을 모델링 합니다.
2. 예제 파일을 추가 합니다.
3. 예제 파일에 레이블을 지정 합니다.
4. 설명을 만듭니다.
5. 모델을 테스트 합니다.

> [!NOTE]
> 모델에서 분류자를 사용 하 여 문서 형식을 식별 하 고 분류 하는 동안 모델에 의해 식별 되는 각 파일에서 특정 정보를 가져오도록 선택할 수도 있습니다. 모델에 추가할 **추출기** 를 만들어이 작업을 수행 합니다. [추출기 만들기를](create-an-extractor.md)참조 하세요.

## <a name="name-your-model"></a>모델 이름

모델을 만드는 첫 번째 단계는 이름을 지정 하는 것입니다.

1. 콘텐츠 센터에서 **새로 만들기**를 선택 하 고 **모델을 만듭니다**.
2. **새 문서 이해 모델** 창의 **이름** 필드에 모델의 이름을 입력 합니다. 예를 들어 계약 갱신 문서를 확인 하려면 모델 *계약 갱신*이름을 지정 하면 됩니다.
3. **만들기**를 선택합니다. 이를 통해 모델에 대 한 홈 페이지가 만들어집니다.</br>

    ![분류자 모델 홈 페이지](../media/content-understanding/model-home.png)

모델을 만들 때 새 SharePoint 콘텐츠 형식도 만듭니다. SharePoint 콘텐츠 형식은 공통 특성을 가지 며 특정 콘텐츠에 대 한 열 또는 메타 데이터 속성의 컬렉션을 공유 하는 문서 범주를 나타냅니다. SharePoint 콘텐츠 형식은 [콘텐츠 형식 갤러리](https://support.microsoft.com/office/create-or-customize-a-site-content-type-27eb6551-9867-4201-a819-620c5658a60f)를 통해 관리 됩니다. 이 예에서는 모델을 만들 때 새 *계약 갱신* 콘텐츠 형식을 만듭니다.

이 모델을 SharePoint 콘텐츠 형식 갤러리의 기존 콘텐츠 형식에 매핑하려면 해당 스키마를 사용 하려면 **고급 설정을** 선택 합니다. 기존 콘텐츠 형식을 사용 하 여 해당 스키마를 통해 식별 및 분류에 도움이 되는 경우에도 식별 되는 파일에서 정보를 추출 하는 모델을 학습 해야 합니다.</br>

![고급 설정](../media/content-understanding/advanced-settings.png)

## <a name="add-your-example-files"></a>예제 파일 추가

모델 홈 페이지에서 모델을 학습 하 여 문서 형식을 식별 하는 데 필요한 예제 파일을 추가 합니다. </br>
</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4D0iX] 

</br>

> [!NOTE]
> 분류자 및 [추출기 학습](create-an-extractor.md)에 동일한 파일을 사용 해야 합니다. 나중에 추가할 수 있지만 일반적으로는 전체 예제 파일 집합을 추가 하는 옵션을 사용할 수 있습니다. 레이블을 지정 하 여 모델을 학습 하 고, 레이블이 없는 나머지 항목을 테스트 하 여 모델 적합성을 평가 합니다. 

교육 집합의 경우 다음과 같은 긍정 및 부정 예를 모두 사용 하려고 합니다.
- 긍정적인 예: 문서 유형을 나타내는 문서입니다. 여기에는 해당 문서 유형에 서 항상 포함 되는 문자열 및 정보가 들어 있습니다.
- 부정적 예: 문서 형식을 나타내지 않는 문서입니다. 이러한 유형의 문서에는 누락 된 문자열 및 정보가 표시 됩니다.

모델을 학습 하려면 최소한 5 개의 긍정 예와 하나 이상의 부정적 예제를 사용 해야 합니다.  교육 프로세스 후에 모델을 테스트 하기 위한 추가 작업을 만들려고 합니다.

샘플 파일을 추가 하려면:

1. 모델 홈 페이지의 **빌드 예제 라이브러리** 타일에서 **파일 추가**를 클릭 합니다.
2. **모델의 샘플 파일 선택** 페이지의 콘텐츠 센터에 있는 샘플 파일 라이브러리에서 예제 파일을 선택 합니다. 아직 업로드 하지 않은 경우 **업로드** 를 클릭 하 여 샘플 파일 라이브러리로 이동 하 여 업로드 하도록 선택 합니다.
3. 모델을 학습 하는 데 사용할 예제 파일을 선택한 후 **추가**를 클릭 합니다.

    ![예제 파일 선택](../media/content-understanding/select-sample.png) 

## <a name="label-your-example-files"></a>예제 파일에 레이블 지정

예제 파일을 추가한 후에는 양수 또는 음수 예제 중 하나로 레이블을 지정 해야 합니다.

1. 모델 홈 페이지의 **파일 분류 및 교육 실행** 타일에서 **분류자 트레인**을 클릭 합니다.
   이렇게 하면 첫 번째 파일이 보기에 표시 되는 예제 파일 목록을 보여 주는 레이블 페이지가 표시 됩니다.
2. 첫 번째 예제 파일의 맨 위에 있는 뷰어에서 파일이 방금 만든 모델의 예 인지 묻는 텍스트가 표시 됩니다. 긍정적인 예 이면 **예**를 선택 합니다. 음수 예 이면 **아니요**를 선택 합니다.
3. 왼쪽의 **레이블이 표시 된 예제** 목록에서 예제로 사용 하려는 추가 파일을 선택 하 고 레이블을 지정 합니다. 

    ![분류자 홈 페이지](../media/content-understanding/classifier-home-page.png) 


> [!NOTE]
> 레이블에는 5 개 이상의 양수 예제와 하나의 음수 예제가 나와 있습니다. 

## <a name="create-an-explanation"></a>설명 만들기

다음 단계에서는 열차 페이지에 대 한 설명을 작성 합니다. 설명은 모델에서 문서를 인식 하는 방법을 이해 하는 데 도움이 됩니다. 예를 들어 계약 갱신 문서에는 *추가 공개 텍스트 문자열에 대 한 요청이* 항상 포함 됩니다.

> [!Note]
> 추출기와 함께 사용 하는 경우 문서에서 추출할 문자열이 설명에 식별 됩니다. 

설명을 만들려면 다음을 실행 합니다.

1. 모델 홈 페이지에서 **열차** 탭을 선택 하 여 열차 페이지로 이동 합니다.
2. 학습 페이지의 **숙련 된 파일** 섹션에 이전에 표시 한 예제 파일 목록이 표시 됩니다. 목록에서 양수 파일 중 하나를 선택 하면 뷰어에 표시 됩니다.
3. 설명 섹션에서 **새로 만들기** 를 선택 하 고 **비어**있습니다.
4. **설명 만들기** 페이지에서 다음을 수행 합니다.</br>
    a. **이름** (예: "공개 블록")을 입력 합니다.</br>
    b. **유형을**선택 합니다. 예제에서 텍스트 문자열을 추가 하므로 **구 목록을**선택 합니다.</br>
    c. 여기에 **입력** 상자에 문자열을 입력 합니다. 예제를 보려면 "추가 공개 요청"을 추가 합니다. 문자열에서 대/소문자를 구분 해야 하는 경우 **대/소문자를 구분** 하도록 선택할 수 있습니다.</br>
    d. **저장**을 클릭합니다.

    ![설명 만들기](../media/content-understanding/explanation.png) 
    
 
5. 이제 모델에서는 작성 한 설명이 긍정적이 고 부정적 예제와 같이 레이블이 지정 된 나머지 예제 파일을 정확 하 게 식별할 수 있는지 확인 합니다. 숙련 된 파일 섹션에서 학습을 완료 한 후 **평가** 열을 확인 하 여 결과를 확인 합니다. 작성 한 설명이 양수나 음수로 표시 된 것과 일치 하기에 충분 한 경우 파일은 **match**값을 나타냅니다.

    ![Match 값](../media/content-understanding/match.png) 

레이블이 지정 된 파일이 **일치 하지 않는** 경우에는 추가 설명을 만들어 문서 형식을 식별 하는 데 필요한 자세한 정보를 모델에 제공 해야 할 수 있습니다. 이 경우 파일을 클릭 하 여 불일치가 발생 한 이유에 대 한 자세한 정보를 확인 하세요.

## <a name="test-your-model"></a>모델 테스트

레이블이 지정 된 샘플 파일에 대해 일치 하는 항목을 받은 경우에는 레이블 없는 나머지 예제 파일에서 모델을 테스트할 수 있습니다.

1. 모델 홈 페이지에서 **테스트** 탭을 선택 합니다.  이렇게 하면 레이블이 지정 되지 않은 샘플 파일에서 모델이 실행 됩니다.
2. **테스트 파일** 목록에서 예제 파일이 표시 되 고 모델에서 해당 파일을 양수나 음수로 예측 하는지 여부를 보여 줍니다. 이 정보를 사용 하 여 문서를 식별 하는 분류자의 유효성을 확인할 수 있습니다.

    ![레이블이 지정 되지 않은 파일 테스트](../media/content-understanding/test-on-files.png)
