# <a name="introduction-to-explanation-types"></a>설명 유형 소개

설명을 사용 하 여 레이블을 지정할 정보를 정의 하 고 문서에서 추출 하는 데 도움이 되는 Microsoft SharePoint Syntex 모델을 파악 합니다. 설명을 만들 때 설명 유형을 선택 해야 합니다. 

이 문서는 다양 한 설명 유형 및 사용 방법을 이해 하는 데 도움이 됩니다.

   ![설명 유형](../media/content-understanding/explanation-types.png) 
   
이러한 설명 유형을 사용할 수 있습니다.

- **구 목록**: 추출 하려는 문서나 정보에 사용할 수 있는 단어, 구, 숫자 또는 기타 문자에 대 한 목록입니다. 예를 들어 **의사를 참조** 하는 텍스트 문자열은 식별 하려는 모든 의료 리퍼럴 문서에 있습니다.</br>

- **패턴 목록**: 추출할 정보를 식별 하는 데 사용할 수 있는 숫자, 문자 또는 기타 문자의 패턴을 나열 합니다. 예를 들어 식별 중인 모든 의료 리퍼럴 문서에서 참조 하는 의사 결정의 **전화 번호** 를 추출할 수 있습니다.</br>

- **근접성**: 서로 근접 한 설명을 어떻게 들을 것인지 설명 합니다. 예를 들어 *전화 번호* 패턴 목록은 다음 사이에 토큰 없이 (이 문서의 뒷부분에 나오는 토큰에 대해 자세히 알아봅니다.) *주소* 구문 목록 바로 앞에 옵니다. 
 
## <a name="phrase-list"></a>구 목록

구 목록 설명 유형은 일반적으로 모델을 통해 문서를 식별 하 고 분류 하는 데 사용 됩니다. *참조 의사* 레이블 예제에 설명 된 것 처럼 식별 하려는 문서에서 일관 되 게 사용할 수 있는 단어, 구, 숫자 또는 문자 문자열입니다.

요구 사항은 아니지만 캡처가 있는 구가 문서에서 일관 된 위치에 있는 경우에는 설명에 따라 성공을 높일 수 있습니다. 예를 들어 참조 하는 *의사* 레이블은 문서의 첫 번째 단락에 일관 되 게 배치 될 수 있습니다.

레이블을 식별 하는 데 대/소문자 구분이 요구 되는 경우 구 목록 형식을 사용 하면 **정확한 대문자화** 확인란을 선택 하 여 설명에 지정할 수 있습니다.

   ![대/소문자 구분](../media/content-understanding/case-sensitivity.png) 

## <a name="pattern-lists"></a>패턴 목록

패턴 목록 유형은 특히 문서에서 정보를 식별 하 고 추출 하는 설명을 작성할 때 유용 합니다. 일반적으로 날짜, 전화 번호 또는 신용 카드 번호와 같은 다양 한 형식으로 표시 됩니다. 예를 들어 날짜는 여러 가지 형식으로 표시할 수 있습니다 (1/1/2020, 1-1-2020, 01/01/20, 01/01/2020, Jan 1, 2020 등). 패턴 목록을 정의 하면 식별 하 고 추출할 데이터에서 가능한 변형을 캡처하여 보다 효율적으로 설명을 사용할 수 있습니다. 

**전화 번호** 예제에서 모델이 식별 하는 모든 의료 리퍼럴 문서에서 각 참조 하는 의사에 대 한 전화 번호를 추출 합니다. 설명을 만들 때 패턴 목록 유형을 선택 하 여 반환 될 것으로 예상 되는 다양 한 형식을 허용 합니다.

   ![전화 번호 패턴 목록](../media/content-understanding/pattern-list.png)

이 예제에서는 **0-9에서 임의의 숫자** 를 선택 합니다. 이 항목을 선택 하면 패턴 목록에서 사용 되는 각 "0" 값이 0부터 9 까지의 숫자로 인식 됩니다.

   ![0-9부터 시작 하는 모든 숫자](../media/content-understanding/digit-identity.png)

마찬가지로 텍스트 문자를 포함 하는 패턴 목록을 만드는 경우 **에는 a-z** 확인란을 선택 합니다. 이 항목을 선택 하면 패턴 목록에서 사용 되는 각 "a" 문자가 "a"에서 "z" 까지의 문자로 인식 됩니다.

예를 들어 **날짜** 패턴 목록을 만들고 *2020 년 1 월 2* 일과 같은 날짜 형식이 인식 되도록 하려면 다음을 수행 해야 합니다.
- 패턴 목록에 *aaa 0, 0000* 및 *aaa 00, 0000* 을 추가 합니다.
- **A-z의 문자도** 선택 되어 있는지 확인 합니다.

   ![A-z의 문자](../media/content-understanding/any-letter.png)

또한 패턴 목록에 대문자화 요구 사항이 있는 경우 **정확히 대문자** 표시 확인란을 선택 하는 옵션이 있습니다. 날짜 예제에서 월의 첫 글자가 대문자로 시작 해야 하는 경우 다음을 수행 해야 합니다.

- 패턴 목록에 *Aaa 0, 0000* 및 *Aaa 00, 0000* 을 추가 합니다.
- 정확 하 게 **대/소문자** 도 선택 해야 합니다.

   ![정확한 대/소문자만](../media/content-understanding/exact-caps.png)

> [!NOTE]
> 패턴 목록 설명을 수동으로 만드는 대신, [설명 라이브러리]() 를 사용 하 여 *날짜*, *전화 번호*, *신용 카드 번호*등의 일반 패턴 목록에 미리 구성 된 패턴 목록 서식 파일을 사용 합니다. 

## <a name="proximity"></a>근접 

근접 설명 유형을 사용 하면 모델에서 다른 데이터에 얼마나 근접 하는지 정의 하는 방식으로 데이터를 식별할 수 있습니다. 예를 들어 모델에서 고객의 *주소 번호* 와 *전화 번호*에 레이블을 지정 하는 두 가지 설명을 정의 했습니다. 

또한 고객 전화 번호가 항상 주소 번호 보다 앞에 표시 되는 것을 볼 수 있습니다. 

Alex Wilburn<br>
555-555-5555<br>
One Microsoft Way<br>
Redmond, WA 98034<br>

근접 설명을 사용 하 여 문서에서 주소 번호를 보다 쉽게 식별할 수 있도록 전화 번호 설명의 거리를 정의 합니다.

   ![근접 설명](../media/content-understanding/proximity.png)</br>

#### <a name="what-are-tokens"></a>토큰 이란?

근사 설명 유형을 사용 하기 위해 토큰의 수는 근접 설명에서 한 설명 간의 거리를 측정 하는 방식을 이해 하는 것과 같습니다.  

토큰은 문자와 숫자의 연속 범위 (공백이 나 문장 부호 제외)입니다. 공간이 토큰이 아닙니다. 각 문장 부호 문자는 토큰입니다. 다음 표에서는 구의 토큰 수를 결정 하는 방법에 대 한 몇 가지 예를 보여 줍니다.

|어간|토큰 수|설명|
|--|--|--|
|`Dog`|1 |문장 부호 나 공백이 없는 단일 단어입니다.|
|`RMT33W`|1 |레코드 로케이터 번호입니다. 숫자와 문자를 포함할 수 있지만 문장 부호를 포함 하지는 않습니다.|
|`425-555-5555`|5 |전화 번호입니다. 각 문장 부호는 단일 토큰이 며  `425-555-5555` 5 개의 토큰으로 표시 됩니다.<br>`425`<br>`-`<br>`555`<br>`-`<br>`5555` |
|`https://luis.ai`|7 |`https`<br>`:`<br>`/`<br>`/`<br>`luis`<br>`.`<br>`ai`<br>|

#### <a name="configure-the-proximity-explanation-type"></a>근접 설명 유형 구성

예제의 경우 *전화 번호* 설명이 *주소 번호* 설명에서 가져온 토큰 수 범위를 정의할 수 있도록 근접 설정을 구성 합니다.

전화 번호와 주소 번호 사이에는 토큰이 없기 때문에 최소 범위는 "0"입니다.

그러나 예제 문서에 있는 일부 전화 번호에는 *(mobile)* 이 추가 됩니다.

## <a name="use-the-explanation-library"></a>설명 라이브러리 사용

설명에 대해 다양 한 패턴 목록 값을 수동으로 추가할 수 있지만 설명 라이브러리에서 제공 하는 미리 만들어진 서식 파일을 사용 하는 것이 훨씬 더 쉽습니다.

예를 들어, *날짜*에 대 한 모든 변형을 수동으로 추가 하는 대신 패턴 목록 서식 파일 *에 패턴을 사용*하 여 해당 값을 이미 많이 포함 하는 것이 좋습니다.</br>

   ![설명 라이브러리](../media/content-understanding/explanation-template.png)</br>
 
설명 라이브러리에는 다음을 비롯 하 여 일반적으로 사용 되는 패턴 목록 설명이 많이 포함 되어 있습니다.</br>

- 날짜</br>
- 날짜 (숫자)</br>
- Time</br>
- 숫자</br>
- 전화 번호</br>
- 우편번호</br>
- 문장의 첫 번째 단어</br>
- 신용 카드</br>
- 주민 등록 번호</br>

설명 라이브러리에는 다음과 같은 용어 목록 설명 템플릿도 포함 되어 있습니다.
- 문장의 끝
- 통화

#### <a name="to-use-a-template-from-the-explanation-library"></a>설명 라이브러리의 서식 파일을 사용 하려면

1. 모델 **학습** 페이지의 설명 섹션에서 **새로 만들기**를 선택한 다음 **템플릿에서**를 **선택 합니다.**</br>

   ![템플릿에서 만들기](../media/content-understanding/from-template.png)</br>

2.  **설명 템플릿** 페이지에서 사용 하려는 설명을 선택 하 고 **추가**를 선택 합니다.</br>

       ![서식 파일 선택](../media/content-understanding/phone-template.png)</br>

3. 선택한 서식 파일에 대 한 정보가 **설명 만들기** 페이지에 표시 됩니다. 필요한 경우 설명 이름을 편집 하 고 패턴 목록에서 항목을 추가 하거나 제거 합니다. </br> 

   ![서식 파일 편집](../media/content-understanding/phone-template-live.png)</br>

4. 완료 되 면 **저장**을 선택 합니다.
