# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR 간소화: 소규모 기업에 대 한 가이드

 *비즈니스에 대 한 Microsoft 365을 사용 하 여 GDPR 준수 완화 및 관리* 
  
GDPR (일반 데이터 보호 규정)은 조직에서 개인 데이터를 처리 하는 방법을 결정 하는 EU (유럽 연합) 규정입니다. GDPR은 5 월 25 2018 일에 적용 된 것으로 인해 발생 합니다. 비즈니스에서에 게 판매 하는 경우 유럽 연합 회사에 대 한 서비스를 제공 하거나 사용할 경우 [Gdpr](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) 이 귀하에 게 영향을 줍니다. 
  
이 문서는 GDPR의 개념, 제공 되는 이유 및 Microsoft의 비즈니스 365을 통해 GDPR을 준수 하는 데 도움이 되는 방법을 이해 하는 데 도움이 됩니다.
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR에 대 한 간략 한 개요

GDPR은 1995에서 초기에 이전 Data Protection 지시어 (DPD)를 업데이트 하 고 확장 하는 EU 규정입니다. GDPR은 개별 조직의 데이터에 대 한 개인 정보 (클라이언트, 고객, 직원 또는 비즈니스 파트너)와 관련 되어 있습니다. GDPR의 목표는 eu 내에 있는지 여부에 관계 없이 EU 시민에 대 한 개인 데이터 보호를 강화 하는 것입니다. 규정은 기대를 설정 하 고 기대치를 달성 하는 방법을 제안 합니다. GDPR이 지난 5 월 2018에 enforceable 되는 경우 조직에 GDPR의 요구 사항을 충족 하는 측정값이 있어야 합니다.
  
GDPR은 모든 데이터와이를 사용 하는 방법에 대해 설명 합니다. 데이터를 수명 주기가 있는 것으로 간주 합니다. 주기는 데이터를 수집 하 고, 저장 한 후 계속 해 서 사용 하 고 (처리 중) 시스템에서 완전히 삭제할 때 종료 됩니다. 
  
GDPR은 다음과 같은 유형의 데이터와 관련 되어 있습니다. 
  
- **개인 데이터:** 데이터를 개별 사용자에 게 연결 하 여 식별할 수 있는 경우 해당 데이터는 GDPR과 관련 하 여 개인으로 간주 됩니다. 개인 데이터의 예로는 이름, 주소, 생년월일 및 IP 주소가 있습니다. GDPR은 데이터가 개별에 연결 될 수 있는 경우 데이터의 모호 하거나 기술적에 관계 없이 "익명" 정보 라고도 하는 인코딩된 정보를 개인 데이터로 간주 합니다.
    
- **중요 한 개인 데이터** 개인 데이터에 더 많은 세부 정보를 추가 하는 데이터입니다. 예를 들면 religion, 무역 union 멤버 자격, 인종 기원 원본 등이 여기에 포함 됩니다. 중요 한 개인 데이터에도 생체 인식 데이터 및 DNA가 포함 됩니다. GDPR에서는 중요 한 데이터에 개인 데이터 보다 더 엄격한 보호 규칙이 있습니다. 
    
## <a name="gdpr-terms"></a>GDPR 용어

일부 용어는 GDPR에서 자주 참조 됩니다. 이러한 용어를 이해 하는 것이 중요 합니다.
  
 **동의**
  
GDPR 상태: "개인 데이터의 처리는 mankind을 지원 하도록 디자인 해야 합니다." GDPR은 개인 데이터를 처리할 때 동의를 사용 하 여 이러한 목표를 달성 하려고 합니다. 이는 회사에서 전자 메일 메시지를 받을 수 있는지 여부를 고객에 게 묻는 간단한 작업입니다. 또한 마케팅에 대 한 데이터를 사용 하려는 경우에는 웹 사이트에서 더 이상 옵트아웃 (opt out) 확인란을 사용할 수 없습니다. "Clear affirmative act"를 사용 하 여 명시적 동의를 고려해 야 합니다. 또한 동의를 가져오거나 취소할 때의 레코드도 기록해 야 합니다.
  
 **데이터 주체 권한**
  
GDPR은 데이터 주체 권한을 설정 하며,이는 개인 데이터, 고객, 직원, 비즈니스 파트너, 클라이언트, 계약자, 학생, 공급자 등을 사용 하 여 다음을 수행 한다는 것을 의미 합니다.
  
- **데이터에 대해 알려 주어 야 합니다.** 사용자에 게 데이터 사용에 대 한 정보를 알려야 합니다. 
    
- **데이터에 대 한 액세스 권한이 있어야 합니다.** 사용자가 보유 한 모든 데이터 (예: 계정 액세스 권한 또는 일부 수동 방식)에 액세스할 수 있도록 해야 합니다. 
    
- **데이터 수정 묻기:** 사용자가 부정확 한 데이터를 수정 하도록 요청할 수 있습니다. 
    
- **삭제할 데이터를 요청 합니다.** ' 지울 '이 라고도 하는이 권한은 개인이 수집한 모든 개인 데이터를 사용 하거나 공유 하는 모든 시스템에서이를 삭제 하도록 하는 권한을 허용 합니다. 
    
- **제한 된 처리 요청:** 개인이 사용자에 게 데이터를 표시 하지 않거나 제한 하도록 요청할 수 있습니다. 그러나 특정 상황 에서만 적용 됩니다. 
    
- **데이터 이식성을 보유 하 고 있습니다.** 개인이 자신의 데이터를 다른 회사로 전송 하도록 요청할 수 있습니다. 
    
- **개체:** 개별 사용자는 직접 마케팅을 비롯 한 다양 한 용도로 사용 되는 데이터에 대 한 개체를 사용할 수 있습니다. 
    
- **프로 파일링을 포함 하 여 자동화 된 의사 결정을 내리는 것을 요청 합니다.** GDPR에서는 사용자 프로필에 데이터를 사용 하 고 해당 프로 파일링에 따라 결정을 자동화 하는 데 대 한 엄격한 규칙이 있습니다. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>GDPR에 대 한 일반적인 소규모 비즈니스 질문

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>저는 GDPR에 대해 걱정 해야 하는 유일한 proprietor?

GDPR은 사용자가 보유 한 직원 수가 아니라 처리 하는 데이터에 대 한 것입니다. 모든 규모의 회사, 심지어는 단독 proprietors 영향을 줍니다. 그러나 직원이 250 명 미만인 회사에는 축소 된 레코드 유지와 같은 일부 예외가 있지만 데이터 처리가 개별 사용자의 권한에 영향을 주지 않고 가끔씩 처리 해야 하는 경우에만 가능 합니다.
  
예를 들어 개인 없는 데이터를 처리 하는 작업은 제외 되거나 제한 된 측정값이 필요 합니다. 그러나 "특수 범주 중요 데이터"로 표시 되는 데이터를 처리 하는 경우에도이 데이터 처리를 기록해 야 합니다. "가끔씩 처리"에 대 한 정의는 모호 하지만 한 번 또는 드물게 사용 되는 데이터에 적용 하는 것이 좋습니다.
  
또한 수집한 개인 데이터가 보호 되는지도 확인 해야 합니다. 즉, 암호를 암호화 해야 하며,이를 통해 해당 액세스 권한을 최소한의 암호로 제어 해야 합니다. 보호를 사용 하지 않고 데스크톱의 스프레드시트에 고객 데이터를 보관 하면 GDPR 기대치가 충족 되지 않습니다. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>회사 웹 사이트가 GDPR 규격 인지 어떻게 확인할 수 있나요?

사용자에 게 표시 되는 첫 번째 질문은 다음과 같습니다. 사이트의 모든 위치에서 개인 데이터를 수집 하나요? 예를 들어 이름과 전자 메일 주소를 묻는 연락처 양식이 있을 수 있습니다. 마케팅 전자 메일을 보내려면 데이터를 사용 하는 대상을 정확히 설명 하는 ' 옵트인 ' 확인란을 추가 해야 합니다. 받는 사람이 해당 상자에서 마케팅 용도로 개인 데이터를 사용할 수 있는지 확인 하는 경우에만 가능 합니다.
  
또한 데이터를 저장 하는 데이터베이스가 보호 되는지 확인 합니다. 웹 호스팅 회사 또는 클라우드 저장소 공급 업체가이에 대해 권고를 받을 수 있습니다. 비즈니스를 위해 Microsoft 365을 사용 하는 경우 데이터 저장소는 GDPR 규격입니다. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>내 회사는 유럽에 속하지 않습니다. GDPR이 미국에 실제로 영향을 미칩니까?

GDPR은 EU 시민을 보호 하는 규정입니다. 회사에서 EU 시민을 처리 하 고 있거나 나중에이 경우에는 영향을 받게 됩니다. 이는 EU 상태에 사는 모든 시민과 다른 곳에 사는 모든 시민이 적용 됩니다.
  
다음 예제를 고려해 보십시오.
  
- EU 시민이 자동차를 고용 하는 미국 회사에서는 고객의 데이터를 수집 하 고 처리할 때 GDPR 요구 사항을 충족 해야 합니다. 회사는 고객의 데이터를 받아 데이터를 안전 하 게 저장할 수 있는 경우 동의 해야 합니다. 또한 고객이 모든 데이터 주체 권한을 적용할 수 있는지 확인 해야 합니다. 
    
- 호주 회사는 온라인으로 제품을 판매 하 고 사용자는 온라인 계정을 설정 합니다. GDPR 데이터 주체 권한 및 동의는 계정을 여는 EU 시민에 적용 됩니다. 이 회사에서는 고객이 모든 데이터 주체 권한을 적용할 수 있는지 확인 해야 합니다. 
    
- 국제 자선 기부금은 기부자에 대 한 데이터를 수집 하 고이를 사용 하 여 업데이트 및 기부 요청을 보낼 수 있습니다. GDPR 상태: ' ... 직접적인 마케팅 목적을 위한 개인 데이터 처리는 합법적인 관심을 위해 수행 된 것으로 간주할 수 있습니다. " 그러나 조직의 관심사가 데이터 주체의 우선을 다시 정의 한다는 것을 입증 하기 위한 책임은 조직에 있습니다. 회사 (또는이 경우에는 자선 기부 조직)는 항상 정보 제공, 명시적, 옵트인 동의를 받아야 합니다.
    
GDPR은 고객 데이터가 테두리를 가로질러 이동 하는 경우에도 적용 됩니다. 데이터 저장소에 클라우드 컴퓨팅을 사용 하는 경우에는 서비스가 완전히 GDPR 규격 인지 확인 해야 합니다. 데이터 저장소가 데이터 보호 품질이 낮은 위치에 있는 경우에는이를 복잡 하 게 만들 수 있습니다. 비즈니스를 위해 Microsoft 365을 사용 하는 경우 GDPR 요구 사항을 다루는 올바른 법적 문서가 제공 됩니다.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>물론 데이터를 수집 했지만 다른 회사에서이를 저장 합니다. 후크를 해제 하 시겠습니까?

GDPR에서 데이터를 수집 하는 경우 어느 정도까지 영향을 받습니다. GDPR은 데이터 프로세서와 데이터 컨트롤러의 개념을 포함 합니다. 
  
- **데이터 컨트롤러:** 개인 또는 조직 (조인트 컨트롤러를 가질 수 있음)은 데이터 수집 방법, 방법 및 이유를 결정 합니다. 다른 회사의 클라우드 서버를 사용 하 여 저장 될 수 있습니다. 예를 들어 고객 데이터를 수집 하는 웹 사이트는 컨트롤러입니다. 
    
- **데이터 프로세서:** 컨트롤러를 대신 하 여 데이터를 저장 하 고 요청이 있을 때 이러한 데이터를 처리 하는 개인 또는 조직입니다. 예를 들어 비즈니스 데이터 저장소를 위한 Microsoft 365 앱은 프로세서 역할을 하며 완전 GDPR과 호환 됩니다. 
    
    조직 또는 시스템은 컨트롤러와 프로세서 모두로 작동할 수 있습니다. 비즈니스에 대 한 Microsoft 365는 GDPR과 함께 작동할 수 있습니다.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>여전히 이전 고객에 게 마케팅 전자 메일을 보낼 수 있나요?

사용자가 연 수를 포함 하 여 고객에 게 consented 데이터를 사용 하는 것을 확인 해야 합니다. 이전에 동의한 승인 및 표시할 레코드가 표시 되었을 수 있습니다. 이 경우 마케팅을 계속 진행 하도록 설정 됩니다. 그렇지 않은 경우 마케팅을 계속 하려면 고객에 게 권한을 요청 해야 합니다. 이 작업은 일반적으로 고객에 게 요청 하는 전자 메일을 보내 사이트를 방문 하 여 향후 전자 메일을 받기 위해 동의 하는 옵션을 선택 합니다. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>새 직원을 초대할 때 GDPR에 대해 걱정 해야 하나요? 현재 직원

GDPR은 고객 데이터에만 영향을 주지는 않습니다. 직원 데이터에도 확장 됩니다. 새 recruits LinkedIn과 같은 소셜 미디어 플랫폼을 사용 하는 경우가 많습니다. Express 권한 없이 잠재적 초대 데이터를 저장 하지 않도록 합니다.
  
기존 직원과 새 직원 계약의 경우에는 계약 끝에 있는 서명이 특히 계약에서 affirmative이 아닌 절을 사용 하는 경우 동의를 가정 하지 않을 수 있습니다. 이 경우에는 절과 연결 된 명시적인 방식으로 동의를 캡처해야 합니다. 이는 직원 계약에 따라 결정 되는 것을 나타내지만, 어떤 경우에는 "합법적인 관심 분야"를 사용 하 고 직원 들이 데이터를 사용 하 여 수행 하는 작업을 확인할 수 있습니다.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>비즈니스를 위한 Microsoft 365을 사용 하 여 개인 정보 보호 문제 충족

GDPR을 준수 하는 것은 개인 데이터가 보호 되는지 확인 하는 것입니다. GDPR에는 디자인 및 기본적으로 개인 정보 보호 라는 개념이 있습니다. 즉, 데이터 보호는 두 번째 특성을 충족 하기 위해 시스템 및 제품에 "baked"가 되어야 합니다. 
  
규모가 큰 it와 마찬가지로 소규모 기업에서는 보안을 희생 하지 않고 편리 하 게 사용 해야 합니다. 비즈니스에 대 한 Microsoft 365는 직원이 300 명 미만인 회사를 위한 것입니다. 소규모 회사에서는 Microsoft 클라우드 기반 도구를 사용 하 여 비즈니스 생산성을 향상 시킬 수 있습니다. Microsoft 365 for business를 사용 하는 소규모 기업에서는 전자 메일, 문서 및 모임 및 이벤트를 관리할 수 있습니다. 또한 GDPR 준수에 필수적인 보안 조치와 장치 관리가 기본 제공 됩니다.
  
비즈니스에 대 한 Microsoft 365에서는 다음과 같은 방식으로 GDPR 프로세스를 지원할 수 있습니다.
  
- **검색:** GDPR 준수에 대 한 중요 한 단계는 보유 하 고 있는 데이터를 파악 하는 것입니다. 
    
- **관리:** 데이터에 대 한 액세스 제어 및 해당 사용 관리는 GDPR의 필수 요소입니다. 비즈니스에 대 한 Microsoft 365는 장치에 적용 하려는 정책을 기반으로 비즈니스 데이터를 보호 합니다. 장치 관리는 직원이 원격으로 작업 하는 연령에 중요 합니다. 비즈니스에 대 한 Microsoft 365에는 모든 장치에서 데이터를 보호 하도록 하는 장치 관리 기능이 포함 되어 있습니다. 예를 들어 회사의 모든 Windows 10 장치가 Windows Defender를 통해 보호 되도록 지정할 수 있습니다. 
    
- **보호:** 비즈니스에 대 한 Microsoft 365는 보안을 위해 디자인 되었습니다. 장치 관리 및 데이터 보호 컨트롤은 데이터를 안전 하 게 유지 하기 위해 원격 장치를 비롯 한 회사 네트워크에서 작동 합니다. 비즈니스용 Microsoft 365 Office 응용 프로그램의 개인 정보 설정 및 문서 암호화와 같은 컨트롤을 제공 합니다. Microsoft 365 for business를 사용 하 여 GDPR 준수 모니터링을 수행 하 여 적절 한 수준의 보호 설정이 있는지 확인할 수 있습니다. 
    
- **보고서:** GDPR은 보고에 많은 중점을 둡니다. 단일 직원이 포함 된 비즈니스 조차도 많은 양의 데이터를 처리 하는 경우에도 해당 절차를 문서화 하 고 보고 해야 합니다. 비즈니스용 Microsoft 365에서는 소규모 조직에 대 한 보고 요구 사항을 headache 합니다. 
    
    감사 로그와 같은 도구를 사용 하면 데이터 이동을 추적 하 고 보고할 수 있습니다. 보고서에는 수집 하 고 저장 하는 데이터를 분류 하 고, 데이터를 사용 하 여 작업 하 고, 데이터를 전송 하는 작업이 포함 됩니다. 
    
고객, 직원 및 클라이언트는 데이터 개인 정보 중요성을 보다 잘 이해 하 고 있으며, 이제 회사 또는 조직이 개인 정보를 존중 하 고 있는 것으로 기대 합니다. 비즈니스에 대 한 Microsoft 365에서는 비즈니스에 대 한 대규모 upheaval 없이 GDPR 준수를 달성 하 고 유지 관리할 수 있는 도구를 제공 합니다.
