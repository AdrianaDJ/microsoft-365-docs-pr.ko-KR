# <a name="teamwork--people-experiences"></a>팀 작업 – 사용자 환경

생산성 점수 조직은 조직에서 Microsoft 365 앱과 지원 되는 기술 환경을 사용 하는 방법에 대 한 메트릭으로 작업을 수행 하는 방식을 변환 합니다. 점수에는 사용자와 기술 경력 범주에 대 한&#39;s 조직의 성과를 반영 하 고 사용자의 성과와 같은 조직에 대 한 점수가 비교 됩니다. 팀 작업 범주는 사용자 환경에서 발생 하는 측정값의 일부입니다. 자세한 내용은 [생산성 점수 개요](productivity-score.md) 를 참조 하세요.

## <a name="why-your-org39s-teamwork-score-matters"></a>조직의&#39;s 팀 작업 점수가 중요 한 이유

조직의 생산성에 대 한 기본 기둥은 공통 목표가 있는 사용자 그룹이 서로 작업을 수행 하 고 집합적으로 성공적인 성공을 위해 일반적인 리소스를 사용 하는 경우입니다. 리서치는 공유 작업 영역에서 정보를 공유 하 고 공동 작업을 수행 하는 경우 일주일에 최대 4 시간을 절감할 수 있음을 나타냅니다. 이러한 사용자는 관련 문서를 찾고, 이전 토론의 컨텍스트를 찾고, 공유 목표를 달성 하는 데 사용할 수 있습니다. [증거](https://www.microsoft.com/microsoft-365/blog/wp-content/uploads/sites/2/2019/04/Total-Economic-Impact-Microsoft-Teams.pdf) 를 참조 하세요.

## <a name="how-we-calculate-the-score"></a>점수를 계산 하는 방법

이 범주에 대 한 주요 메트릭이 포함 된 환경에서 주요 통찰력을 제공 합니다. 그런 다음 아래에서 설명 하는 점수 매기기 프레임 워크를 사용 하 여 점수가 계산 됩니다.

### <a name="primary-insight"></a>기본 통찰력

기본 정보는 Microsoft 팀의 전자 메일 및 메시지를 사용 하 여 통신 하 고 공유 작업 영역에서 클라우드의 콘텐츠와 상호 작용 하는 모든 사용자를 대상으로 합니다. Microsoft 365 그룹은 사용자가 그룹 사서함으로 전자 메일을 보내고, SharePoint 팀 사이트에서 파일을 공유 하 고, Microsoft 팀을 통해 채널 메시지를 보낼 수 있는 기능을 사용 하 여 공유 작업 영역에 함께 제공 되는 토대입니다. 365

:::image type="content" source="../../media/teamwork-score.png" alt-text="팀 작업 점수에 대 한 기본 정보를 보여 주는 차트":::

1. **머리글:** 다음 활동 중 하나를 수행 하는 사용자의 주요 메트릭이 제공 됩니다.
      - Exchange를 통해 그룹 사서함으로 전자 메일 보내기
      - 팀을 통해 채널 메시지 보내기
      - SharePoint 팀 사이트에서 콘텐츠를 읽고 만듭니다 (집합적으로 콘텐츠 상호 작용을 지칭 함).

        공유 작업 영역 내부 또는 외부에서 다음 활동을 수행 하는 모든 사용자의 비율입니다.
        - Exchange를 통해 전자 메일 보내기
        - Microsoft 팀에서 메시지 (채팅 또는 채널 메시지) 보내기
        - OneDrive 또는 SharePoint에서 콘텐츠를 읽고 만듭니다.

            그리고 Exchange, Microsoft 팀 또는 SharePoint 서비스 중 하나 이상에 액세스할 수 있어야 합니다.

1. **본문:** 공유 작업 영역 내에서 작업을 진행 하 고 콘텐츠를 보내고 상호 작용 하는 방법에 대 한 자세한 정보를 제공 하 고 조직의 생산성에 대 한 긍정적인 결과를 얻을 수 있습니다.
2. **시각화 (현재 상태):**
      - 파란색 부분이 머리글에 표시 된 백분율을 나타내는 가로 막대입니다.
      - 헤더에 표시 된 백분율을 계산 하는 데 사용 되는 분수 (분자/분모)를 강조 표시 합니다.
        - 분자: Exchange를 통해 그룹 사서함으로 전자 메일을 보내거나 팀을 통해 채널 메시지를 보내거나 SharePoint 팀 사이트에서 콘텐츠를 읽고 만드는 사용자 수입니다.
        - 분모: Exchange를 통해 전자 메일을 보내거나 Microsoft 팀에서 메시지 (채팅 또는 채널 메시지)를 보내거나 OneDrive 또는 SharePoint에서 콘텐츠를 읽고 만들고 다음 서비스 중 하나 이상에 게 액세스 권한을 부여 하는 사용자 수: Exchange, Microsoft 팀, SharePoint
   - 키 메트릭의 피어 벤치 마크 값도 백분율로 표시 됩니다.
3. **팀 작업 관련 리소스 보기:** 도움말 콘텐츠를 보려면이 링크를 선택 합니다.

## <a name="explore-more-about-teamwork-in-your-organization"></a>조직의 팀 작업에 대해 자세히 알아보기

또한 조직의 사용자가 함께 작업 하는 방법에 대 한 추가 정보를 제공 합니다. 이러한 추가&#39;메트릭은 it에서 생산성 점수에 직접 기여 하지만, 조직에서 사용자가 작업 하는 방식에 대 한 추가 최적화를 지원 하기 위한 작업 계획을 세우는 데 도움이 됩니다.

### <a name="breakdown-of-how-people-engage-in-shared-workspaces"></a>공유 작업 영역에서 사용자가 참여 하는 방식 분석

:::image type="content" source="../../media/people-sharedworkspace-engagement.png" alt-text="팀 작업 점수에 대 한 기본 정보를 보여 주는 차트":::

1. **머리글:** 측정 되는 다양 한 유형의 팀 작업에 대 한 세부 분석 결과를 표시 합니다.
2. **본문:** 팀이 보다 효율적으로 작업할 수 있도록 공유 작업 영역에서 작업 하는 값에 대 한 정보를 제공 합니다.
3. **시각화:** 시각화는 다음과 같이 공유 작업 영역에서 콘텐츠를 교환 하거나 상호 작용 하는 사용자가 수행 하는 범위를 보여 줍니다.
      - **전자 메일 보내기** : 색 부분과 분수는 그룹 사서함에 전자 메일을 보내는 사용자의 백분율을 나타냅니다. 분수는 다음으로 구성 됩니다.
        - 분자: 지난 28 일 동안 그룹 사서함에 전자 메일을 보내는 사용자입니다.
        - 분모: 지난 28 일 동안 전자 메일을 보내는 사용자입니다. 이 그룹은 통신 생산성 점수가 주요 파악 되는 전자 메일을 전송 하는 것으로 표시 된 동일한 그룹과 동일 합니다.
      - **메시지 보내기** : 색 부분과 분수는 Microsoft 팀의 채널에서 메시지를 보내는 사용자의 백분율을 나타냅니다. 분수는 다음으로 구성 됩니다.
        - 분자: 지난 28 일 동안 채널 메시지를 보내는 사용자입니다.
        - 분모: 지난 28 일 동안 채팅 또는 채널 메시지를 보내는 사용자입니다. 이 그룹은 통신 생산성 점수가 기본적으로 파악 되는 Microsoft 팀에서 메시지를 전송 하는 것으로 표시 되는 것과 동일 합니다.
    - **콘텐츠 만들기** : 색 부분과 분수는 Microsoft 365 SharePoint 팀 사이트에서 콘텐츠를 읽거나 만드는 사용자의 백분율을 나타냅니다.
        - 분자: Microsoft 365 그룹에 연결 된 팀 사이트에서 콘텐츠를 읽거나 만드는 사용자 수입니다.
        - 분모: 지난 28 일 동안 OneDrive 또는 SharePoint 사이트에서 모든 종류의 콘텐츠를 읽거나 만든 SharePoint에 대 한 액세스 권한이 있는 사용자 수입니다.
4. **관련 콘텐츠 보기:** 도움말 콘텐츠를 보려면이 링크를 선택 합니다.

### <a name="breakdown-of-workspace-engagement-by-size-and-age"></a>크기 및 연령에 따라 작업 영역 계약 분석

:::image type="content" source="../../media/workspace-engagement-bysizeage.png" alt-text="팀 작업 점수에 대 한 기본 정보를 보여 주는 차트":::

1. **머리글:** 작업 영역의 구성원 수 및 작업 영역 보존 기간 (월)에 따라 작업 영역의 계약 범주를 표시 합니다.
2. **본문:** 조직의 사용자가 보다 효율적인 팀 작업을 승격 하는 데 필요한 작업 영역만 유지 하도록 하는 값에 대해 설명 합니다.
3. **시각화:** 계약 분석은 두 차원에 걸친 열 지도 형태로 표시 됩니다.

      - **작업 영역 크기:** 작업 영역은 구성원 수: n: 2-10 사람, 11-100 명 및 100 이상 사용자에 따라 세 가지 범주로 분류 됩니다. &quot;모든 범주에는 &quot; 모든 크기 범주가 포함 됩니다.
      - **작업 영역 보존 기간:** 작업 영역은 처음으로 만들어진 후의 개월 수로 분류 됩니다. &quot;All 범주에는 &quot; 모든 연령 범주가 포함 됩니다.

        차트의 각 셀에는 해당 종류에 속한 작업 영역의 백분율에 따라 숫자와 색이 지정 됩니다. 작업 영역 범주는 해당 셀의 교차 부분에 표시 되는 기간과 크기를 기반으로 합니다. 예를 들어, 11-100 사용자와 4-12 월의 값이 52% 인 경우,이는 해당 작업 영역의 52%가 11-100에서 이전 달 사이에 있는 4-12 구성원을 포함 하는 것을 의미 합니다. 백분율은 다음과 같이 계산 됩니다.

        - **분자:** 통신 (전자 메일 및 채널 메시지) 형태의 계약이 있는 작업 영역 또는 지난 28 일 동안의 콘텐츠 상호 작용
        - **분모:** 지난 28 일 동안 조직에서 사용할 수 있는 모든 작업 영역입니다.

1. **관련 콘텐츠 보기:** 도움말 콘텐츠를 보려면이 링크를 선택 합니다.

### <a name="breakdown-of-workspaces-by-level-of-engagement"></a>계약 수준별로 작업 영역 분석

:::image type="content" source="../../media/workspace-by-engagement.png" alt-text="팀 작업 점수에 대 한 기본 정보를 보여 주는 차트":::

1. **머리글:** 그룹 전자 메일, 채널 메시지, 콘텐츠 상호 작용을 사용 하 여 계약 수준으로 분류 된 작업 영역을 분석 합니다.
2. **본문:** 팀 작업에서 보다 효율적으로 작업할 수 있도록 공유 작업 영역의 일관성 있는 서비스 값에 대 한 정보를 제공 합니다.
3. **시각화:** 주당 계약 강도에 따라 조직의 작업 영역에 대 한 보기를 제공 합니다. 이 보기에는 다음 범주를 포함 하는 모든 계약 외에도 팀 작업 내에서 측정 되는 다양 한 활동 유형에 대 한 분포가 포함 되어 있습니다.
      - **그룹 전자 메일:** 지난 28 일 동안 매주/1 일/2-3 일/4 일 + 하루 그룹 전자 메일 활동을 포함 하는 작업 영역의 비율입니다.
      - **채널 메시지:** 지난 28 일 동안 주간/1 일/2-3 일/4 일 + 하루 채널 메시지를 포함 하지 않는 작업 영역의 비율입니다.
      - **콘텐츠 읽기 또는 만들기:** 지난 28 일 동안 매주/1 일/3 일/4 일이 포함 된 작업 영역의 콘텐츠를 읽거나 만들 수 있습니다.
4. **관련 콘텐츠 보기:** 도움말 콘텐츠를 보려면이 링크를 선택 합니다.

### <a name="use-of-teams-within-microsoft-teams"></a>Microsoft 팀 내에서의 팀 사용

:::image type="content" source="../../media/useof-teams-within-teams.png" alt-text="팀 작업 점수에 대 한 기본 정보를 보여 주는 차트":::

1. **머리글:** Microsoft 팀 팀이 연결 된 공유 작업 영역의 수를 표시 합니다.
2. **본문:** 공유 작업 영역에 Microsoft 팀 팀이 연결 되어 있는 사용자가 팀 작업에서 보다 효율적으로 연결할 수 있도록 하는 값에 대 한 정보를 제공 합니다.
3. **시각화:** 도넛형 차트에서 색칠 된 부분은 Microsoft 팀 팀이 연결 된 작업 영역의 비율을 반영 합니다. 백분율은 다음과 같이 계산 됩니다.

      - 분자: 지난 28 일 동안 Microsoft 팀 팀이 연결 했던 조직의 공유 작업 영역 수입니다.
      - 분모: 지난 28 일 동안의 조직에 있는 공유 작업 영역의 수입니다.

        도넛형 차트의 가운데에 있는 숫자는 Microsoft 팀 팀이 연결 되어 있는 총 공유 작업 영역 수를 나타냅니다.
4. **관련 콘텐츠 보기:** 도움말 콘텐츠를 보려면이 링크를 선택 합니다.

## <a name="people-in-your-organization"></a>조직 내부 사용자

이 섹션에서는 페이지의 모든 통찰력과 관련 된 작업에 대 한 사용자 수준의 세부 정보를 제공 하 여 중점적으로 확인할 메트릭에 대해 설명 합니다.

사용자 수준에서 다음 열이 표에 표시 됩니다.

- **Username:**  은 사용자의 전자 메일 주소입니다.
- **표시 이름:**  은 사용자의 전체 이름입니다.

 - **총 구성원:** 작업 영역의 구성원 수입니다.
 - **만든 날짜:** 작업 영역을 만든 날짜입니다.
 -  **팀 팀의 경우:** 작업 영역에 연결 된 Microsoft 팀 팀이 있는지 여부를 나타내는 Boolean 값입니다.
 - **활동을 포함 하는 일:** 작업 영역에 지난 28 일 동안 연결 된 SharePoint 사이트의 그룹 전자 메일, 채널 메시지 또는 파일 활동을 포함 한 활동이 있는 일 수입니다.

 - **콘텐츠 읽기 활동을 사용한 일:** 작업 영역에 지난 28 일 동안 연결 된 SharePoint 사이트에서 액세스 한 Office 파일 (예: Word, Excel, PowerPoint, OneNote 또는 PDF)이 있는 기간 (일)입니다.

 - **모든 활동을 가진 사용자:** 지난 28 일 동안 연결 된 SharePoint 사이트에서 그룹 전자 메일, 채널 메시지 또는 파일 활동을 포함 하는 활동을 수행한 사용자 수입니다.
 - **전자 메일 활동을 가진 사용자:** 지난 28 일 동안 작업 영역에서 그룹 전자 메일 활동을 수행 하는 사용자 수입니다.
 - **채널 활동을 가진 사용자:** 지난 28 일 동안 작업 영역에서 채널 메시지를 보내는 사용자 수입니다.
 - **콘텐츠 읽기 활동을 가진 사용자:** 연결 된 SharePoint 사이트에서 최근 28 일 동안 Office 파일에 액세스 한 사용자 수입니다.

## <a name="notes"></a>참고

- 공유 작업 영역에 대 한 모든 참조에는 Microsoft 365 그룹과 연결 된 팀 팀, SharePoint 팀 사이트 및 그룹 사서함이 포함 됩니다.
- 2 ~ 500 구성원을 포함 하는 공유 작업 영역만 생산성 성과에 팀 작업에 대 한 것으로 간주 됩니다.
- 팀 작업에서는 조직 내 사용자의 계약을 측정 합니다. 외부 구성원의 활동을 캡처할&#39;없습니다.

- 해결 하는 데 사용 하는 통신 및 팀 작업 범주 간의 전자 메일 메트릭과 채널 메시지 사이에는 약간의 차이점이 있을 수 있습니다.
