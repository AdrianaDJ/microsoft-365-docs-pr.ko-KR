# <a name="microsoft-productivity-score"></a>Microsoft 생산성 점수 

생산성 점수는 조직이 Microsoft 365 및이를 지 원하는 기술 환경을 사용 하는 방법에 대 한 정보를 사용 하 여 디지털 변환으로의 여행을 지원 합니다. 조직의 점수가 사용자 및 기술 경험 측정법을 반영 하며, 크기와 비슷한 조직의 벤치 마크와 비교할 수 있습니다.

다음과 같은 기능을 제공 합니다.

- **메트릭은** 디지털 변환 여행에서 해당 위치를 확인 하는 데 도움이 되는 정보입니다.
- 조직의 생산성 및 만족도를 향상 시킬 수 있는 기회를 식별 하는 데 도움이 되는 데이터 **에 대 한 정보를** 소개 합니다.
- 조직에서 Microsoft 365 제품을 효율적으로 사용 하기 위해 수행할 수 있는 **권장 작업** 입니다.

Microsoft는 다음과 같은 두 가지 분야에서 메트릭, 통찰력 및 권장 사항을 제공 합니다. 

- **사용자 환경:** Quantifies에서는 콘텐츠 공동 작업, 모바일, 통신, 모임 및 팀 작업 같은 Microsoft 365 범주를 사용 하 여 조직의 작업을 수행 하는 방법을 설명 합니다.  

    각각의 설명 된 범주에 대해 공개 조사를 통해 몇 가지 모범 사례와 관련 혜택을 파악 하 고 조직의 효율성을 형성 합니다. 예를 들어 [Forrester] ( https://vc2prod.blob.core.windows.net/vc-resources/TEIStudies/TEI%20of%20Microsoft%20365%20E5%20-%20Oct%202018.pdf 사용자가 첨부 파일을 전자 메일로 보내는 대신 클라우드에서 콘텐츠를 공동으로 작업 하 고 공유 하는 경우에는 한 주에 최대 100 분까지 절감할 수 있음을 보여 줍니다. 또한 조직에서 이러한 모범 사례 사용을 정량화 하 여 디지털 변환 여행에서의 위치를 확인 하는 데 도움을 드립니다. 

- **기술 환경:** 조직은 안정적이 고 잘 작동 하는 기술 및 Microsoft 365의 효율적인 사용에 의존 합니다. [끝점 분석](https://aka.ms/endpointanalytics) 은 하드웨어 및 소프트웨어의 성능 및 상태 문제에 따라 조직이 영향을 받는 방식을 이해 하는 데 도움이 됩니다. Microsoft 365 apps health를 사용 하 여 조직의 장치가 권장 채널에서 Microsoft 365 앱을 실행 하는지 여부를 파악할 수 있습니다.

개요와 필수 구성 요소 세부 사항에 대한 자세한 내용은 [엔드포인트 분석이란](https://docs.microsoft.com/mem/analytics/overview)을 참조하세요. Microsoft 365 네트워크 연결 인사이트에 대한 자세한 내용은 [네트워크 연결 개요](https://docs.microsoft.com/microsoft-365/enterprise/microsoft-365-networking-overview)를 참조하세요.
  

## <a name="how-the-score-is-calculated"></a>점수가 계산 되는 방식

생산성 점수는 사용자 및 기술 경험 범주에 대한 총 점수를 기준으로 합니다. 각 범주는 총 100포인트의 동등한 가중치를 가집니다. 가능한 최대 생산성 점수는 800점입니다.

### <a name="score-categories"></a>점수 범주 

- 커뮤니케이션(100포인트)
- 모임(100포인트)
- 콘텐츠 공동 작업(100포인트)
- 팀워크(100포인트)
- 이동성(100포인트)
- 엔드포인트 분석(100포인트)
- 네트워크 연결(100포인트)
- Microsoft 365 앱 상태(100포인트)
- **가능한 총 합계 = 800포인트**
 
 각 점수 범주에서는 조직이 디지털 변환에서 Microsoft 365을 사용 하는 방법에 대 한 주요 지표를 수량화 합니다. Microsoft는 주요 활동에 대한 28일 및 180일 보기를 제공합니다. 또한 점수 계산에 포함 되지 않는 지원 메트릭을 제공 하지만 주소를 지정할 수 있는 기본 사용 통계 및 구성을 식별 하는 데 도움이 되는 것이 중요 합니다.

### <a name="products-included-in-productivity-score"></a>생산성 점수에 포함된 제품 

생산성 점수는 Exchange, SharePoint, OneDrive, Teams, Word, Excel, PowerPoint, OneNote, Outlook, Yammer 및 Skype의 데이터를 포함합니다.

조직의 점수가 매일 업데이트 되며 마지막 28 일 (현재 날짜 포함)에서 완료 된 사용자 작업을 반영 합니다.


## <a name="pre-requisites"></a>필수 구성 요소 

사용자 경험 데이터의 경우 비즈니스용 Microsoft 365 또는 Office 365 enterprise 구독이 필요합니다. 테넌트에 대한 엔드포인트 분석 데이터의 경우 구독에 Microsoft Intune을 추가해야 합니다. Intune은 장치와 앱을 관리하여 조직의 데이터를 보호하는 데 도움이 됩니다. Intune을 설치하면 Intune 환경에서 엔드포인트 분석을 켤 수 있습니다. 자세한 내용은 [Microsoft Intune](https://docs.microsoft.com/mem/intune/)을 참고 하세요. 
> [!NOTE]
> 생산성 점수 기능을 사용하기 위해 Workplace Analytics에 대한 라이선스는 필요하지 않습니다.

생산성 점수는 Microsoft 365 관리 센터 에서만 사용할 수 있으며, 다음 역할 중 하나를 가진 IT 전문가가 액세스 하는 것만 가능 합니다.  

- 전역 관리자 
- Exchange 관리자
- SharePoint 관리자 
- 비즈니스용 Skype 관리자 
- Teams 관리자 
- 전역 읽기 권한자 
- 보고서 구독자 

이 정보는 Microsoft 365를 사용 하는 furthering 디지털 변환에만 사용 되므로 신중 하 게 공유 해야 합니다. 

Microsoft는 개별 개인 정보를 보호 하기 위해 최선을 다하고 있습니다. 이 [개인 정보 문서](privacy.md)  에서는 Microsoft에 제공 하는 트러스트를 손상 시 키 지 않고 정보를 제대로 사용할 수 있도록 조직의 IT 관리자가 제공한 컨트롤에 대해 설명 합니다.

**보고서** > **생산성 점수** 의 Microsoft 365 관리자 홈에서 경험에 액세스할 수 있습니다.

## <a name="interpreting-your-organizations-productivity-score"></a>조직의 생산성 점수 해석 

생산성 점수 홈 페이지에는 조직의 총 점수와 점수 기록 및 각 범주에 대 한 주요 통찰력이 표시 됩니다.

:::image type="content" source="../../media/prodscore-landing.png" alt-text="보고서의 생산성 점수 페이지.":::

**조직의 점수가** 백분율 값과 포인트 단위로 표시 됩니다. 분자의 포인트와 분모의 가능한 최대 포인트를 볼 수 있습니다.

**피어 벤치 마크** 를 사용 하면 조직의 성과를 원하는 조직과 비교할 수 있습니다. 사용자 경험 범주의 피어 벤치마크는 유사한 조직 집합 내의 측정값 평균으로 계산됩니다. 조직 집합은 라이선스가 있는 사용자, 라이선스 유형, 산업, Microsoft 365 사용 기간 수가 유사한 귀하 지역의 조직으로 구성됩니다. 

엔드포인트 분석 피어 벤치마크에는 장치 시작 성능 및 모든 테넌트의 집계된 중간값 값을 기반으로 권장되는 소프트웨어 구성에 대한 대상이 포함됩니다.

네트워크 연결의 경우 권장되는 벤치마크는 80포인트입니다.

**점수 분석** 섹션에서는 사용자 및 기술 경험 영역별로 벤치마크와 함께 생산성 점수 분석을 제공합니다.

점수 기록에서는 지난 6개월 동안 각 범주의 점수가 어떻게 변했는지를 표시합니다.

**사용자 경험** 과 **기술 경험** 영역에는 해당 영역의 범주에 대한 주요 인사이트가 포함되어 있습니다. 각 범주를 클릭하여 세부 인사이트를 확인할 수 있습니다.
